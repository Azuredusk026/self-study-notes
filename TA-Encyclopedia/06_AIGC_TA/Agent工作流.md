# Agent工作流

## 定义与解释

Agent 工作流是让模型或脚本按目标拆解任务、调用工具、检查结果并迭代输出的自动化流程。它在 TA 场景中常用于资料整理、批量标注、规范检查和资产处理辅助。

## 核心原理

Agent 工作流的核心不是让模型自由发挥，而是把任务拆成受控步骤：读取上下文、选择工具、执行动作、验证结果、记录状态和交付产物。每一步都需要边界、权限和回滚策略。

用于 AIGC TA 时，Agent 应服务可追踪流程，例如生成素材筛选、Caption 初稿、文件命名检查或报告汇总。涉及外部资料、版权、删除文件和批量改资产时必须有人类确认，不能把模型输出当成最终事实。

## 用途

- 在 AIGC 资产流程中定位与 Agent工作流 相关的可复现性、质量、风格一致性或审核风险。
- 把生成过程转成可记录、可复跑、可比较的工作流，而不是只保存单张结果图。
- 连接概念探索、参考生成、DCC 精修、引擎导入和来源审核，避免临时素材直接混入正式资产。

## 常见误区

1. 让 Agent 直接决定正式资产是否入库。
2. 没有记录工具输入输出，导致结果不可追溯。
3. 把模型生成的检查结论当作确定事实。
