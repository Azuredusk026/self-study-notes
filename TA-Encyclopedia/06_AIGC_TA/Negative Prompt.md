# Negative Prompt

## 一句话定义

Negative Prompt 用来描述生成结果中应尽量避免的内容、缺陷或风格偏差。

## 为什么需要它

生产中常见问题包括多余手指、错误文字、水印、低清晰度、风格跑偏、材质脏污和不符合项目美术方向。负向提示词可以作为质量控制的一部分，但不能替代审核。

## 核心原理

负向提示词作为条件输入参与采样，引导模型远离某些特征。不同模型、采样器和工作流对负向词的敏感度不同。

> 待核验：具体模型对 Negative Prompt 的响应差异需要按模型和工作流测试。

## 技术美术中的典型用途

- 批量图标生成时排除文字、水印和复杂背景。
- 角色图排除多余肢体、错误比例和不需要的风格。
- 材质参考排除噪点、脏污或非 PBR 风格。

## Unity 中的相关场景

生成 UI 图标、贴图参考或道具草图时，负向词可帮助减少后期清理成本，但导入前仍需检查透明边缘、尺寸和压缩。

## Unreal Engine 中的相关场景

用于环境概念和 VFX 参考时，可以排除与项目调性冲突的元素，保持风格一致性。

## 常见误区

1. 负向词越多越好，导致结果变僵或影响主题。
2. 用负向词解决本应由 ControlNet 或参考图控制的问题。
3. 不记录失败样例，团队重复踩坑。

## 面试可能怎么问

### Negative Prompt 能解决什么，不能解决什么？

回答要点：它能降低某些不希望特征出现的概率，但不能保证结构、版权、风格和项目规范全部正确。

## 实践建议

建立一份项目负向词清单，按图标、角色、场景、材质四类维护，不同任务不要混用一套长清单。
