# IP-Adapter

## 一句话定义

IP-Adapter 是一种使用图像参考来影响生成结果的控制方法，常用于风格、角色或构图参考。

## 为什么需要它

单靠文本很难稳定表达一个角色、道具或风格。图像参考能让生成更接近已有方向。TA 需要关心它如何进入工作流、如何记录参考图来源、如何避免版权和风格污染。

## 核心原理

IP-Adapter 通常把参考图编码成额外条件，与文本条件共同影响生成过程。它更像“参考约束”，不是严格复制。

> 待核验：IP-Adapter 的具体节点、模型文件、适配模型和参数随工具链变化，需要查阅对应项目文档。

## 技术美术中的典型用途

- 角色风格参考。
- 图标风格统一。
- 道具造型参考。
- 与 [[ControlNet]] 组合：一个控制结构，一个控制参考风格。

## Unity 中的相关场景

可用于 UI 图标、角色头像、道具图标的风格统一，但输出仍要经过尺寸、透明和导入规范检查。

## Unreal Engine 中的相关场景

可用于环境概念、VFX 风格图和宣传视觉参考，最终仍需美术团队审核和重制。

## 常见误区

1. 把参考图当成可以无风险复制的素材。
2. 只加参考图，不管理 Prompt、Seed 和控制强度。
3. 参考图风格混乱，导致项目风格不稳定。

## 面试可能怎么问

### IP-Adapter 和 ControlNet 在控制上有什么区别？

回答要点：ControlNet 更常用于姿态、边缘、深度等结构控制；IP-Adapter 更偏图像参考和风格/身份约束。实际效果取决于模型和工作流。

## 实践建议

为一组技能图标选定 3 张合法参考图，测试不同参考强度下的风格一致性。
