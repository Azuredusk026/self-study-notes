# Diffusion Model

## 一句话定义

Diffusion Model 是通过逐步去噪生成图像、音频或其他数据的生成模型。

## 为什么需要它

AIGC 美术资产生成、风格探索、概念图、图标、材质参考和贴图修复，大量建立在扩散模型工作流上。TA 不需要把它当论文推导背诵，但需要理解“噪声、条件控制、采样、Seed、可控性和审核”这些工程问题。

## 核心原理

训练阶段模型学习如何从带噪数据恢复干净数据；生成阶段从随机噪声开始，在文本、图像、姿态、深度等条件约束下逐步去噪，得到结果图。

> 待核验：具体模型结构、训练细节和最新能力随模型版本变化，需要查阅对应官方文档或论文。

## 技术美术中的典型用途

- 概念图和风格探索。
- UI 图标、道具草图、材质参考。
- 与 [[ControlNet]]、[[LoRA]]、[[IP-Adapter]] 组合控制结构和风格。
- 建立可复用的 [[ComfyUI]] 工作流。

## Unity 中的相关场景

Unity 项目中，扩散模型结果通常不会直接变成最终资产，而是进入图标、贴图、概念稿、材质参考或工具链，再经过人工审核和引擎导入。

## Unreal Engine 中的相关场景

Unreal 项目中可用于概念设计、材质参考、贴花、UI 和宣传资产前期探索。落地时仍要经过分辨率、压缩、版权和风格一致性检查。

## 常见误区

1. 以为生成图可以直接进项目，不需要美术审核。
2. 只关注 Prompt，不管理 Seed、模型、工作流和版本。
3. 忽略版权、训练数据来源和团队风格一致性。

## 面试可能怎么问

### TA 如何看待扩散模型在美术管线中的作用？

回答要点：它适合提升探索和批量生成效率，但 TA 需要负责工作流搭建、资产规范、质量审核、自动化导入和风险边界。

## 实践建议

搭建一个“图标生成到 Unity 导入”的小流程：生成、筛选、统一尺寸、命名、压缩、导入和审核记录。
