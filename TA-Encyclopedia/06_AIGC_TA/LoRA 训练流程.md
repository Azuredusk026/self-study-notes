# LoRA 训练流程

## 一句话定义

LoRA 训练流程是从数据集准备、Caption、训练、验证到部署复用的一套轻量微调流程。

## 为什么需要它

项目想要稳定生成特定角色、道具风格或 UI 风格时，只靠 Prompt 往往不够。LoRA 可以让模型学习项目特征，但它也带来数据版权、过拟合、风格污染和质量审核问题。

## 核心原理

流程通常包括收集合法数据、清洗图片、统一尺寸、编写 Caption、设置训练参数、训练 LoRA、抽样验证、记录版本、上线工作流。

> 待核验：训练参数、学习率、分辨率、网络维度、训练轮数等强依赖训练框架、模型和数据集，不能写成固定推荐值。

## 技术美术中的典型用途

- 角色风格 LoRA。
- UI 图标风格 LoRA。
- 道具或材质参考 LoRA。
- 项目美术风格一致性实验。

## Unity 中的相关场景

可用于生成 Unity 项目的图标、头像、道具参考和贴图方向图，但最终资产仍需走导入规范和审核。

## Unreal Engine 中的相关场景

可用于 Unreal 项目的高保真概念参考、环境风格探索和 VFX 方向图，不能直接替代引擎资产制作。

## 常见误区

1. 数据集没有版权确认。
2. 数据太少或太脏导致过拟合。
3. 只看训练 loss，不做实际生成验证。
4. LoRA 版本和触发词不记录。

## 面试可能怎么问

### TA 负责 LoRA 训练流程时要关注什么？

回答要点：数据合法性、数据清洗、Caption 规范、训练记录、版本管理、生成验证、审核标准和引擎落地边界。

## 实践建议

用一组自制图标训练小型风格 LoRA，记录数据来源、Caption 规则、训练参数和失败样例。
