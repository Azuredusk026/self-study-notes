# LoRA 训练流程

## 定义与解释

LoRA 训练流程是从数据收集、清洗、标注、参数配置、训练、评估到版本管理的一套轻量模型微调流程。

## 核心原理

训练流程的核心是数据质量和可控变量。先确定目标概念，再准备授权数据，统一尺寸和裁剪，编写 Caption/Tag，设置触发词、训练轮数、学习率、网络维度和保存间隔，最后用固定测试 Prompt 评估。

TA 需要把训练产物当成资产管理：记录数据来源、配置、基础模型、训练脚本版本、样例图、适用范围和禁用场景。没有记录的 LoRA 很难复用，也很难排查版权和质量问题。

## 用途

- 在 AIGC 资产流程中定位与 LoRA 训练流程 相关的可复现性、质量、风格一致性或审核风险。
- 把生成过程转成可记录、可复跑、可比较的工作流，而不是只保存单张结果图。
- 连接概念探索、参考生成、DCC 精修、引擎导入和来源审核，避免临时素材直接混入正式资产。

## 与其他概念的区别

| 概念 | 区别 |
|---|---|
| [[AIGC 资产审核规范]] | 审核规范判断训练数据和产物能否入库；训练流程负责产出模型。 |
| [[Prompt Engineering]] | Prompt 用于测试和触发概念，不能替代训练数据质量。 |

## 常见误区

1. 只看单张样例好看就认为训练成功。
2. 不记录基础模型和参数。
3. 训练数据版权和授权状态不清。
