# 渲染管线

## 定义与解释

渲染管线是把场景数据转化为最终画面的阶段化流程，涵盖资源准备、可见性、几何处理、光栅化、着色、缓冲写入、光照、透明和后处理等环节。

## 核心原理

渲染管线的核心不是一条固定步骤，而是围绕数据如何在 CPU、GPU、引擎系统和图形 API 之间流转。场景对象先被剔除、排序和提交，GPU 执行顶点处理、光栅化、片元着色、深度/模板测试、混合和多 Pass 输出，最终经过后处理呈现。

不同管线会重新组织这些阶段：Forward、Deferred、URP、HDRP、移动端 Tile Renderer 或自定义 SRP 的 Pass 设计都不同。TA 需要能从 Frame Debugger/RenderDoc 看到具体项目的真实管线，而不是背一条抽象流程。

## 用途

- 在渲染调试中定位与 渲染管线 相关的画面异常、性能成本或资源配置问题。
- 为美术、TA 和图形程序建立统一术语，减少材质、灯光、后处理和管线配置沟通偏差。
- 把概念落到 Unity、Unreal 或 RenderDoc/Frame Debugger 中可观察的状态、缓冲、Pass 或材质参数上。

## 与其他概念的区别

| 概念 | 区别 |
|---|---|
| [[Render Pass]] | Render Pass 是具体阶段；渲染管线是这些阶段的整体组织。 |
| [[Vulkan]] | Vulkan 是底层 API；渲染管线是引擎和 GPU 工作流概念。 |

## 常见误区

1. 把教材管线图当成所有引擎的固定实现。
2. 只看 Shader，不看 Pass、队列、缓冲和后处理顺序。
3. 不使用 Frame Debugger 或 RenderDoc 验证实际执行路径。
