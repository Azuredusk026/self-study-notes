# 项目经历表达模板

## 结构

### 1. 项目背景

- 项目类型：
- 引擎：
- 平台：
- 团队规模：
- 你的角色：

### 2. 问题

用一句话说清楚当时遇到的具体问题。

示例：项目中角色技能特效 Overdraw 偏高，移动端战斗场景 GPU 时间不稳定。

### 3. 你的工作

- 你负责了什么？
- 你用了哪些工具或技术？
- 你和谁协作？
- 哪些部分是你独立完成的？

### 4. 结果

- 性能变化：
- 工作效率变化：
- 资产错误率变化：
- 团队使用情况：

没有量化数据时，不要编造。可以说“通过工具把检查流程前置到导入阶段，减少了人工逐项检查”。

## 可复用表达

```text
我在这个项目里主要负责 X。最初的问题是 Y，影响到了 Z。
我先用 A 定位问题，再做了 B 工具/方案，并和美术/程序确认了 C 规范。
最终这个方案让团队可以在 D 阶段发现问题，避免问题进入后续流程。
```

## 常见误区

1. 只说“参与了项目”，不说明自己的贡献。
2. 只列技术名词，不说明解决的问题。
3. 结果全是主观描述，没有证据。
