# DCC 工具开发面试题

## 问题 1：你会如何设计一个 Maya/Blender 批量导出工具？

### 考察点

- DCC 到引擎的流程理解
- 工具安全性
- 规范落地能力

### 推荐回答

我会先把导出规范拆成检查项，例如单位、轴向、冻结变换、命名、材质槽、Root、动画范围和导出路径。工具执行时先 dry-run，列出错误和警告；通过后再按配置导出 FBX，并写入日志。这样导出工具不只是“一键导出”，而是把规范前置到 DCC 阶段。

### 追问方向

- 如何处理导出失败？
- 如何支持不同资产类型？
- 如何避免破坏美术当前场景？

## 问题 2：DCC 工具 UI 需要注意什么？

### 考察点

- 工具可用性
- 面向美术团队的表达
- 错误处理

### 推荐回答

工具 UI 要让使用者知道当前选择、将要修改什么、有哪些风险和结果保存在哪里。批量修改类工具必须提供预览、确认、日志和失败原因。能自动修复的也要允许只检查不修改。

### 易错回答

- “脚本能跑就行。”
- “报错看控制台。”

## 问题 3：如何保证 DCC 工具长期可维护？

### 推荐回答

把规则、执行逻辑和 UI 分离；规则配置外置；工具输出稳定日志；关键操作有 dry-run；版本更新有 changelog；复杂逻辑写测试样例或固定测试资产。
