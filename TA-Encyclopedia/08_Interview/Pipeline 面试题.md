# Pipeline 面试题

## 问题 1：如何设计一个资源检查工具？

### 考察点

- 工程化思维
- 跨美术、程序、策划协作
- 工具可用性

### 推荐回答

先定义项目规范和检查优先级，再实现只读扫描和报告输出，明确错误、警告、建议三类问题。工具应能定位具体资产、说明失败原因、给出修复建议。自动修复要谨慎，最好支持预览、备份或撤销。

### 追问方向

- 哪些问题应该阻塞提交？
- 如何接入 CI？
- 如何避免误报让团队反感？

## 问题 2：FBX 导出规范要管哪些内容？

### 考察点

- DCC 到引擎的资产链路
- 常见生产事故
- 规范落地能力

### 推荐回答

需要管单位、轴向、Pivot、冻结变换、命名、层级、材质槽、法线切线、LOD、碰撞体、骨骼、动画帧率和导出预设。关键不是写一份文档，而是把规范做进导出工具和检查工具。

## 问题 3：Git LFS 适合解决什么问题？

### 考察点

- 二进制资产管理
- 版本控制理解
- 团队协作风险

### 推荐回答

Git LFS 适合管理大文件，避免普通 Git 历史被贴图、模型、音频等二进制资源撑大。但它不自动解决多人同时编辑二进制文件的问题，团队仍需要锁文件策略、目录规范和提交规则。
