# CI/CD for Game Assets

## 定义与解释

CI/CD for Game Assets 是把美术资产检查、转换、打包和报告接入持续集成流程的做法。它让资产问题在提交、合并或构建阶段自动暴露。

## 核心原理

资产 CI/CD 的核心不是简单跑脚本，而是定义触发时机、检查范围、规则版本、失败等级和报告格式。提交后系统拉取资源，执行命名、尺寸、依赖、导入、构建或性能规则，再把结果反馈给相关人员。

游戏资产和代码不同，文件大、二进制多、依赖复杂，且很多检查需要引擎或 DCC 无界面运行。TA 需要把检查设计成可增量、可解释、可白名单、可追溯的流程，否则 CI 会变成噪音或阻塞迭代。

## 用途

- 把 CI/CD for Game Assets 纳入资产生产、检查、导出、导入或版本管理流程，减少人工操作和沟通成本。
- 把团队规范转成可执行规则、工具入口或自动化报告，让问题尽早暴露。
- 连接 DCC、引擎、版本库和 CI/CD，让美术资产从制作到落地有可追踪的状态。

| [[目录规范]] | 目录规范提供规则；CI/CD 负责自动验证规则。 |

## 常见误区

1. 只在构建末尾检查资产，反馈太晚。
2. 错误信息不指向具体文件和规则，团队无法修复。
3. 把所有警告都设为阻塞，导致 CI 被绕过。
