# 批量重命名工具

## 一句话定义

批量重命名工具用于按项目命名规范快速修改大量资产、节点或文件名。

## 为什么需要它

命名不一致会影响搜索、自动导入、资源检查、打包规则和团队协作。TA 工具应把命名规范从文档变成可执行操作。

## 核心原理

工具通常支持前缀、后缀、编号、查找替换、正则规则、预览和冲突检查。安全工具必须支持 dry-run 和日志。

## 技术美术中的典型用途

- DCC 场景节点命名。
- 引擎资产命名。
- 贴图通道和后缀统一。
- 批量修复历史资源。

## Unity 中的相关场景

Unity 中重命名要保持 `.meta` GUID 稳定，避免直接在系统文件层破坏引用。

## Unreal Engine 中的相关场景

Unreal 中应通过编辑器 API 重命名资产，让引用重定向和 Redirector 正确处理。

## 常见误区

1. 直接改文件系统名称导致引用丢失。
2. 没有预览和冲突处理。
3. 命名规范只适合一种资产类型。

## 面试可能怎么问

### 批量重命名工具如何避免破坏引用？

回答要点：应使用引擎或 DCC 提供的重命名 API，先预览冲突，执行后保存日志并处理引用或 Redirector。

## 实践建议

做一个支持规则预览的批量重命名窗口，先列出旧名、新名、冲突和跳过原因。
