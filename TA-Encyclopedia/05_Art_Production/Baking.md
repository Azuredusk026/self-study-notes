# Baking

## 一句话定义

Baking 是后续需要扩充的技术美术相关主题。本页当前用于补全知识库双链，避免核心条目引用到不存在的页面。

## 为什么需要它

该主题已经在第一轮核心条目中被引用，说明它与渲染、Shader、引擎、资产生产或算法基础存在直接关系。

## 核心原理

- 输入：模型、拓扑、UV、贴图、骨骼、动画曲线、特效资源和命名规范。
- 处理过程：在 DCC、材质、导出和引擎导入阶段执行规范化、压缩、烘焙、绑定、LOD 或批量检查。
- 输出：符合项目规范的可用资产、性能指标、检查报告和可复用制作流程。
- 所在层级：DCC / Art Asset / Runtime Budget。

## 技术美术中的典型用途

- 制定并检查美术资产规范。
- 定位导入后表现差异和性能问题。
- 为角色、场景、动画、VFX 建立可复用流程。

## Unity 中的相关场景

常见于 Model Importer、Texture Importer、Animator、VFX Graph、Prefab、LODGroup 和资源检查工具。

## Unreal Engine 中的相关场景

常见于 Static Mesh/Skeletal Mesh 导入、Material Instance、Niagara、Animation Blueprint、LOD/Nanite 配置和资产审查流程。

## 与其他概念的区别

## 常见误区

1. 只记概念名，不确认它在项目中的输入、输出和所在管线阶段。
2. 把引擎默认效果当成固定标准，忽略渲染管线、平台和项目配置差异。
3. 没有保留可复现的测试场景，导致问题只能靠截图或主观描述沟通。

## 面试可能怎么问

### 问题 1

`Baking` 解决的核心问题是什么？

回答要点：先说明它在美术资产生产中处理哪类输入和输出，再结合一个项目场景说明为什么需要它。

### 问题 2

在 Unity 和 Unreal 中落地 `Baking` 时，TA 需要分别关注什么？

回答要点：比较两边的工具入口、资源规则、调试方式和平台限制，不要只背 API 名称。

### 问题 3

如果 `Baking` 相关效果或资产在项目中出问题，你会怎么排查？

回答要点：从资源输入、引擎配置、运行时状态、性能指标和最小复现场景逐层缩小范围。

## 实践建议

- 为 `Baking` 保留一个最小测试场景或示例资产，便于回归检查。
