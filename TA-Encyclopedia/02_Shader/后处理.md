# 后处理

## 定义与解释

后处理是在主要场景渲染完成后，对颜色、深度、法线或其他屏幕缓冲进行图像处理的阶段。它用于 Bloom、Tone Mapping、颜色校正、SSAO、描边、景深和风格化效果。

## 核心原理

后处理的核心是屏幕空间处理。一个或多个全屏 Pass 读取 Scene Color、Depth、Normal、Motion Vector 或自定义 Render Target，经过采样、卷积、阈值、混合或颜色映射，再写入新的 Render Target 或最终显示目标。

后处理对画面风格影响很大，也容易隐藏前面管线的问题。TA 需要检查执行顺序、HDR/LDR 空间、分辨率缩放、临时 RT、透明物体参与情况和移动端带宽。

## 用途

- 在材质或 Shader 调试中定位与 后处理 相关的画面异常、编译问题、性能成本或资源配置错误。
- 把概念落到 Unity ShaderLab/Shader Graph、Unreal Material Editor、RenderDoc 或引擎材质面板中可观察的参数和状态。
- 为美术暴露稳定的材质控制项，同时限制采样次数、变体数量、精度和平台差异带来的风险。

## 常见误区

1. 用后处理弥补灯光、材质或曝光基础问题。
2. 忽略透明物体和粒子是否写入所需缓冲。
3. 叠加多个全屏效果而不评估带宽和临时 RT 成本。
