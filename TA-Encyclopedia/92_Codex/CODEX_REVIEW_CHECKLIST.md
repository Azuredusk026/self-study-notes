# CODEX_REVIEW_CHECKLIST

每次 Codex 完成任务后，必须按以下清单检查。

## 内容检查

- 是否使用了正确模板？
- 是否有详细的定义与解释，而不是一句空泛定义？
- 核心原理是否基于条目自身机制展开，而不是套用固定的输入/处理/输出模板？
- 是否说明具体用途？
- 是否包含常见误区？
- 相关条目是否有真实关联，并写明关联原因？

## 结构检查

- 是否更新目录 README？
- 是否更新总目录？
- 是否更新术语索引？
- 是否避免重复条目？
- 是否没有破坏已有内容？

## 可信度检查

- 是否把不确定内容标记为 `待核验`？
- 是否把来源记录到 `91_Sources/source_registry.md`？
- 是否避免编造最新版本、API 行为或论文结论？

## Git 检查

- 是否只修改本任务相关文件？
- 是否运行检查脚本？
- 是否能清楚说明每个文件改了什么？
