# CODEX_REVIEW_CHECKLIST

每次 Codex 完成任务后，必须按以下清单检查。

## 内容检查

- [ ] 是否使用了正确模板？
- [ ] 是否有一句话定义？
- [ ] 是否解释了为什么需要该技术？
- [ ] 是否包含核心原理？
- [ ] 是否说明 TA 实际用途？
- [ ] 是否包含 Unity 相关场景？
- [ ] 是否包含 Unreal 相关场景？
- [ ] 是否包含常见误区？
- [ ] 是否包含面试问题？
- [ ] 是否添加相关条目链接？

## 结构检查

- [ ] 是否更新目录 README？
- [ ] 是否更新总目录？
- [ ] 是否更新术语索引？
- [ ] 是否避免重复条目？
- [ ] 是否没有破坏已有内容？

## 可信度检查

- [ ] 是否把不确定内容标记为 `待核验`？
- [ ] 是否把来源记录到 `91_Sources/source_registry.md`？
- [ ] 是否避免编造最新版本、API 行为或论文结论？

## Git 检查

- [ ] 是否只修改本任务相关文件？
- [ ] 是否运行检查脚本？
- [ ] 是否能清楚说明每个文件改了什么？
