# 程序化纹理与建模暨UV展图与材质绘制全流程解析

## 一、 核心知识点与原理剖析

### 1. 程序化纹理生成技术（Substance Designer）
* **PBR 工作流与高度图核心作用**
  * 在 PBR（Physically Based Rendering，基于物理的渲染）流程中，写实材质（如石头、砖墙、木头）的核心特征由表面高度的微观几何决定。高度图（Height Map）是所有纹理的逻辑起点，其亮度代表高度（白色代表高，黑色代表低）。
  * 尽管在实时游戏引擎中，由于性能开销限制，通常不直接将高度图拖入材质通道（因为视差偏移渲染或曲面细分网格顶点偏移极其消耗性能），本图依然是自动派生法线贴图、粗糙度贴图、环境光遮蔽图的基础。
* **法线贴图转换**
  * 通过 **Normal** 节点，将高度图的明暗过渡转换为计算着色（Shading）所需的法线方向向量（RGB 通道分别映射 XYZ 轴的法线向量）。
* **混合（Blend）与色阶重映射**
  * **Blend（混合）** 节点：在默认的复制（Copy）模式下等同于线性插值（Lerp）运算。通过调节 Opacity（不透明度）作为插值权重（$T$ 值）。此外支持加（Add）、减（Subtract）、乘（Multiply，即正片叠底）以及最大值（Max）等图层像素混合模式。
  * **Levels（色阶）** 节点：用于控制黑白对比度与灰度分布。通过调整中间灰（Middle Gray）的输入/输出映射，可以精细控制图层灰度过渡，常用于将高度图反转并重映射为粗糙度贴图（Roughness）。
* **Tile 模式与 Flood Fill（洪流填充）工作流**
  * **Tile Generator（平铺生成器）**：可以按照特定规则排列预设的图案（Pattern，如砖块、圆盘、方形等）。
  * **Flood Fill 节点**：能够自动识别图像中由黑色边界分割的闭合单元，将其转换为独立的坐标块。连接在该节点之后的派生节点包括：
    * **Flood Fill to Random Grayscale**：赋予每个砖块单元不同的随机高度值。
    * **Flood Fill to Gradient / Slope**：为每个单元随机分配不同的坡度与倾斜方向，用以模拟石板路、瓦片等铺设不平整的物理效果。
    * **Flood Fill to Color / HSL**：为各个单元进行随机的色相、饱和度与明度抖动。
  * **边缘模糊（Blur）对法线的重要作用**：若砖缝或图案边缘是生硬的 90 度折边，直接转法线会因为像素没有渐变而导致法线方向毫无过渡。通过接入一个微小的 **Blur** 节点（例如强度设置为 0.5 或 1.0），能使边缘形成缓坡，在光照渲染下即可呈现饱满圆润的立体边缘受光效果。

### 2. Houdini 程序化建模与 UV 展图原理
* **程序化建模优势**
  * Houdini 采用非线性的图结构建模方式。所有操作（如 Grid, Group, PolyBevel, PolyExtrude, Transform, PolyWire）皆为网络节点。这种工作流的优势在于可以随时返回历史节点调整参数，而不会破坏后续的操作流。
* **倒角（PolyBevel）工具**
  * 倒角工具支持点倒角、边倒角和面倒角。如需制作圆角矩形，可在 Point（点）模式下使用 **Group** 节点选定四个角点，在 PolyBevel 节点中引入该点组，并设置倒角类型为点倒角。
* **线挤出管道（PolyWire）的扭曲陷阱**
  * 利用 **Convert Line** 将模型的选定边缘提取为样条曲线，再使用 **PolyWire** 节点可直接将曲线扫掠成管状实体。
  * **注意：PolyWire 节点的 Division（细分数）参数必须为偶数（如 4, 6, 8 等）。若将其设为奇数，管道的侧面网格会出现扭曲并自旋 45 度的模型瑕疵。**
* **UV 展开的四步工作流**
  1. **切线/分缝（Seaming）**：规划网格切割线。利用 **Group** 节点（通过控制夹角、包围盒范围或手动选边）创建一个缝线组（例如命名为 `seam0`）。缝线通常应放置在模型隐蔽处（如底面或内侧）。
  2. **展开/拍平（Flattening/Unfolding）**：利用 **UV Flatten** 节点，将缝线组内的切口切开，并将三维网格表面平摊到二维 UV 空间。
  3. **松弛（Relaxing/Smoothing）**：通过松弛算法释放局部顶点的剪切应力，确保贴图棋盘格大小均匀无拉伸。
  4. **排版/布局（Layout）**：利用 **UV Layout** 节点，将各块 UV 孤岛（Islands）紧密、合理且无重叠地塞入 $[0,1]$ 的二维 UV 空间中。
* **纹素密度（Texel Density）一致性**
  * 统一的纹理像素密度对于规避游戏场景中“局部模糊，局部超清”的现象至关重要。应当通过 **Labs Texel Density** 节点统一模型单位物理尺寸（如每米）所对应的像素分辨率，然后再进行最终的排版布局。

### 3. Substance Painter 材质绘制原理
* **贴图烘焙（Mesh Map Baking）的基础作用**
  * Substance Painter 中的智能材质（Smart Materials）与生成器（Generators）均通过模型特有的几何特征图来进行驱动。这些几何特征图（曲率图、AO图、法线图、位置图等）需要通过烘焙生成。
  * 在没有高模的“中模直出流程”中，必须在烘焙设置中勾选 **"Use Low Poly Mesh as High Poly Mesh"（将低模网格用作高模网格）**，同时勾选高度图（Height）通道。
* **生成器工作原理**
  * **Curvature（曲率）**：提取凸起边缘（白色）和凹陷裂缝（黑色），常用于磨损、掉漆或高亮描边。
  * **Ambient Occlusion（环境光遮蔽/AO）**：模拟遮蔽阴影，光线不易照到的夹角与凹槽处呈现黑色，常用于泥土积灰和锈斑模拟。
  * **Mask Builder（遮罩生成器）**：集成了曲率、AO、随机噪声与垂直渐变的综合图层混合器，主要通过乘法等融合模式制造复杂的材质老化痕迹。
* **三面投影（Triplanar Projection）技术**
  * 传统的 UV 采样在面对斜切或非规则 UV 面时，极易造成纹理（如布料纤维、泥土砂砾）偏转扭曲。
  * 三面投影技术直接在三维空间中从 X、Y、Z 三个轴向上将纹理投射到模型表面，并在其交界处根据表面法线进行渐变融合，可完美解决纹理拉伸与物理接缝问题。
* **图层多通道与通道混合（Channel Blending）模式**
  * 每一个图层在 SP 中同时携带着 Color（颜色）、Height（高度）、Roughness（粗糙度）、Metallic（金属度）、Normal（法线）五种通道数据。
  * 不同的通道有着独立的混合逻辑：例如，颜色与粗糙度通常默认为 Normal 模式（直接覆盖下层数据）；而高度通道默认则是 Add 模式（相加）。如果误将上层高度通道修改为 Normal 模式，将会导致下层的所有起伏细节被粗暴抹平。

## 二、 重点术语与概念解析

* **PBR (Physically Based Rendering，基于物理的渲染)**：一套使用符合物理定律的数据（如金属度 Metallic、粗糙度 Roughness 等参数）来真实模拟光线与材质发生反射、折射、吸收等交互的着色技术。
* **Uniform Color (均一颜色)**：Substance Designer 中提供纯色填充输出的节点，支持切换为 Color（三通道彩色）与 Grayscale（单通道灰度）。
* **Levels (色阶)**：用于控制和重构黑白数值范围的图像调节节点。通过移动黑点、白点和中间灰滑块，可拉伸对比度、平移数值，或将高度图数据反转输出给粗糙度通道。
* **Blend (混合)**：接受前景、背景与遮罩输入的图层混合节点。支持乘法（正片叠底，保留暗部）、加法/最大值（叠加细节）和减法（刻画局部凹陷破损）等逻辑运算。
* **Normal Combine / Normal Blend (法线混合)**：专门用于融合两张法线贴图的特殊计算节点。因为法线贴图储存的是法向量而非普通颜色，使用传统的 Blend（如相加或正片叠底）会导致向量失真，必须通过法线混合节点重新计算其切线空间向量。
* **Flood Fill (洪流填充)**：将包含连通区块（如被黑线切分的格栅）的图像进行分析，输出每个隔离块的唯一坐标、倾斜度或灰度随机码的核心预处理节点。
* **UV Layout (UV 布局)**：自动排列 UV 孤岛的布局计算。包含 Island Padding（孤岛间距）和 Border Padding（边缘留空）等参数，目的是防范贴图采样溢出。
* **Texel Density (纹素密度)**：三维物体每单位世界尺寸所分配到的纹理像素像素量（如 512 像素/米）。是控制全场景材质精度一致的关键指标。
* **Triplanar Projection (三面投影)**：不借助 UV 坐标，而是在三维空间从世界的三个基轴方向把材质投影至网格表面，并在过渡区域产生法线加权混合的采样算法。
* **Ambient Occlusion (环境光遮蔽 - AO)**：一种描述全局光照下局部光线受阻强度的灰度贴图，通过在像素点打出半球面半随机射线检测碰撞，为模型死角和转弯处附带软阴影。

## 三、 工程经验与避坑指南

* **金属度（Metallic）“非黑即白”原则与 0.5 值的行业误区**
  * **工程经验**：在标准 PBR 材质规范中，自然界的物质通常要么是绝缘体（非金属，Metallic = 0.0，如泥土、石头、橡胶），要么是纯导体（金属，Metallic = 1.0）。
  * **避坑指南**：很多初学者在发现引擎光照反射不好看时，会尝试将金属度拉到 0.5 等中间数值来凑效果，这其实是在用错误的材质参数掩盖错误的引擎环境光照（如缺失反射探针或天空盒）。**在正规工作流中，除非是金属生锈过渡区、极脏的覆盖层、或是某些半导体材质，否则严禁使用 0.5 左右的金属度中间值。**
* **高度图破损细节丢失与叠加陷阱**
  * **工程经验**：在 SD 中使用减法（Subtract）去刻画红砖表面的破损凹坑时，如果底层砖块原本就是最黑处（0.0），减法将无法在更黑的值上产生任何变化，破损细节会直接被“切顶”丢失。
  * **避坑指南**：在做减法细节前，使用 Levels 节点将底层图案的暗部拉高（保留 0.1-0.2 的空间）。同时，**在往低洼处（如沙浆缝隙中）增加噪声或碎石细节时，不要使用普通的 Add（加法）或 Copy，应该选择 Max（最大值）混合模式，确保高出的砖面不被污染。**
* **Houdini 曲线生成管道的奇数边扭曲**
  * **工程经验**：在 Houdini 中通过 Convert Line 和 PolyWire 节点制作医疗包边角管道或锁扣线段时，常会出现网格自旋 45 度、管道走向怪异的问题。
  * **避坑指南**：**务必检查 PolyWire 节点的 Division 参数，必须设定为偶数。** 如果输入奇数，由于顶点环拓扑对齐算法问题，管道拐角网格必定会扭转，直接导致法线烘焙报废。
* **UV 边缘间距与 Mipmaps 导致的黑边渗色**
  * **工程经验**：在排版 UV 时，不能只图利用率而将 UV 孤岛塞得过满或死贴 UV 外边框。
  * **避坑指南**：在 UV Layout 节点中必须设定 **Island Padding**（如 5 到 8 像素）与 **Border Padding**。如果间距为 0，当引擎生成低分辨率的 Mipmaps 贴图时，双线性插值滤波会读取相邻 UV 块或外边界之外的像素，导致模型渲染时在接缝处出现明显的黑线或花屏。
* **SP 3D 直刷漏色与 2D UV 笔刷校准**
  * **工程经验**：在三维视口直接绘制锁扣、折边等精细金属部分时，笔刷的 3D 范围（球形笔刷投影）往往会穿透折角，将底面的布料通道也刷成金属。
  * **避坑指南**：**在 Brush 属性栏中，将 Alignment（校准）属性从默认的 Tangent（切线）切换为 UV。** 并在右侧的 2D UV 视图窗口进行绘制。此时笔刷将仅会在选定的 UV 孤岛内涂抹，彻底防止颜色溢出到其他网格块。
* **SP 智能材质高度通道“被覆盖”的排查手段**
  * **工程经验**：导入智能材质后，原本下层调好的法线细节（如布面纤维、网格编织）会突然变平，但图层 Mask 本身并无大范围遮挡。
  * **避坑指南**：这是因为某些智能材质层的高度/法线通道混合模式被置为了 Normal。**需要在图层面板左上角的通道下拉菜单（默认显示 Base Color）中切换到 Height 和 Normal 通道，逐一检查是否有非 Add 或非 Combine 模式的覆盖型图层。** 必须将其更改为 Linear Dodge/Add（线性减淡）才能使上下层的高度信息正确叠加。
