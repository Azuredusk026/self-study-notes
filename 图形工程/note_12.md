# 大气散射原理与实时渲染实现（基于 URP）

## 一、 核心知识点与原理剖析

### 1. 大气散射的分类与视觉表现
大气散射是体积光照效果的基础，主要负责渲染自然蓝天、夕阳黄昏、空中透视（远景泛蓝）以及太阳周围的光晕。依据参与介质的不同，分为两种主要散射：
*   **瑞利散射 (Rayleigh Scattering)：** 
    *   **介质：** 由高空大气中的空气分子（纯净气体）提供。
    *   **特性：** 散射强度与波长相关。波长越短（蓝光）受到的散射越强，弥漫在整个天空中，因此天空显蓝色；波长越长（红/绿光）穿透性越强。
    *   **视觉效果：** 在大气越厚的角度（如地平线附近看夕阳），蓝光被完全散射殆尽，直射入眼的只剩红黄光，产生夕阳的变红效果。决定了天空的基础颜色变化。
*   **米氏散射 (Mie Scattering)：**
    *   **介质：** 由低空的气溶胶、雾霾、PM2.5 提供。
    *   **特性：** 散射强度与波长基本无关（RGB三通道系数通常相等）。
    *   **视觉效果：** 主要影响低空雾的浓度和泛白效果，以及太阳周围的强烈光晕（因为其具有强烈的各向异性前向散射）。

### 2. 通用散射模型与消光公式
一束光穿过参与介质时，会受到四种效应的影响：**吸收 (Absorption)**、**自发光 (Emission)**、**外散射 (Out-scattering)**、**内散射 (In-scattering)**。
*   **消光 (Extinction)：** 光束强度的衰减，等于吸收与外散射之和。
*   **透过率 (Transmittance, $T$)：** 光经过介质吸收与外散射后，还能在当前方向上投射出的比率。
    *   **消光公式：** $T(x) = e^{-\alpha x}$。其中 $\alpha$ 为消光系数（材质属性），$x$ 为穿过的距离。
    *   **步进累乘特性：** 距离在指数项上，因此 $e^{-\alpha(x_1+x_2)} = e^{-\alpha x_1} \cdot e^{-\alpha x_2}$。在步进（Ray Marching）计算时，总透过率等于每一小段透过率的乘积。
*   **注意：** 本次实现仅涉及**单次散射（Single Scattering）**。即只计算阳光外散射一次进入人眼的内散射贡献，不包含光在大气中经过多次反射再进入人眼的多重散射。

### 3. 大气密度与尺度高度模型 (Scale Height Model)
消光系数与散射介质的密度成正比。为了计算不同高度的消光系数，需要构建大气的密度模型。
*   **核心假设：** 气体分子和气溶胶的密度仅随高度呈指数下降。
*   **公式：** $\rho(h) = e^{-h / H}$。
    *   $h$：当前高度。
    *   $H$：**尺度高度 (Scale Height)**，表示密度下降一个数量级所需的高度。
*   **经验数值（地球）：** 
    *   瑞利散射（气体）：尺度高度约为 8000 米。海平面散射系数通常给一个经验RGB向量（如偏蓝）。
    *   米氏散射（气溶胶）：尺度高度很低，约为 1200 米（雾气集中在地表）。海平面散射系数由单次散射反照率（Albedo）推导而来（因为米氏粒子存在吸收光能转化为热能的情况）。
*   **当前点消光系数计算：** `海平面消光系数 * exp(-h / H)`。

### 4. 离散步进积分 (Ray Marching) 与 透过率查找表 (Transmittance LUT)
我们需要计算从相机出发，沿视线穿过大气的每一段介质被阳光照亮并反射进人眼的颜色总和。
*   **单点光照贡献公式：** `太阳颜色 * Ts * 散射系数 * 相函数 * Tv * 步长`
    *   **$T_s$：** 当前点到太阳的大气透过率。
    *   **$T_v$：** 当前点到相机的累计透过率（由当前 Ray Marching 循环实时累乘计算）。
*   **预计算 $T_s$ (Transmittance LUT)：**
    *   如果每次采样光照都要向太阳方向再次步进计算 $T_s$，性能开销将呈几何倍数暴增（如 $O(N \times M)$）。
    *   **降维优化：** 无论在全球哪个坐标，到大气的透过率仅与**当前高度 $h$** 和 **天顶方向与光照方向的夹角 $\cos\theta$** 相关。
    *   使用 Compute Shader 预先将不同 $h$ 和 $\cos\theta$ 组合下的透射率积分结果，烘焙到一张 2D 纹理中（Look-Up Table）。运行时 $O(N)$ 步进时，直接采样该纹理获取 $T_s$。

### 5. 相函数 (Phase Function)
相函数决定了一束光打中粒子后，往各个方向散射能量的概率分布。
*   在代码实现中，**散射系数**与**相函数**是剥离的。
*   **瑞利相函数：** 依赖于入射光与出射光夹角的余弦值，前后向散射较多，侧向较少。
*   **米氏相函数 (Henyey-Greenstein 模型)：** 由于气溶胶散射呈现强烈的各向异性（Anisotropy），引入各向异性系数 $g \in [-1, 1]$。$g=0$ 为全向均匀散射，$g>0$ 为前向散射（产生太阳光晕），$g<0$ 为后向散射。
*   **最终散射能量合并：** `瑞利散射系数 * 瑞利相函数 + 米氏散射系数 * 米氏相函数`，分别计算后相加。

## 二、 重点术语与概念解析

*   **吸收系数 (Absorption) & 散射系数 (Scattering)：** 两者相加等于**消光系数 (Extinction)**。瑞利散射的吸收系数为 0，米氏散射存在吸收系数。
*   **反照率 (Albedo)：** 在米氏散射中，指散射系数占消光系数的比例。通常通过定义散射系数和反照率来倒推消光系数。
*   **Compute Shader (计算着色器)：** 利用 GPU 大量线程进行通用计算的技术。在预计算 LUT 时，每个线程负责计算纹理上的一个像素。
*   **线程组 (Thread Group)：** GPU 执行 Compute Shader 的基本调度单位。它具有三维 ID 结构，包含组内线程 ID (Group Thread ID) 和全局分发 ID (Dispatch Thread ID)。
*   **射线与球体求交 (Ray-Sphere Intersection)：** 数学函数，输入射线起点、方向以及球体中心和半径，输出射线穿过球体的两个交点距离（进入点和离开点）。

## 三、 工程经验与避坑指南

### 1. Compute Shader 与 LUT 生成避坑
*   **线程组大小对齐：** 在定义 `[numthreads(x, y, z)]` 时，建议线程组总数（$X \times Y \times Z$）无脑向 **64** 靠拢（例如二维贴图使用 `[numthreads(8, 8, 1)]`）。这是为了对齐底层显卡（A卡/N卡）计算单元中的 Wavefront/Warp 大小（通常为32或64），以避免性能浪费。
*   **读写纹理 (RWTexture2D) 限制：** 在 Compute Shader 中，用来写入结果的纹理必须声明为 `RWTexture2D`。
    *   C# 端必须分配 `RenderTexture`，且必须设置 `enableRandomWrite = true`。
    *   Shader 中不能用 UV 和 Sampler 采样 `RWTexture2D`，只能通过像素整数坐标索引（如 `Result[id.xy] = color;`）进行直接读写。
*   **HDR 数据保存：** 烘焙出的高精度浮点数LUT如果需要保存到本地，必须使用 `.exr` 格式（`EncodeToEXR`），禁止使用 PNG/JPG，否则会丢失大于 1 的数据并产生严重的压缩截断。
*   **LUT 非线性重映射映射：** 大气密度在靠近地平线附近变化极为剧烈，而在高空变化平缓。如果 UV 进行线性映射，会导致有效像素全集中在几个像素内。必须参考开源实现，对 UV 映射的半径和角度使用**非线性重映射**（通常是平方关系的变换），将更多像素分配给变化率大的地平线区间。

### 2. URP Render Feature 半成品陷阱 (基于 Unity 2022)
*   **RTHandle 系统的严格管制：** 在 2022 版本的 URP 中，渲染目标必须通过 `RTHandle` 管理。
    *   必须在 Render Pass 的 `OnCameraSetup` 阶段使用 `ConfigureTarget` 声明你要写入的目标。如果在 `Execute` 阶段临时分配并绑定，会导致管线内部状态机错乱，渲染崩溃。
    *   原则上一个 Pass 只能声明并写入一个受管制的渲染目标。
*   **Blit 自己到自己的报错：** 绝不能将读取的 `CameraColor` 处理后再直接 Blit 回 `CameraColor` (`source` 和 `destination` 不能是同一个 `RTHandle`)。
    *   **破解方案：** 必须使用 `RenderingUtils.ReAllocateIfNeeded` 开辟一个与相机颜色相同格式的**临时 RTHandle**。
    *   执行步骤：先将 CameraColor Blit 到 临时 RT（附带大气散射材质进行计算），再从临时 RT 原样 Blit 回 CameraColor。由于临时 RT 不受系统严格管制，可以在此进行“偷鸡”操作。
*   **URP 后处理的标准化获取：**
    *   获取屏幕深度：Include `DeclareDepthTexture.hlsl`，使用 `SampleSceneDepth(uv)`。
    *   重构世界坐标：使用 `ComputeWorldSpacePosition(uv, depth, UNITY_MATRIX_I_VP)` 还原绝对世界坐标。
    *   获取场景阴影：URP 仅支持一张全局方向光阴影，后处理中可通过引入 `Lighting.hlsl` 并使用 `TransformWorldToShadowCoord` 及 `MainLightRealtimeShadow` 直接采样阴影纹理。

### 3. 光线步进 (Ray Marching) 的边界情况处理
*   **防地下退化：** 步进计算时，可能由于数值精度导致高度 $h$ 小于 0，必须执行 `max(h, 0)` 将其拉回地表高度，防止计算消光系数时引发异常。
*   **各种相机位置与视线方向的碰撞逻辑：**
    *   **在大气层外未看向地球：** 射线与大气无交点，直接返回原场景颜色。
    *   **场景物体遮挡：** 比较原场景深度还原的物体距离与大气退出点距离，射线步进终点应取两者中的最小值。
    *   **大气层外被物体遮挡：** 如果从太空看向地球，但场景物体深度距离小于射线进入大气的距离（即物体挡在了大气层前面），应直接放弃大气计算，返回原色。
    *   **视线撞击地面：** 遇到与地表的正交点（$X>0$），射线的步进终点必须截断在地表交点处。
    *   **相机沉入地表以下：** 如果存在矿洞等需求，常规大气模型无法直接处理相机在地下的情况（理论上海平面以下无大气）。需要做特殊的分支处理（本节课暂时规避，通过把高度 Clamp 为 0 强行计算）。
*   **离散步进偏移：** 在对每一段参与介质进行步进采样时，采样点应取当前步长的中点（`current_length += step_size * (i + 0.5)`），以保证积分结果最接近理论真实值。
