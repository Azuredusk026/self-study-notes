# Houdini程序化建模基础与实践

## 一、 核心知识点与原理剖析

### 1. 三维建模基础与底层数据结构
- **DCC (Digital Content Creation) 软件基础**：无论是 Houdini 还是普通的 3DMax、Blender 等建模软件，学习其基本操作时，均遵循“视角控制 -> 新建物体 -> 元素选择 -> 基础形变”的路径。
- **三维模型底层结构**：
  - **点 (Points)**：记录空间坐标属性（P.x, P.y, P.z）的物理存在。它是决定模型位置与长相的最核心要素。
  - **面 (Primitives/图元)**：记录由哪些点依次连接而成的拓扑关系。
  - **边 (Edges)**：在底层的基本几何数据表（Geometry Spreadsheet）中**并不独立存在**，只是为了方便美术人员操作而提供的一种关联选择视图。移动一条边本质上是同时移动它所关联的两个端点；移动一个面本质上是移动该面所关联的所有顶点。
  - **顶点 (Vertices)**：与渲染强相关的顶点信息，用于定义面上每个点对应的独立属性（如 UV、顶点色等）。

### 2. Houdini 节点网络架构
- **双层网络设计**：
  - **Object (OBJ) 级网络**：对应游戏引擎（如 Unity）的 Hierarchy 层级窗口。用于管理场景中的独立对象、相机、灯光等。节点间的连线表示**父子继承关系**（移动父物体，子物体跟着移动；移动子物体，父物体不动）。
  - **SOP (Surface Operator) 级网络**：双击 OBJ 级节点进入的内层网络，专门用于几何体内部的具体建模。节点间的连线代表**几何数据流 (Data Flow)**，流动的是包含点、面、顶点等信息的几何属性表。
- **节点标记 (Flags)**：
  - **蓝色标记 (Display/Render Flag)**：代表该 SOP 网络向外输出的几何体。同一 SOP 网络内（未合并前）只能激活一个蓝色标记。
  - **紫色标记 (Template/Reference Flag)**：参考线框标志。在视口中以线框形式显示该节点代表的几何体，常用于建模时的对齐与对照。

### 3. 选择机制与组 (Group) 的程序化应用
- **程序化建模与手选的冲突**：在视口中手动选择面并进行删除或移动，会在 SOP网络中自动插入 `Edit` 节点。此类操作会将具体的顶点/面 ID 硬编码在节点内，无法通过外部参数动态控制，破坏了程序化流程。
- **Group 节点的核心逻辑**：通过规则将特定的点、线、面归纳进一个命名组中，后续编辑节点（如 Transform、PolyExtrude）可通过引用组名（如 `Group1`）定向操作，实现全流程的参数化。
- **四种程序化分组模式**：
  - **Base Group (基础组)**：直接输入点面 ID（以空格分隔，如 `2 4 0`）。支持通过视口交互箭头选中后按回车自动填入。
  - **Keep in Bounding Regions (包围盒选择)**：
    - 支持 Box (立方体包围盒) 与 Sphere (椭球包围盒) 形式。
    - 勾选 `Include partially contained entities` 后，只要元件与包围盒发生部分接触即可被选中。
    - 支持外接包围盒对象 (Bounding Object) 进行动态裁剪。
  - **Keep by Normals (按法线方向选择)**：根据面的法线方向向量（如 `0, 1, 0` 代表朝上）与设定夹角（Spread Angle）进行筛选。例如：90度夹角可选中球体的上半球；30度夹角可精准选取顶部区域。
  - **Keep by Edges (按边选择)**：通过设置边两侧面法线夹角的阈值（Min/Max Edge Angle）来筛选边。常用于自动提取模型中锐利的边缘，进而应用倒角。
  - **随机选择 (Random)**：按特定比例或概率随机筛选点、面。
- **组属性在底层的数据表现**：在几何数据表（Geometry Spreadsheet）中，Group 表现为 `group:组名` 格式的列属性。值为 `1` 代表该元素在组内，`0` 代表不在组内。

### 4. 常用几何操作节点与参数
- **Transform (变换)**：对选定对象或组进行平移（Translate）、旋转（Rotate）、缩放（Scale）。
- **Soft Transform (软变换)**：基于点集进行变换，并通过设置 `Soft Radius`（软半径）使其平滑地影响周围未被选中的点。半径为0时仅移动选中的点，半径越大影响范围越广。
- **PolyExtrude (多边形挤出)**：沿法线方向新建几何体。支持调整挤出距离（Distance）、缩放（Inset）、扭转（Twist）等参数。
- **PolyBevel (多边形倒角)**：对锐利边缘进行切角或圆角处理。支持调整距离（Distance）与细分段数（Divisions）。
- **PolyBridge (多边形桥接)**：在两组选定的面（Group1 与 Group2）之间建立过渡网格，会自动删去原有的桥接面。即使两端面数不一致也能强制桥接，但布线会发生扭曲。
- **Hinge Extrude (铰链挤出)**：以指定的边组（Group）为旋转轴，将选定的面以铰链形式呈扇形挤出。
- **Merge (合并)**：仅将多路几何数据流合并到同一个通道中，不进行几何拓扑上的缝合，内部重叠面依然保留。

### 5. 布尔运算与网格重构拓扑
- **Boolean (布尔)**：提供 Union (并集)、Intersect (交集)、Subtract (差集) 三种逻辑。
- **布尔带来的拓扑缺陷**：极易产生多边面（N-gons），若直接在其后连接 **Triangulate (三角化)** 节点，会生成大量极其细长、扁平的“针状”三角面。这类面在烘焙法线或应用 Shader 时会产生严重的光影瑕疵。
- **程序化优化三板斧**：
  - **Remesh (网格重构)**：以指定的 `Target Size` 重新划分均匀的三角网格。缺点是会使模型边缘变钝、塌陷。可通过大幅降低 Target Size（如 `0.05`）转换为超高模来保真，但面数会暴增。
  - **Smooth (平滑)**：松弛网格顶点，消除噪点，但会牺牲模型体积与高度细节。
  - **Sharpen (锐化)**：在曲率变化大（夹角锐利）的地方压缩、加密顶点以凸显硬边；在平坦区域扩张、稀疏顶点。能有效修复平滑带来的边界模糊问题。

### 6. 散点、复制与体积转换控制
- **Scatter (散点)**：在几何体表面均匀或随机撒落点云。
- **Copy to Points (复制到点)**：将几何体（如立方体）拷贝到目标点云上。
- **利用 Attribute (属性) 影响 Copy to Points 的行为**：
  - `pscale` (float)：控制复制物体的统一缩放比例。
  - `scale` (vector)：控制三维独立轴向缩放比例。
  - `orient` (vector4)：控制物体三维朝向。
  - `N` (vector) & `up` (vector)：法线方向与上方向，联合定义旋转朝向。
  - `rot` (vector4)：附加旋转。
  - `P` (vector)：点的位置。
- **属性噪音节点 (Attribute Noise)**：
  - **Attribute Noise Float**：在指定的浮点型属性（如 `pscale`）上叠加噪波。可通过调整 Frequency (频率) 和 Amplitude (幅度) 来改变缩放分布的随机性。
  - **Attribute Noise Vector**：在矢量属性（如 `scale`）上应用三维噪波。
- **ISO Offset (体积/SDF转换)**：将封闭的 Polygon 网格转换为体积数据（Volume），可在体积内部（而仅是表面）进行 Scatter 散点操作，常用于堆叠类物体（如乱石、填充物）的程序化生成。

### 7. 表达式与参数化暴露
- **Edit Parameter Interface (编辑参数接口)**：
  - 右键节点选择 `Parameters and Channels -> Edit Parameter Interface`。
  - 可以从 `From Nodes` 栏目中直接将 SOP 内部节点的深层参数（如弯曲节点的 Squash Pivot）拖拽到 OBJ 节点的根目录下，在最外层直接提供滑条控制，实现参数化。
- **核心内置表达式函数**：
  - **`ch(path)`**：获取指定通道/参数的值。常用相对路径，如 `ch("../random_seed")` 获取父级节点的随机种子。
  - **`rand(seed)`**：基于哈希算法的伪随机函数，输入固定种子返回 `[0, 1]` 之间的浮点数。
  - **`fit01(value, new_min, new_max)`**：将 `[0, 1]` 范围的值重映射到新区间 `[new_min, new_max]`。
- **独立随机数的偏移 (Offset) 技巧**：
  - 若直接使用 `rand(ch("../random_seed"))` 控制多个参数，会导致所有随机属性表现出完全一样的变化趋势。
  - 必须对种子施加不同的数学偏移或乘数（如 `rand(ch("../random_seed") * 13 + 533)`），使其哈希定位到不同的数据空间，从而生成互不干扰的独立随机值。

### 8. 循环结构 (For-Each Loop)
- **For Loop with Feedback**：
  - 产生一对节点（`block_begin` 与 `block_end`）。
  - **Feedback 模式**：每一次循环迭代以上一次循环的输出结果作为输入，层层累加。
  - **Merge 模式**：每一次循环的执行结果在最后进行合并输出。
- **元数据节点 (Create Meta Import Node)**：
  - 点击 `block_begin` 上的按钮生成，该节点存储有 `Detail` 级的全局参数。
  - 使用 `detail()` 函数获取当前循环的迭代次数：`detail("../block_begin1_metadata", "iteration", 0)`（返回值从0开始递增）。
- **避坑：Feedback 的级联缩放问题**：
  - 若在 Feedback 循环体内部直接应用 `Transform` 且缩放比例不为 1，经过多次循环后，缩放会呈指数级级联放大/缩小。
  - **正确做法**：将循环起点 `block_begin` 设为 `Fetch Input`，使其每次都基于初始输入开始运算；然后在 `Transform Axis` 节点中，使用表达式 `iteration * 步进值`（如 `iteration * 35`）来线性控制旋转，并结合 `iteration` 属性和外层随机种子计算每一步的独立缩放与平移，最后通过 `Merge` 模式将所有片瓣合并。

## 二、 重点术语与概念解析

- **DCC (Digital Content Creation)**：数字内容创作。泛指 3ds Max、Maya、Blender、Houdini 等用于创建三维数字内容的专业软件。
- **SOP (Surface Operator)**：表面操作节点。Houdini 中最核心的网络层级之一，专用于网格构建、拓扑修改和程序化几何体编辑。
- **Geometry Spreadsheet (几何数据表)**：Houdini 视口旁的数据分析表，实时以表格形式展现当前几何体的 Points、Vertices、Primitives、Detail 各层级的数据列与属性值。
- **Group (组)**：对模型元素（点/线/面）进行逻辑划分的标签体系，是连接选择逻辑与形变逻辑的桥梁。
- **Group Promote (组级别转换)**：将一个类型的组（如点组）转换成另一种类型（如面组）的节点，常用于在点编辑与面挤出操作之间进行数据传递。
- **Attribute (属性)**：附着在几何元件上的数据信息，可自定义命名和类型（如标量、矢量），并能被下游节点读取或被表达式调用。
- **pscale 与 scale**：`pscale` 为单通道浮点数，代表点的大小；`scale` 为三通道矢量，代表三个轴向的缩放。Houdini 的 `Copy to Points` 会优先读取这两项属性来确定生成物体的比例。
- **ISO Offset**：一种将 Polygon 几何体转成隐式体积（Implicit Volume / SDF 符号距离场）的节点。常用于定义散点的内部实体范围。
- **Detail Attribute**：全局属性。区别于点、面属性，Detail 属性仅有一行，属于当前节点整体，通常用来记录循环迭代次数（iteration）或全局统计量。
- **fit01**：线性重映射函数。将输入值（通常由 rand 函数产生）从 `[0, 1]` 线性映射到目标区间。
- **For-each Feedback**：反馈式循环。下一次迭代的输入依赖上一次迭代的输出。
- **For-each Merge**：合并式循环。每次迭代使用相同的初始输入，循环结束时将所有迭代产生的变体几何合并输出。

## 三、 工程经验与避坑指南

### 1. 软件环境与开发推荐清单
- **软件版本**：Houdini 的版本更新迭代较快，本课程要求使用 **Houdini 20.0 或 20.5 及以上版本**，请勿使用 19 及其以下版本，避免由于底层算法更改导致节点行为不一致。
- **软件推荐**：本学期及下学期将陆续使用以下软件，建议提前准备好对应的学习版或免费版：
  - **Unity**：主游戏引擎。
  - **Houdini**：程序化建模与特效制作。
  - **Substance Painter (SP)**：模型贴图绘制。
  - **Substance Designer (SD)**：程序化材质制作。
  - **Blender**：常规中模与低模制作（可直接在 Steam 下载免费版本，无需学习版）。
  - **3ds Max**：部分传统辅助建模任务（Blender 亦可平替）。

### 2. 三维建模的底层思考与操作避坑
- **避免硬编码的 Edit 节点**：在 Houdini 中应尽量减少直接在视口中拖动点面的操作，因为手动操作生成的 Edit 节点无法接收程序化参数。一切选择应尽量转化为 Group，一切变换应转化为 Transform 节点。
- **视口相机控制技巧**：在左侧工具栏切换至 **View** 工具时，可以直接通过鼠标左键旋转、滚轮缩放、中键按下平移，**无需按住 Alt 键**，这能大幅提升操作的顺畅度。
- **布尔运算在不同工作流中的定位**：
  - **传统建模流**：极力避免频繁使用布尔，因为布尔产生的破面、极细针状三角面会严重破坏拓扑，导致后期烘焙法线或写 Shader 时光影穿帮。
  - **程序化建模流**：布尔是快速组合基础形态的核心利器。Houdini 针对布尔进行了极高程度的算法优化。布尔产生的凌乱拓扑，可以通过后续的 **Remesh**（网格重构）配合 **Sharpen**（锐化）或者 **Smooth**（平滑）来自动修复。
- **Mountain (山脉/噪声) 变形自交错问题**：
  - 当对具有尖锐内角的几何体应用 Zero-centered (有正有负) 的噪声位移时，顶点会在凹角处相向运动，容易导致面与面相互穿插（自交错），破坏流形结构。
  - **解决方法**：位移幅度不宜过大；或者在噪声位移后，连接一个 **Boolean Union** 节点并设置为**自身并集 (Self Union)**，Houdini 会自动剔除穿插产生的内部面，保持外壳完整。
- **高度尺寸联动计算公式**：
  - 在参数化改变 Tube（圆管）的高度时，若要使圆管底部始终贴合在 Y=0.1 的水平面上，直接放大高度会导致圆管双向拉伸。
  - **数学公式锚定法**：将 Tube 节点的 Center.Y（中心点Y轴位置）设置为表达式：`0.1 + Height / 2`。这样高度 Height 发生任何变化时，圆管底部位置均能被锁死在 Y=0.1 处。
- **关于课程工程文件与作业的说明**：
  - 基础程序化建模课（如本节课的石头、花瓶、果子）**绝不提供节点工程文件**。程序化建模极度依赖肌肉记忆与逻辑思维，学生必须对照思路，自行在软件中拖拽节点连线进行复现。
  - 后期复杂地形课（Terrain）由于节点繁复、细节严苛，极易因参数配置不当导致软件崩盘，届时会统一分发标准项目工程作为参考。
