# 游戏后处理技术：描边与水彩效果原理及Unity实现

## 一、 核心知识点与原理剖析

### 1. 内置管线后处理基础配置
*   **脚本创建与继承配置**
    *   继承 `PostProcessEffectSettings` 类：控制后处理在 Volume 面板上暴露的参数（需添加 `[Serializable]` 和 `[PostProcess]` 属性）。
    *   `[PostProcess]` 属性参数设定：
        *   关联具体渲染器（Renderer）。
        *   设定注入点：`PostProcessEvent.AfterStack`（在所有后处理、抗锯齿之后执行）、`BeforeTransparent`（透明物体前执行）。描边理论上应在 `BeforeTransparent`，但课上演示为方便设为了 `AfterStack`。
        *   场景可见性（控制Scene视图是否渲染）：最后传入布尔值 `true` 方便调试。
    *   继承 `PostProcessEffectRenderer<T>` 类：实现具体的渲染逻辑。
*   **全屏渲染机制 (Blit)**
    *   利用 `PropertySheet`（类似于内置在后处理功能中的 `Material`）来设置属性、开启宏。
    *   调用 `context.command.Blit(context.source, context.destination, sheet, 0)`。
    *   **原理**：`Blit` 本质上是绘制一个覆盖全屏的网格（一个大三角形或两个小三角形），并将其 UV 对应到屏幕四角（0~1）。`context.source` 为上一层处理完毕的纹理，`context.destination` 为目标渲染纹理。

### 2. 描边检测原理与实现 (Outline)
主要利用法线和深度来实现。法线用于提取**折痕边缘 (Crease)**，深度用于提取**轮廓边缘 (Silhouette)**。

#### 2.1 法线描边 (Normal Outline)
*   **获取法线纹理**：
    *   在 Render 脚本中重写 `GetCameraFlags()`，返回 `DepthNormals` 即可获得 `_CameraDepthNormalsTexture`。
    *   在 Shader (ASE) 中，使用 `TextureObject` 节点（设置为 Global 模式）获取该纹理。注意必须通过 `DecodeDepthNormal` 宏对其进行解码。
*   **边缘计算机制**：
    *   **计算夹角**：对当前像素与相邻像素（右、上）的法线进行点乘 (`Dot Product`)，得到它们夹角的余弦值 (Cosine)。余弦值越小，夹角越大，说明该处存在折痕。
    *   **获取相邻像素**：通过 `TexelSize` (`1/Width`, `1/Height`) 获取单一像素在 UV 空间的偏移量，与当前 UV 相加得到右侧和上方像素的 UV。
    *   **合并 X/Y 轴描边**：分别在水平和垂直方向求点积，然后取较小值 (Min)，代表该点在任意方向存在较大折痕即可视为边缘。
    *   **过滤非边缘区域**：使用 Remap 将点积结果 (-1 到 1) 映射为 0 到 1。引入阈值 `Min Cosine` (如 -0.8)，将余弦值大于该阈值的平滑表面（如球体弧面）过滤掉，避免曲面产生不必要的变灰/伪描边。

#### 2.2 深度描边 (Depth Outline)
深度纹理的 16 位精度在计算轮廓时不够用，因此需要开启获取 24 位深度的 `_CameraDepthTexture`（在 ASE 中通过 `ScreenDepth` 节点）。
*   **线性深度的必要性**：
    *   默认采样的深度是非线性的 (Non-Linear)，非线性深度的差值会随着相机距离产生剧烈变化（越近越亮，越远越暗），这会导致边缘判定在距离改变时不一致。
    *   必须使用线性深度 (`LinearEye`)，即片元到相机的实际世界空间距离。
*   **一阶差分与斜面问题**：
    *   取当前像素线性深度与右侧像素线性深度的差的绝对值。
    *   **问题**：即使是平滑的倾斜地面，由于透视关系，相邻像素的深度差也会持续存在且越远越大，产生大面积的错误描边。
*   **二阶差分过滤平面**：
    *   取“当前与右一”的差值，再取“右一与右二”的差值。
    *   若两个差值相等（差值的差为极小值），说明深度是线性递增的，这代表它是一个平面，直接剔除（使用 `Step` 函数过滤）。
*   **生成最终轮廓**：
    *   使用非线性深度来生成边缘线（非线性深度判定更稳定），乘以二阶导算出的平面过滤遮罩。
    *   通过阈值减法（如减去 `0.2` 的最小深度差 `MinDepth`），滤除深度差极小的不明显轮廓。分别处理 X 和 Y 轴后取最大值合并。
*   **法线与深度描边合并**：将法线描边乘以一个强度系数（如 0.5），叠加到深度描边上。法线描边较淡，深度描边较深。

### 3. 滤波 (Filtering) 与卷积 (Convolution) 原理
滤波通过与核函数 (Kernel) 进行卷积来实现图像的模糊、锐化或边缘检测。
*   **卷积操作**：将卷积核（如 3x3、5x5 的权重矩阵）置于图像某像素上，将对应区域的像素值与权重相乘并求和，作为新图像对应位置的像素值。
*   **高斯滤波 (Gaussian Blur)**：
    *   最常用的平滑滤波器，其核权重由高斯函数计算得出，越靠近中心权重越大。
    *   高斯核的大小 (5, 7, 9) 通过 $\sigma$ (Sigma) 计算。通常超过一定范围的权重极小，可忽略不计。
    *   **二维高斯核的可分离性 (Separable)**：一个二维高斯核等效于一个一维水平核和一维垂直核的乘积。
    *   **性能优化**：不在 Shader 中动态计算高斯权重（涉及昂贵的指数 `exp` 运算），而是将预计算的权重直接写成固定数组。使用一维卷积替代二维卷积：原本 $9\times9=81$ 次采样，可优化为水平 $9$ 次 + 垂直 $9$ 次 $=18$ 次采样，大幅降低开销。

### 4. 水彩效果实现 (Watercolor Effect)
水彩效果核心特征：边缘水渍晕染（高斯差分）、色彩特性（低对比高饱和）、纸张肌理、手工边缘扰动。

#### 4.1 高斯差分 (Difference of Gaussians, DoG)
用于模拟水彩颜色向外晕染、扩散的视觉效果。
*   **原理**：原本是色彩边缘检测算法，通过两个不同大小的高斯滤波器（如 9x9 和 5x5）处理图像，将结果相减。如果减去的量较小，可以模拟水彩扩张的效果。
*   **实现步骤**：
    *   需要申请中间渲染纹理 (RenderTexture)，依次经过：原图 -> 水平 9x1 模糊 -> 垂直 1x9 模糊 (得到结果A)；原图 -> 水平 5x1 模糊 -> 垂直 1x5 模糊 (得到结果B)。
    *   水彩是在白纸上叠加变暗的过程，因此需要先取反相 (`1 - color`)，再用大核 (9) 的结果减去小核 (5) 的结果。
    *   引入倍增补偿系数，调控水彩晕染的范围和强度，最后再反相回来。

#### 4.2 色彩调校 (Color Grading)
*   水彩特点：对比度低、饱和度高。
*   通过 Unity 的 Color Grading / 后曝光调整。注意：Unity 中对比度拉到 0 会直接变全灰，需要手动拉高曝光度（明度）来补偿丢失的亮度。
*   添加低精度的环境光遮蔽 (AO)，其阴影聚集的质感能大幅增强水彩感。

#### 4.3 边缘扰动与材质融合
*   **UV扰动 (UV Distortion)**：手绘线条并不笔直。采样两个噪声图映射到 `-1 到 1` 之间，乘以极小的扰动系数（如 `0.001`）附加到 UV 上，使线条产生轻微扭曲。
*   **局部褪色 (Dispersion/Fading)**：模拟水彩的不均匀干涸。
    *   使用正方形噪波纹理。将屏幕 UV 乘以屏幕像素宽高，转化为像素级坐标，再缩放后进行采样，确保无论屏幕比例如何，噪点图始终保持正方形不拉伸。
    *   通过此纹理控制局部区域的饱和度降低。
*   **纸张混合 (Paper Merge)**：引入宣纸/水彩纸噪波纹理。使用特定公式进行融合：`Origin^2 - Origin + (Paper - 1) * (Origin^2 - Origin) ...` 等一系列非线性叠加，赋予画面纸张颗粒感。

## 二、 重点术语与概念解析

*   **PP / Post Process Volume**：Unity 后处理体积组件，用于管理和混合场景中各种后处理效果的参数。
*   **CommandBuffer.Blit**：后处理中最核心的指令，将源纹理渲染到目标纹理上，通常用于应用一个 Shader pass 进行全屏图像处理。
*   **PropertySheet**：在后处理管线中用来管理和传递材质参数的封装类。相当于一个绑定了后处理 Shader 的临时 Material。
*   **DecodeDepthNormal**：Unity 内置宏。由于 `_CameraDepthNormalsTexture` 是将 16位深度与 16位法线打包在一个 32位纹理中，必须通过此宏解包提取真实的法线向量。
*   **TexelSize**：单一纹素的大小。值为 `(1/Width, 1/Height, Width, Height)`。在 Shader 中用于精确偏移恰好一个像素的 UV 坐标。
*   **LinearEye / 非线性深度**：非线性深度随距离倒数变化，近处精度高远处精度低；LinearEye 是转换为真实的线性世界空间距离，用于做等距数学推导。
*   **Temporal Anti-Aliasing (TAA)**：时域抗锯齿。利用多帧历史信息平滑锯齿。
*   **Custom Expression (ASE节点)**：Amplify Shader Editor 中的自定义代码节点，允许外部直接引入 `.hlsl` 文件代码以执行复杂的数学、循环或数组操作。

## 三、 工程经验与避坑指南

### 1. 深度与法线的精度问题
*   **深度的严重坑点**：`_CameraDepthNormalsTexture` 提供的深度由于只有 16 位，精度极低。不要用它来做边缘差值运算，会产生严重的精度断层。必须开启并使用全精度的 `ScreenDepth` (`_CameraDepthTexture`，24位或更高)。
*   **相机 Target Texture 兼容**：在申请独立的 RenderTarget 时获取屏幕大小，要区分相机是直接渲染到屏幕 (`Display`) 还是渲染到了某张贴图上 (`Target Texture`，例如做传送门画中画效果)。需做判断：如果 `camera.targetTexture` 不为空则用其长宽，否则使用 `camera.targetDisplay` 或 `Screen.width/height`。

### 2. 抗锯齿 (AA) 对后处理的干扰
*   如果描边严重呈现锯齿现象，可以通过开启抗锯齿解决。但 **坚决不要使用 TAA (时域抗锯齿)** 处理需要深度差分的描边，TAA 的帧间混合会导致深度差分边缘疯狂抖动 (Jitter)。
*   **解决方案**：推荐将相机的抗锯齿选项改为 SMAA 或 FXAA，尤其是 SMAA 对后处理描边边缘的平滑效果更好且不产生抖动。

### 3. 多 Pass 的渲染资源释放
*   在使用类似于高斯滤波、水彩这种需要大量中间 RenderTarget (RT) 来倒腾纹理的后处理时，**切记在 `Release()` 函数和 `ResetHistory()` 中释放所有自行 `GetTemporaryRT` 创建的纹理**，否则会导致极为严重的显存泄漏与画面渲染错误。

### 4. 内置管线与 URP 管线后处理移植困境
*   **URP RenderFeature 坑点**：在 Unity 2022 / 2023 版本的 URP 中，`RenderFeature` 处于极其不稳定的中间状态（混合了旧 API 和新的 Render Graph System）。很多 API 会抛出无征兆的异常。
*   URP 的彻底稳定要等到 URP 17 (Unity 6.0+) 的 Render Graph 完全普及。国内开发如果被锁死在 2022 版本，手写 URP 的后处理必定会遇到各种莫名其妙的红字报错。遇到问题只能严格照抄 Package Manager 里的官方 URP Example 源码中 Blit 的实现。

### 5. 颜色与调色的底层逻辑
*   水彩由于水浸湿纸张，实际绘画时带有边缘发黑的自然现象。利用本课提取的**黑色描边**直接盖在水彩效果上，刚好完美模拟了水渍变黑的真实物理表现。
*   Unity `Color Grading` 对比度降为 0 的暴力处理逻辑会毁掉整体亮度。写图形学 Shader 时，不能轻信常规图形软件（如 PS）的视觉逻辑，需针对引擎特性自行做后曝光亮度补偿。
