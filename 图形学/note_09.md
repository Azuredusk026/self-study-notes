# Unity Shader进阶：渲染排序、表面着色器与噪声特效解析

## 一、 核心知识点与原理剖析

### 1. 渲染列表与绘制顺序 (Render List & Sorting)
Unity 在渲染每一帧时，并不仅是简单地遍历 Shader 中的 Pass 就能直接出图，而是通过创建**渲染列表 (Render List)** 对所有相关的 DrawCall 进行收集、排序和合批。底层流程如下：
*   **收集与视锥体剔除 (Frustum Culling)**：首先收集场景内所有带 `Renderer` 组件（如 `MeshRenderer`, `SpriteRenderer`, 带骨骼蒙皮的 `SkinnedMeshRenderer`）的物体，剔除在相机视锥体之外、不会被看到的渲染器。
*   **Pass Tag 筛选**：通过解析 SubShader 或 Pass 中定义的 `LightMode` 等 Tag 来筛选当前需要的 Pass。例如，当前处于前向渲染主光阶段，引擎就会专门筛选带有 `ForwardBase` 标签的 Pass。
*   **排序规则 (Sorting Criteria)**：为了优化性能和保证透明效果，引擎会严格按照以下规则进行排序：
    1.  **Sorting Layer**：主要针对 2D 游戏中的精灵 (Sprite)，按照图层顺序排列。
    2.  **Render Queue (渲染队列)**：在 `Tags` 中指定的队列值（`Queue`），值域为 1 到 5000。
        *   **不透明物体组 (Opaque, 0 ~ 2500)**：为了极致的性能优化，利用 **Early-Z (早期深度测试)**，渲染顺序为**从前往后**（离相机近的先画）。这样被近处物体遮挡的远处片元在光栅化阶段会被直接丢弃，不再执行昂贵的 PBR 光照代码。
        *   **透明物体组 (Transparent, 2500 ~ 5000)**：渲染顺序必须**从后往前**（离相机远的先画），以确保透明像素的颜色能够以正确的先后顺序混合在背景上。
    3.  **合并与 SetPassCall**：排序完成后，将相同材质和相同 Pass 的渲染器合并在一起合批 (Batching)，最后统一调用 `SetPassCall` 准备显卡状态并下发绘制指令。

### 2. 表面着色器 (Surface Shader) 工作流
表面着色器是 Unity 内置管线（Built-in）独有的高度封装形式。它将复杂的顶点变换和光照模型计算隐藏在了底层，让开发者只需关注物体“表面”的物理属性。
*   **输入结构体 (`Input`)**：
    *   通过 `uv_MainTex` 等变量获取纹理的 UV 坐标。
    *   自带内置参数提供计算支持：`viewDir`（世界空间视线方向）、`worldPos`（世界空间顶点位置）、`worldNormal`（世界空间法线）。
    *   `screenPos`：裁剪空间坐标，需经过齐次除法（除以 W 分量）后转为 NDC 坐标，方可用于屏幕空间纹理采样。
*   **输出结构体 (`SurfaceOutputStandard`)**：必须向此结构体填充计算好的表面数据：反射率 (`Albedo`)、法线 (`Normal`)、金属度 (`Metallic`)、光滑度 (`Smoothness`)、自发光 (`Emission`) 和透明度 (`Alpha`)。
*   **编译指令声明 (Pragma)**：
    *   `addshadow`：让材质能够投射和接收阴影。
    *   `vertex:vert`：指派自定义的顶点着色器函数，用来干预并移动顶点位置（如法线外扩膨胀、随时间波动的顶点动画）。
    *   `alpha:fade`：开启标准的 Alpha 混合透明度模式。
*   **局限性**：由于底层被接管，无法直接使用常量缓冲区 (CBUFFER) 优化，且无法轻易干预光源计算，因此极难用于非真实感渲染 (NPR) 开发。

### 3. 纹理采样与法线映射 (Texture Sampling & Normal Mapping)
*   **UV 采样与空间操作**：通过 `tex2D(tex, uv)` 进行采样。UV 坐标空间中，左下角为纯黑 (0,0)，纯红代表 X 轴极值，纯绿代表 Y 轴极值。通过对 UV 坐标进行**加减法（移动贴图）**和**乘除法（缩放贴图，如 UV 除以 2 则贴图放大一倍）**，可以极度灵活地改变贴图映射状态。
*   **多材质平滑过渡**：当基于蒙版图混合两种不同材质（如生锈金属与光滑金属）时，严禁使用硬生生的 `if` 判断（会造成严重的锯齿和突变），必须使用 **`lerp(a, b, t)` 线性插值** 以保证灰度区域平滑过渡。
*   **切线空间与法线映射**：
    *   **切线空间 (Tangent Space)**：基于模型表面构建的局部空间，由**模型法线 (Normal, Z轴)**、**切线 (Tangent, X轴)** 和通过叉乘算出的**副切线/副法线 (Bitangent/Binormal, Y轴)** 构成。
    *   **提取法线**：法线纹理在 RGB 中存储了切线空间法线方向，需使用内置方法 `UnpackNormal()` 进行提取映射（将 0~1 的颜色值转换为 -1~1 的方向向量）。
    *   **法线强度控制**：通过 `UnpackNormalWithScale(tex, scale)` 可以控制凹凸强度。其数学本质是让切线空间下的法线向远离 Z 轴的方向“倾斜”与“偏转”。
    *   **法线插值的坑**：普通的 `lerp` 线性插值不能均匀处理方向向量的角度变化，最精确的方向插值应使用 **球面线性插值 (`slerp`)**。但在日常非极端情况下，直接 `lerp` 在视觉上也能接受。

### 4. GPU 随机数与噪声生成算法 (Noise)
*   **哈希算法 (Hash) 替代 CPU 随机数**：CPU 的 `Random` 基于单线程内部迭代器，完全无法适应 GPU 几千个运算单元的并行架构。在 GPU 中生成随机数，本质是通过设计精妙的**哈希映射**（如本课采用的 **Squirrel3 哈希算法**），基于当前传入的空间坐标 `(x, y, z)` 和特殊大质数，返回一个伪随机的 `uint` 整数，最终通过二进制移位和 IEEE 浮点数标准强转，映射为 `0 ~ 1` 的随机浮点数。
*   **白噪声 (White Noise)**：直接将坐标向下取整后送入哈希生成。相邻像素值各自独立，视觉上体现为明显的正方形“马赛克”格子感。
*   **线性噪声 (Value/Bilinear Noise)**：提取网格四个顶点的白噪声值，在格内利用坐标小数部分进行**双线性插值 (Bilinear Interpolation)**（先在上下 X 轴各插值一次，再将两次结果在 Y 轴插值）。这种方法虽然平滑了内部，但由于在网格边缘处的**偏导数不连续**，人眼仍能敏锐察觉到横平竖直的网格线。
*   **柏林噪声/单形噪声 (Perlin/Simplex Noise)**：不基于正方形网格，而是基于单形（三角形/四面体等）构建的连续噪声。具有完美的各向同性，极度适合生成火焰边缘、云朵、水波纹等自然现象。
*   **三维噪声的应用价值**：三维纹理（3D Texture）会带来灾难级的内存与显存开销，极少使用；但在 Shader 中通过传入 3D 世界坐标直接计算出的三维噪声，由于是按片元实时计算，开销极低，被广泛应用于体积云等三维体积效果中。

### 5. 燃尽特效 (Burn-out Effect) 节点实现逻辑
基于 ASE / Shader Graph 的消融效果底层实现闭环：
1.  **3D 噪声采样**：向 `3D Simplex Noise` 节点传入世界空间坐标，获取物体表面的连续噪声值 (0~1)。
2.  **设置阈值与相减**：用噪声值减去外部传入的消融控制条 `Burn Edge`（从 0 滑动到 1）。
3.  **剔除片元 (Alpha Clipping)**：将运算结果接入 Alpha 测试节点，开启 `Alpha Clip` 模式（对应 HLSL 代码的 `clip()` 函数），阈值设为极小值（如 `0.001`）。所有结果小于 0 的片元会被管线**直接丢弃 (Discard)**，形成消融镂空。此举避免了常规透明混合带来的深度排序错乱问题，仍然能够写入深度。
4.  **边缘捕捉 (Remap重映射)**：利用 `Remap` 节点，专门剥离出刚刚被消融掉的边缘邻近值（如 `0 ~ 0.05` 区间），并将其翻转映射为 `1 ~ 0`，以此精准捕捉最亮边缘。
5.  **HDR 发光 (Bloom泛光)**：将提取出的边缘蒙版乘以一个极大的 **HDR 超亮颜色**，最后赋值给 `Emission` (自发光)。配合相机的屏幕后处理组件 **Bloom**，实现边缘高温泛光的视觉震撼效果。

## 二、 重点术语与概念解析

*   **Render Queue 常用预设值**：
    *   `Background` (1000)：最先渲染的背景（如天空盒）。
    *   `Geometry` (2000)：默认的不透明几何体。
    *   `AlphaTest` (2500)：透贴/镂空材质（如树叶、铁丝网），利用 Alpha 剔除。
    *   `Transparent` (3000)：半透明材质（如玻璃、水）。
    *   `Overlay` (4000)：全屏贴花、UI界面等最终覆盖层。
    *   *(注：可以通过加减法精调，如 `Geometry+1` 用来确保某物体比普通几何体晚渲染。)*
*   **Pass Tag (LightMode)**：
    *   `ForwardBase`：主光源和环境光渲染阶段。
    *   `ForwardAdd`：额外的点光源和聚光灯叠加阶段。
    *   `Deferred`：将基础色、法线、光滑度等数据输出到 G-Buffer 的延迟渲染阶段。
    *   `ShadowCaster`：专门用于向深度图投射并生成阴影的 Pass。
    *   `Meta`：非实时渲染，用于编辑器下向光照贴图烘焙器提供发射光等材质数据。
*   **HDR (High Dynamic Range)**：高动态范围颜色。传统显示器 LDR 下颜色最高只能到纯白 (1, 1, 1)，HDR 允许使用超过 1 的乘数（如 `Color * 2^4`）。但超过 1 的值在屏幕上单纯显示依然是白色，**必须开启后处理的 Bloom 效果** 才能让高亮像素溢出扩散，产生光晕。
*   **Sizzle/Swizzle**：通道重组技术。在获取变量时，将提取的通道进行重构（例如 `vec.xz` 或将通道反转 `1-r`）。
*   **Node-Based Shader Editor (节点式编辑器)**：如 Unity 自带的 **Shader Graph** 或广受好评的商业插件 **Amplify Shader Editor (ASE)**，通过将代码逻辑（如相加、插值、UV计算）具象化为节点连线，最终自动编译出相应的 HLSL 着色器代码。

## 三、 工程经验与避坑指南

### 1. 开发工具的避坑与替代方案
*   Unity 最新版 (URP/HDRP) 已经彻底废弃了内置管线的 Surface Shader 手写工作流，全面拥抱连连看模式。
*   **Shader Graph 严重 Bug 警告**：官方的 Shader Graph 在向下兼容 Built-in 管线或处理双管线时，经常会遇到打不开图表、节点全黑等离谱 Bug。实战工程中若被其卡住，强烈建议使用 **Amplify Shader Editor (ASE)** 插件作为完美替代平替，后者兼容性好且高度稳定。

### 2. 纹理资产的导入设置红线
*   **法线贴图设置**：导入法线贴图后，必须将其 `Texture Type` 设置为 `Normal map`。否则引擎不仅无法做平台压缩优化，还无法正确将其带入切线空间。
*   **颜色空间与 sRGB 勾选**：
    *   **基础颜色 (Albedo/Color)** 纹理必须勾选 `sRGB`。
    *   **金属度 (Metallic)、粗糙度 (Roughness)、光滑度 (Smoothness)、法线 (Normal)** 等充当“物理数据”的纹理，**绝对不能勾选 `sRGB`**。如果误选，数据会被施加伽马校正曲线，导致计算出的光照和质感彻底崩坏。
*   **白嫖通道操作**：从外部（如 Substance Designer）获取的 PBR 粗糙度贴图，如果想要转为 Unity 的光滑度，无需在外部软件里翻转再导出一遍。可以直接在导入设置的高级选项（Advanced）中找到 `Swizzle` 属性，将颜色通道设置为 `1 - R` 等反向操作，一步到位。

### 3. 透明度混合与深度排序的血泪史
*   新手制作隐身或燃烧效果时，容易直接把材质改为了透明混合 (`Transparent`)。这将导致物体失去深度写入 (`ZWrite Off`) 能力，在场景中如果前后有两块该特效叠加，极易出现**深度穿帮**（远处的模型反而画在了近处上）。
*   **标准解法**：保持材质为不透明组 (`Opaque` / `ZWrite On`)，利用着色器代码中的 `clip()` 函数（或节点器中的 Alpha Clip 阈值判定），直接在 GPU 渲染管线内部把废弃的片元丢弃掉。这样留下的像素仍然具备深度测试能力，画面严丝合缝。

### 4. 找工作与职业发展方向指南 (针对技术美术/TA)
*   **特效向路线**：必须建立深厚的效果经验墙。需要大量跟练并手搓不同类型的 Shader 表现（比如火焰、溶解、水面透射、体积云、粒子特效混合等），熟练掌握各路数学节点连线。
*   **场景向路线**：重点攻坚大规模世界的构建技术。作品集应集中在地形渲染、植被实例化分布、场景光影性能优化与管线剔除等大作业上。
*   **角色向路线（高危慎入）**：如果打算去二游公司做角色 TA，需要面对极度孤立且成体系的繁杂技术栈（NPR 卡通渲染、多 Pass 描边阴影干预、复杂的脸部法线编辑、绑骨与动作工作流）。除非有明确项目带路，否则新手自学很容易踩坑且收效极慢，切勿盲目涉足。
