# Unity光照系统与渲染管线核心技术剖析

## 一、 核心知识点与原理剖析

### 1. 环境光照的构成基础
在Unity中，场景物体的光照表现由**直接光照**和**环境光照**叠加构成。同一材质在加入环境光照后，其视觉表现会有质的飞跃（如金属、硅材质半导体、光滑/粗糙绝缘体的表现差异极大）。
环境光照在图形学实现上分为两大类：
- **粗糙的环境光照（漫反射环境光 / Diffuse）**：通过**球谐基函数 (Spherical Harmonics, SH)** 实现拟合。
  - Unity Built-in管线中，使用前三阶（共9个系数）的球谐函数来记录与压缩空间中不同方向的环境光亮度分布。
  - 编写自定义Shader时，Built-in管线可通过内置函数 `ShadeSH9(normal)` 对该环境光进行采样获取。
- **光滑的环境光照（高光环境光 / Specular）**：通过**基于图像的照明 (IBL, Image Based Lighting)** 实现。该模块是基于物理渲染 (PBR) 中最复杂的部分，利用预过滤环境贴图等技术提供精准的反射效果。

### 2. 天空盒 (Skybox) 与环境反射贴图
环境光的初始来源是场景的天空盒，Unity支持以下几种类型：
- **6面体天空盒 (6-Sided)**：由上下左右前后6张贴图拼接而成。
- **Cubemap**：将6张图打包为一个素材格式。
- **经纬度图 (Equirectangular)**：常见的 `.HDR` 格式高动态范围全景图，支持记录大于 `1.0` 的极高亮度值。
- **程序化天空 (Procedural / 大气散射)**：提供一个 `Sun Source`（方向光），通过模拟大气的厚度和色散规律，程序化演算出天空颜色。
- **配置路径**：通过 `Window -> Rendering -> Lighting -> Environment` 面板配置天空盒材质，必须点击 `Generate Lighting` 进行烘焙，才能正确生成 IBL 贴图和 SH 球谐光照数据。

### 3. 全局间接光照与反射烘焙
间接光照分为静态与动态，Unity中主要提供以下技术方案：
- **静态漫反射全局光照 (Baked GI)**：
  - 需要在Inspector勾选物体右上角的 `Static`（特别是 `Contribute GI` 选项）。
  - 在 Lighting 面板开启烘焙，可选择 GPU 或 CPU 计算。
  - 光源模式分三种：
    - `Baked`：仅参与静态烘焙，完全不提供实时光。
    - `Realtime`：完全不参与烘焙。
    - `Mixed`：结合前两者（如 Shadowmask 模式，烘焙环境光与静态阴影，但保留高光等效果）。
  - **重要限制**：Unity 无法烘焙静态高光间接光照（因球谐函数只适合漫反射），仅游戏《教团1886》曾使用 Spherical Gaussian 技术实现过高光烘焙。
  
- **反射探针 (Reflection Probes - 弥补高光反射)**：
  - 动态和静态物体共用的高光反射方案。探针在其位置拍摄360度全景图，作为其包裹范围内物体的“局部天空盒”。
  - **Blend Distance**：实现进入探针范围时的反射平滑过渡，避免高光突变。
  - **Box Projection (盒状透射)**：将探针采样的全景图向房间包围盒投影，修复室内墙壁反射视差（即修正反射不贴墙的问题）。

- **光照探针 (Light Probes - 动态物体的漫反射)**：
  - 在空间布置一组点阵探针，预先烘焙好每个点处的漫反射球谐数据。
  - 动态物体（或未开启Static的物体）移动时，会被其所在的由光照探针构成的**四面体**包围，通过**重心坐标**插值，实时计算出平滑的漫反射光照。
  - **限制**：光照探针只记录“静态光源和静态物体”产生的间接光，接收光的物体可以动，但发光环境不能动。

### 4. 后处理技术 (Post Processing)
Built-in 管线需通过 Package Manager 安装 `Post Processing` 插件，在场景添加 `Post Process Volume` 和相机上的 `Post Process Layer` 运行：
- **Bloom (泛光)**：设定阈值(Threshold)与强度(Intensity)，使大于阈值的高亮区域产生光晕。
- **Depth of Field (景深)**：调节焦距、光圈等产生背景模糊。与Bloom底层常共用高性能的 **Dual Kawase Blur** 模糊算法。
- **SSAO (屏幕空间环境光遮蔽)**：通过屏幕像素深度计算墙角等区域的遮挡程度，压暗角落亮度，使模型避免“悬浮感”。
- **SSR (屏幕空间反射)**：仅在延迟渲染管线支持。在屏幕空间进行光线步进(Raymarching)匹配反射物，适合粗糙度极低（光滑）的地面或水面。
- **Auto Exposure (自动曝光)**：模拟人眼从亮室外进入暗室内时的瞳孔适应过程。
- **Color Grading & Tone Mapping (调色与色调映射)**：
  - **Tone Mapping 核心目的**：将高动态范围（HDR，超过1.0的亮度）压缩回显示器范围（LDR，0-1）。如果不做映射，超出 1.0 的颜色通道被强行截断会导致严重的色相偏移（如橙色因红色通道截断而发黄）。
  - **ACES**：电影级工业标准曲线，增强对比度并保持过曝时的色相准确。
  - **Neutral**：低对比度曲线，尽量不改变原画面直观感受。
  - **Lift/Gamma/Gain**：分别用于调整画面的暗部、中间调、亮部（如经典的暗部偏冷、亮部偏暖调色法）。
- **其他视觉特效**：Vignette(边缘暗角，引导视线)、Chromatic Aberration(色差/红移，体现速度感)、Grain(胶片颗粒，增加画面质感细节)。

### 5. 渲染路径 (Rendering Paths)
- **前向渲染 (Forward Rendering)**：
  - Built-in中按照 `Forward Base` 和 `Forward Add` 以Pass叠加的方式逐个光源绘制。
  - **限制**：单物体受光数量有限，多光源靠近时会互相挤占导致部分光源不生效；单物体同一时间只能受到整个反射探针的统一影响。
- **延迟渲染 (Deferred Rendering)**：
  - **核心机制**：引入**多渲染目标 (MRT, Multiple Render Targets)**，将场景的材质属性（反照率 Albedo、法线 Normal、粗糙度、AO等）先统一绘制到一组 **G-Buffer (Geometry Buffer)** 中，最后进行一次全屏的光照与反射计算。
  - **优势**：场景内光源与反射探针数量解禁（无上限），性能不再受复杂光照制约。物体同一表面可以分别受不同探针渐变影响。
  - **劣势**：无法处理半透明物体渲染（需退回前向管线处理）；G-Buffer 无法开启硬件 MSAA（多重采样抗锯齿），只能使用 TAA（时间抗锯齿）。

### 6. URP (通用渲染管线) 架构特性
- **设计差异**：Built-in 采用多 Pass 绘制光照；而 URP 采用单 Pass 前向渲染，一次性将所有光源数据传入，在 Shader 内部通过循环计算光照。
- **SRP Batcher (高度优化的合批技术)**：
  - 原理：将材质参数缓存到 GPU，允许不同材质但同 Shader 变体的物体实现 Draw Call 极速合批。
  - **生效硬性要求**：
    1. 不能使用 `MaterialPropertyBlocks`（2D项目常用，3D须规避）。
    2. Shader 面板中所有的 `Properties`，必须在代码中声明于名为 `UnityPerMaterial` 的常量缓冲区 (`CBUFFER_START`) 内；内置矩阵变量需处于 `UnityPerDraw` 中。（如果裸露在外，Unity会将其放进 `Globals` 缓冲区，破坏合批）。

### 7. LOD (Level of Detail) 细节层次
通过 `LOD Group` 组件实现。根据物体包围盒占摄像机屏幕的百分比阈值，动态切换物体的网格精细度（近处LOD 0高模，远处LOD 1/2低模，极远处剔除不渲染），以高度技巧性的方式节约性能。

### 8. 阴影系统原理与精度瑕疵
- **阴影贴图 (Shadow Map)**：从光源处放置“相机”拍一张深度图，渲染时将物体投影到光源空间，对比深度值判断是否处于阴影中。
- **Shadow Acne (阴影失真 / 条纹锯齿)**：由于深度图是像素化的矩阵采样，而实际表面是平滑的，导致表面自我遮挡产生一棱一棱的条纹。
- **解决方案与代价**：
  - **Bias / Normal Bias**：向光源方向（或向法线内部）推移阴影采样深度。
  - **Peter Panning (漏光/彼得潘悬空)**：若 Bias 推移过大，会导致接触面阴影断裂，物体看起来像悬浮在空中（通常需搭配 Contact Shadow / 屏幕空间阴影弥补）。
- **Cascade Shadows (级联阴影)**：将视锥体距离分段（如4级），使用4个不同范围的相机绘制深度图。近处分配极高的阴影分辨率，远处分辨率逐渐降低，极大改善阴影锯齿。

## 二、 重点术语与概念解析
1. **IBL (Image Based Lighting)**：基于图像的照明，提取环境全景贴图中的光照信息，经过预滤波处理后用于计算 PBR 材质高光的反射算法。
2. **SH (Spherical Harmonics)**：球谐函数，用一组数学公式高效压缩和还原球面上的低频信息，Unity中专门用于极低开销的漫反射环境光和光照探针计算。
3. **G-Buffer (Geometry Buffer)**：延迟渲染管线的命脉，包含多张屏幕分辨率大小的纹理组合（深度、法线、基础色、PBR参数），将几何信息与光照计算彻底解耦。
4. **Tone Mapping**：色调映射算法，HDR转LDR的核心步骤，用以维持画面在极端光照下的色彩对比度与色相稳定。
5. **CBuffer (Constant Buffer)**：常量缓冲区，HLSL 中用于批量、高速传输变量的内存块。URP中 `UnityPerMaterial` 缓冲区的正确排布是激活 SRP Batcher 的前置条件。

## 三、 工程经验与避坑指南

### 1. 材质与场景搭建的实战经验
- **避免大面积使用高光金属**：引擎烘焙静态场景时无法处理高光，若场景中存在大量纯金属或极光滑表面，光靠反射探针无法提供极度精确的间接光，极易“露馅”。行业标准做法是将其做成粗糙表面或增加污渍、生锈蒙尘效果以掩盖瑕疵。
- **房屋建筑的防穿帮设计**：
  - **斜向建筑需谨慎**：Unity自带的反射探针**不支持旋转**。如果建筑未平行于世界X/Z轴摆放，Box Projection产生的反射背景会彻底错位。
  - **墙体厚度与遮挡**：为了防止室内的反射探针、光照探针范围向外溢出导致穿墙露馅，游戏关卡的建筑墙体通常会被设计得非常厚，或在室外墙根放置杂物（如木桶、箱子）使用户无法紧贴墙壁。

### 2. 探针布置（TA / 地编核心杂活）
- **全局与局部反射探针搭配**：室外天空中务必放置一个影响全图、Importance设置为 `0` 的全局反射探针提供基础环境；进入室内后，设置 Importance 更高的局部探针并开启 Blend 融合距离。
- **光照探针的内外隔离带**：光照探针（Light Probes）的插值机制会导致严重漏光。**切忌只在房间内部打点**，必须在房间内部、墙壁边缘、房间外侧建立两到三层探针隔离带，防止室外刺眼的亮光或红色间接光插值影响到处于暗室内的动态物体。

### 3. 光照烘焙与后处理防坑
- **第三方烘焙替代**：Unity默认的光照烘焙（Enlighten/CPU/GPU）在复杂场景下极度缓慢，工程级开发常推荐使用第三方插件 **Bakery** (依赖Nvidia显卡) 来提速与增强控制。
- **后处理调色的原则**：后处理是为锦上添花，**必须在场景物理光照（基础打光）彻底调对的前提下进行**。不能企图用后处理掩盖灯光亮度的基础错误。
- **Shader 脚本报错**：从 Built-in 转 URP 编写代码时，切记不可图方便将属性裸露在全局。一旦触发全局属性（被编译器打包进 `Globals`），该 Shader 将永远被踢出 SRP Batcher 合批列表，造成极其严重的 Draw Call 性能灾难。
