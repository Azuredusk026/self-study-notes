# 图算法（二）：进阶图算法与最短路径问题

## 一、 核心知识点与原理剖析

### 1. 最短路径问题（Shortest Path Problems）的定义与变体
在图论中，**最短路径问题**是指在一个带权重的图（有向图或无向图）中，寻找两个顶点之间权重之和最小的路径。
*   **数学描述**：给定一个带权重的有向图 $G = (V, E)$ 和权重函数 $w: E \to \mathbb{R}$。一条路径 $p = \langle v_0, v_1, \dots, v_k \rangle$ 的权重 $w(p)$ 定义为构成该路径的所有边的权重之和：
    $$w(p) = \sum_{i=1}^{k} w(v_{i-1}, v_i)$$
*   **最短路径权重 $\delta(u, v)$** 定义为：
    $$\delta(u, v) = \begin{cases} \min \{ w(p) : u \xrightarrow{p} v \} & \text{若存在从 } u \text{ 到 } v \text{ 的路径} \\ \infty & \text{若从 } u \text{ 到 } v \text{ 不存在路径} \end{cases}$$
*   **最短路径问题的常见变体**：
    1.  **单源最短路径（Single-Source Shortest Path, SSSP）**：给定一个源节点 $s \in V$，求 $s$ 到图中每一个节点 $v \in V$ 的最短路径。
    2.  **单目的地最短路径（Single-Destination Shortest Path）**：找出每个节点 $v \in V$ 到给定目的地节点 $t$ 的最短路径。该问题可以通过将图中所有有向边反向，进而转化为单源最短路问题求解。
    3.  **单节点对最短路径（Single-Pair Shortest Path）**：找出从给定节点 $u$ 到给定节点 $v$ 的最短路径。从复杂度的角度看，目前没有比求解单源最短路径更优的算法，因此通常直接运行单源最短路径算法。
    4.  **所有节点对最短路径（All-Pairs Shortest Path, APSP）**：计算图中任意两个节点两两之间的最短路径。

### 2. 最短路径的最优子结构性质（Optimal Substructure）
最短路径具有**最优子结构**的性质。它是应用动态规划和贪心算法解决最短路径问题的前提。
*   **定理**：设 $p = \langle v_0, v_1, \dots, v_k \rangle$ 是从 $v_0$ 到 $v_k$ 的一条最短路径，对于任意的 $i, j$（满足 $0 \leq i \leq j \leq k$），$p$ 中从 $v_i$ 到 $v_j$ 的子路径 $p_{ij} = \langle v_i, v_{i+1}, \dots, v_j \rangle$ 也是从 $v_i$ 到 $v_j$ 的一条最短路径。
*   **反证法证明**：
    1.  设路径 $p$ 可被拆分为三部分：$v_0 \xrightarrow{p_{0i}} v_i \xrightarrow{p_{ij}} v_j \xrightarrow{p_{jk}} v_k$，其总长度为 $w(p) = w(p_{0i}) + w(p_{ij}) + w(p_{jk})$。
    2.  假设从 $v_i$ 到 $v_j$ 存在另一条更短的子路径 $p'_{ij}$，满足 $w(p'_{ij}) < w(p_{ij})$。
    3.  我们可以用该路径替换原有子路径，拼接得到一条新路径 $p' = v_0 \xrightarrow{p_{0i}} v_i \xrightarrow{p'_{ij}} v_j \xrightarrow{p_{jk}} v_k$。
    4.  新路径的权重为 $w(p') = w(p_{0i}) + w(p'_{ij}) + w(p_{jk}) < w(p)$。
    5.  这与“$p$ 是从 $v_0$ 到 $v_k$ 的最短路径”这一前提相矛盾。因此，假设不成立，$p_{ij}$ 必须是从 $v_i$ 到 $v_j$ 的最短路径。

### 3. 负权边与负值环路（Negative-Weight Edges and Cycles）
在带权图中，边的权重可能是负数。负权边的存在会给最短路径的求解带来复杂性。
*   **可达负值环路**：如果图中存在一个环路，且该环路上所有边的权重之和为负数，从源点 $s$ 可以到达该环路，那么从 $s$ 到该环路上的所有节点的最短路都是无定义的（或者可以认为是 $-\infty$）。因为我们可以选择在这个环路上无限次绕圈，每绕一圈，路径的累加权重就会减小，直至无限小。
*   **最短路径是否能包含环路**？
    *   **不能包含负权环**：否则可以通过在环上绕圈使路径权重无限小，此时最短路径无定义。
    *   **不能包含正权环**：如果路径中包含正权环，只要将这个环删去，就能得到一条更短的路径。
    *   **不能包含零权环**：虽然删去零权环不会改变路径的总权重，但会减少路径的边数，我们总可以将其删去以获得一条简单路径（不含环路的路径）。
    *   **结论**：**任何最短路径一定是一条简单路径，因此其包含的边数至多为 $|V| - 1$ 条。**

### 4. 松弛操作（Relaxation）及其性质
在寻找最短路径的算法中，每个节点 $v$ 都需要维护两个属性：
1.  $v.d$：记录从源节点 $s$ 到节点 $v$ 的最短路径权重上界（称为最短路径估计值）。
2.  $v.\pi$：指向最短路径上 $v$ 的前驱节点，用于最终恢复路径结构。

*   **松弛函数（RELAX）**：松弛操作是更新最短路径估计值的核心。它的逻辑是：如果发现从源点 $s$ 经过节点 $u$ 到达节点 $v$ 的路径比当前已知的路径更短，则更新 $v$ 的估计值 $v.d$ 与前驱 $v.\pi$。
    ```text
    RELAX(u, v, w)
    if v.d > u.d + w(u, v) then
        v.d = u.d + w(u, v)
        v.pi = u
    ```
*   **松弛操作的关键性质**：
    *   **三角不等式性质（Triangle Inequality）**：对于任意的边 $(u, v) \in E$，从源点 $s$ 到达 $v$ 的最短路必小于或等于先到 $u$ 再由边 $(u, v)$ 到达 $v$ 的距离，即 $\delta(s, v) \leq \delta(s, u) + w(u, v)$。
    *   **上界性质（Upper-Bound Property）**：对于所有节点 $v$，总有 $v.d \geq \delta(s, v)$。一旦 $v.d$ 达到了其真实最短路径权重 $\delta(s, v)$，它的值将永远保持不变，不会再变小或变大。
    *   **非路径性质（No-Path Property）**：如果从 $s$ 到达 $v$ 不存在可达路径，则始终有 $v.d = \delta(s, v) = \infty$。
    *   **收敛性性质（Convergence Property）**：如果 $s \leadsto u \to v$ 是图中的一条最短路径，且在对边 $(u, v)$ 进行松弛之前的任意时刻，已经求得 $u.d = \delta(s, u)$，则在此松弛之后的所有时刻，都有 $v.d = \delta(s, v)$。
    *   **路径松弛性质（Path-Relaxation Property）**：若 $p = \langle v_0, v_1, \dots, v_k \rangle$ 是从源点 $s=v_0$ 到节点 $v_k$ 的最短路径，如果对其所有的边按照顺序 $(v_0, v_1), (v_1, v_2), \dots, (v_{k-1}, v_k)$ 进行松弛，则最终必有 $v_k.d = \delta(s, v_k)$。**此性质不受在这些松弛中穿插的任何其他边松弛操作的影响。**

### 5. 贝尔曼-福特算法（Bellman-Ford Algorithm）
贝尔曼-福特算法可以解决包含负权重边的一般单源最短路径问题。

*   **算法原理**：
    由于最短路径必定不包含环路，因此最短路径至多由 $|V| - 1$ 条边构成。根据路径松弛性质，如果我们对所有的边进行 $|V| - 1$ 次松弛，那么不管边的松弛顺序如何，在第 $i$ 次对全部边的松弛中，最短路径上第 $i$ 条边的松弛必然会得到正确的结果。因此，重复进行 $|V| - 1$ 轮对所有边的松弛后，所有可达节点的最短路径估计值都会收敛到其最短路径权重。
*   **算法伪代码**：
    ```text
    BELLMAN-FORD(G, w, s)
    INITIALIZE-SINGLE-SOURCE(G, s) // 初始化：将所有节点的 d 设为无穷大，s.d 设为 0
    for i = 1 to |V| - 1 do
        for each edge (u, v) in E do
            RELAX(u, v, w)
    
    // 检查是否存在负权环
    for each edge (u, v) in E do
        if v.d > u.d + w(u, v) then
            return FALSE // 存在从源点可达的负权环，最短路无定义
    return TRUE
    ```
*   **复杂度分析**：
    外层循环执行 $|V| - 1$ 次，每次遍历 $|E|$ 条边进行松弛。因此，时间复杂度为 **$O(V \cdot E)$**。

### 6. 有向无环图（DAG）中的单源最短路径
当图是一个**有向无环图（DAG）**时，我们可以利用拓扑排序的线性顺序来优化最短路径的求解。

*   **算法原理**：
    在 DAG 中，拓扑排序保证了如果存在一条从 $u$ 到 $v$ 的有向路径，那么在拓扑排序的序列中 $u$ 一定排在 $v$ 之前。因此，我们只需要按照拓扑排序的顺序依次对每个节点出发的所有出边进行一轮松弛操作。这样只需处理一次，就能保证满足路径松弛的次序要求。
*   **算法伪代码**：
    ```text
    DAG-SHORTEST-PATHS(G, w, s)
    topologically sort the vertices of G
    INITIALIZE-SINGLE-SOURCE(G, s)
    for each vertex u, taken in topologically sorted order do
        for each vertex v in Adj[u] do
            RELAX(u, v, w)
    ```
*   **复杂度分析**：
    *   拓扑排序复杂度为 $O(V + E)$。
    *   松弛过程对每个节点的出边仅进行一次遍历，时间复杂度为 $O(V + E)$。
    *   **总时间复杂度**：**$O(V + E)$**。此算法对带负权无环图同样有效。

### 7. 迪杰斯特拉算法（Dijkstra's Algorithm）
迪杰斯特拉算法用于解决**非负权重边**的有向图或无向图的单源最短路径问题。

*   **基本原理与贪心策略**：
    算法维护一个集合 $S$，该集合内的节点的最短路径值已经完全确定。在每一步中，算法从不在 $S$ 中的节点集合 $Q = V \setminus S$ 中，选择当前最短路径估计值 $u.d$ 最小的节点 $u$ 加入到 $S$ 中，接着对从 $u$ 出发的所有边 $(u, v)$ 进行松弛。
*   **伪代码**：
    ```text
    DIJKSTRA(G, w, s)
    INITIALIZE-SINGLE-SOURCE(G, s)
    S = empty set
    Q = G.V // 将所有顶点放入优先队列 Q
    while Q is not empty do
        u = EXTRACT-MIN(Q) // 弹出当前 d 值最小的节点
        S = S union {u}
        for each vertex v in G.Adj[u] do
            RELAX(u, v, w)
            if relax updated v.d then
                DECREASE-KEY(Q, v, v.d) // 维护优先队列中的排序
    ```
*   **数据结构选择与复杂度分析**：
    Dijkstra 算法的复杂度取决于优先队列 $Q$ 的底层实现方式。包含三种操作：`Insert`、`Extract-Min`、`Decrease-Key`。
    
    1.  **普通数组（Array）**：
        *   每次 `Extract-Min` 需要扫描整个数组，耗时 $O(V)$。总共执行 $|V|$ 次，耗时 $O(V^2)$。
        *   每次 `Decrease-Key` 只需 $O(1)$。总共执行 $|E|$ 次，耗时 $O(E)$。
        *   **总复杂度**：**$O(V^2)$**。对于稠密图（$E \approx V^2$），此实现非常高效。
    2.  **二叉堆（Binary Heap）**：
        *   每次 `Extract-Min` 耗时 $O(\log V)$。总共执行 $|V|$ 次，耗时 $O(V \log V)$。
        *   每次 `Decrease-Key` 耗时 $O(\log V)$。总共执行 $|E|$ 次，耗时 $O(E \log V)$。
        *   **总复杂度**：**$O((V + E) \log V)$**。在所有顶点均可达时，简化为 **$O(E \log V)$**。适用于稀疏图（$E \ll V^2$）。
    3.  **斐波那契堆（Fibonacci Heap）**：
        *   `Extract-Min` 为 $O(\log V)$，`Decrease-Key` 摊还复杂度为 $O(1)$。
        *   **总复杂度**：**$O(V \log V + E)$**。在理论上是目前最优的渐近运行时间，但由于常数项较大且实现复杂，在实际工程中极少使用。

### 8. 所有节点对最短路径问题（All-Pairs Shortest Paths）与动态规划解法
求图中任意两点之间的最短路径。如果用单源最短路算法解决该问题：
*   **无负权边**：对每个节点运行一次 Dijkstra。如果用普通数组，复杂度为 $O(V^3)$；如果用二叉堆，复杂度为 $O(V E \log V)$。
*   **有负权边**：对每个节点运行一次 Bellman-Ford，复杂度为 $O(V^2 E)$。在稠密图下为 $O(V^4)$。

我们可以使用基于动态规划的解法来求得更好的复杂度。
*   **解法一：基于路径边数的动态规划**
    *   **状态定义**：设 $l_{ij}^{(m)}$ 表示从节点 $i$ 到节点 $j$ 的，且至多包含 $m$ 条边的任意路径的最小权重。
    *   **基本情况 ($m=0$)**：
        $$l_{ij}^{(0)} = \begin{cases} 0 & \text{若 } i = j \\ \infty & \text{若 } i \neq j \end{cases}$$
    *   **状态转移方程**：
        $$l_{ij}^{(m)} = \min_{1 \leq k \leq n} \left\{ l_{ik}^{(m-1)} + w_{kj} \right\}$$
        该转移方程表明：从 $i$ 到 $j$ 至多包含 $m$ 条边的最短路径，等于选择一个中间前驱节点 $k$，使 $i$ 到 $k$ 至多包含 $m-1$ 条边的路径与边 $(k, j)$ 的权重之和最小。
    *   **复杂度**：计算一步矩阵更新的结构类似于矩阵乘法（用 $\min$ 代替加法，用 $+$ 代替乘法），需要 $O(n^3)$ 时间。由于最多经过 $n-1$ 条边，需要迭代计算 $n-1$ 次，因此总复杂度为 **$O(n^4)$**。

### 9. 弗洛伊德-华沙尔算法（Floyd-Warshall Algorithm）
弗洛伊德-华沙尔算法是一种更为高效的求解所有节点对最短路径问题的动态规划算法。

*   **算法原理**：
    不按路径的边数划分阶段，而是按**可以使用的中间节点集合**划分阶段。
    设图的顶点集为 $V = \{1, 2, \dots, n\}$。
    定义 $d_{ij}^{(k)}$ 表示从节点 $i$ 到节点 $j$ 且其所有中间节点均取自集合 $\{1, 2, \dots, k\}$ 的一条最短路径的权重。
*   **状态转移方程**：
    *   **边界情况 ($k = 0$)**：中间不经过任何其他节点，路径即为直接相连的边：
        $$d_{ij}^{(0)} = w_{ij}$$
    *   **状态转移**：对于 $k \geq 1$，考虑当前第 $k$ 个节点是否为最短路径上的中间节点：
        1.  若第 $k$ 个节点**不是**中间节点，则最短路径的中间节点均在 $\{1, \dots, k-1\}$ 中，路径权重为 $d_{ij}^{(k-1)}$。
        2.  若第 $k$ 个节点**是**中间节点，则该路径可拆分为 $i \to k$ 和 $k \to j$ 的两条子路径。因为不能含有权值为负的环路，所以这两条子路径的中间节点必然都属于 $\{1, \dots, k-1\}$，因此路径权重为 $d_{ik}^{(k-1)} + d_{kj}^{(k-1)}$。
        
        由此得到递推式：
        $$d_{ij}^{(k)} = \min \left( d_{ij}^{(k-1)}, d_{ik}^{(k-1)} + d_{kj}^{(k-1)} \right)$$
*   **伪代码实现**：
    ```text
    FLOYD-WARSHALL(W)
    n = W.rows
    D^(0) = W
    for k = 1 to n do
        for i = 1 to n do
            for j = 1 to n do
                d_ij^(k) = min(d_ij^(k-1), d_ik^(k-1) + d_kj^(k-1))
    return D^(n)
    ```
    *注：在实际代码编写中，我们可以把上标维度去掉，直接在一张二维表 `d[i][j]` 上进行原地更新。*
*   **复杂度分析**：
    三重循环嵌套，每层执行 $n$ 次。时间复杂度为 **$O(n^3)$**（或 **$O(V^3)$**），空间复杂度可优化至 **$O(V^2)$**。

### 10. 有向图的传递闭包（Transitive Closure of a Directed Graph）
对于一个有向图 $G=(V, E)$，我们希望确定对于每一对节点 $i$ 和 $j$，是否存在一条从 $i$ 到 $j$ 的有向路径。

*   **定义**：图 $G$ 的传递闭包定义为图 $G^* = (V, E^*)$，其中：
    $$E^* = \{ (i, j) : \text{图 } G \text{ 中存在一条从 } i \text{ 到 } j \text{ 的路径} \}$$
*   **基于 Floyd-Warshall 的求解算法**：
    我们可以将弗洛伊德算法中的算术运算换成逻辑运算。
    定义 $t_{ij}^{(k)}$ 值为 $1$（表示从 $i$ 到 $j$ 存在一条中间节点均取自 $\{1, \dots, k\}$ 的路径）或 $0$（表示不存在）。
    *   **状态转移方程**：
        *   **边界情况 ($k = 0$)**：
            $$t_{ij}^{(0)} = \begin{cases} 1 & \text{若 } i = j \text{ 或 } (i, j) \in E \\ 0 & \text{其他情况} \end{cases}$$
        *   **状态转移 ($k \geq 1$)**：
            $$t_{ij}^{(k)} = t_{ij}^{(k-1)} \lor \left( t_{ik}^{(k-1)} \land t_{kj}^{(k-1)} \right)$$
            其含义是：从 $i$ 到 $j$ 有路径，要么是已经存在不通过 $k$ 的路径（左项），要么是存在一条从 $i$ 到 $k$ 的路径并且存在一条从 $k$ 到 $j$ 的路径（右项）。
*   **性能优化**：由于是逻辑运算，在工程实现上可以利用计算机的位运算（如 `BitSet` 或 `long` 类型按位或运算）极大地降低常数时间，提高运行速度。时间复杂度仍为 **$O(V^3)$**。

## 二、 重点术语与概念解析

*   **单源最短路径（Single-Source Shortest Path）**：从图中某一个特定源顶点出发，到其他所有顶点的最短路径的计算问题。
*   **前驱子图（Predecessor Subgraph）**：由每个顶点的当前前驱节点指针 $\pi$ 隐式定义出的有向子图 $G_\pi = (V_\pi, E_\pi)$。在算法运行结束时，该子图会形成一棵以源点为根的**最短路径树（Shortest-Path Tree）**。
*   **松弛操作（Relaxation）**：最短路径算法的核心步骤。通过引入中间顶点来测试是否能缩短从起点到目标顶点的当前已知估算距离的操作。
*   **三角不等式（Triangle Inequality）**：图论中关于最短路径的基本公理。对于任意三个顶点，直接从 $A$ 到 $C$ 的最短距离不会超过经过 $B$ 中转的距离之和。
*   **拓扑排序（Topological Sort）**：对有向无环图（DAG）的所有顶点进行的一种线性排序，使得任何有向边 $(u, v)$，起点 $u$ 在序列中均排在终点 $v$ 的前面。
*   **传递闭包（Transitive Closure）**：表示图的顶点之间连通性（可达性）的一种矩阵或图结构，主要用于快速查询两点间是否“可达”。

## 三、 工程经验与避坑指南

### 1. 关于无手写堆时的 Dijkstra 实现选择
*   **痛点**：在很多在线评测系统（Online Judge, OJ）或旧版开发环境中，不支持现成的 `PriorityQueue`，需要开发者手写堆来进行 Dijkstra 算法的优化。
*   **堆优化的细节要求**：手写小顶堆不仅要支持弹出堆顶元素（`Extract-Min`），而且在成功松弛一条边使得节点 $v$ 的估计值 $v.d$ 变小时，还需要支持修改堆内元素优先级的操作（即 `DecreaseKey`）。
    *   为了能以 $O(1)$ 的时间找到顶点 $v$ 在堆数组中的当前索引，必须维护一个**反向索引表**（如 `pos[v]` 记录节点 $v$ 在堆中的位置）。
    *   **核心避坑**：在堆进行节点交换时，**不仅要交换堆元素的值，还必须同步更新这两个顶点的反向索引表值**。否则，后续的 `DecreaseKey` 操作无法准确定位节点位置，导致算法出错或超时。
    *   在堆中交换元素的标准逻辑参考：
        ```csharp
        // 交换 heap[i] 和 heap[j] 的数据，并同步更新反向位置索引 pos
        int u = heap[i];
        int v = heap[j];
        heap[i] = v;
        heap[j] = u;
        pos[u] = j;
        pos[v] = i;
        ```
*   **工程折中方案**：如果图的规模较小（例如顶点数 $V \leq 1000$），且边数非常稠密，使用 **$O(V^2)$ 时间复杂度的普通数组 Dijkstra 算法**往往比手写堆性能更佳，且代码实现极短、不易出错，能够有效避免手写堆带来的调试负担。

### 2. 无穷大（$\infty$）的数值选择与溢出防范
在代码实现时，我们常常需要用一个数值代表无穷大（$\infty$）来初始化距离数组。
*   **危险操作**：使用 `int.MaxValue`（C#）或 `INT_MAX`（C++）作为无穷大。
*   **产生后果**：在松弛公式 `if (v.d > u.d + w(u, v))` 中，如果 $u$ 当前不可达，即 `u.d == int.MaxValue`，那么 `u.d + w(u, v)` 就会发生**整数溢出**变成一个极小的负数。这会导致 `v.d` 被错误地更新，进而摧毁整个算法的正确性。
*   **避坑策略**：
    1.  在执行松弛判断前，先显式检查 `if (u.d != INF)`。
    2.  或者，使用一个“足够大但不会溢出”的值作为无穷大。在图论中，最常用的常数是 **`0x3f3f3f3f`**（十进制为 1061109567）。该值的优势在于：
        *   足够大，一般可以覆盖所有的常规路径权重。
        *   **两个该值相加不会溢出**（因为 $1.06 \times 10^9 + 1.06 \times 10^9 \approx 2.12 \times 10^9$，未超出 `int` 的最大上限 $2.14 \times 10^9$），可以直接参与算术运算。
        *   可以使用内存填充函数（如 C++ 中的 `memset`）对其进行快速批量初始化。

### 3. Floyd-Warshall 算法的循环顺序
*   **经典错误**：由于直觉影响，容易将中间节点 $k$ 的循环写在内层，例如把 $i$ 或 $j$ 写在最外层。
*   **正确写法**：
    ```csharp
    // 必须将 k 放在最外层循环
    for (int k = 1; k <= n; k++) {
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= n; j++) {
                d[i, j] = Math.Min(d[i, j], d[i, k] + d[k, j]);
            }
        }
    }
    ```
*   **逻辑解释**：Floyd-Warshall 算法核心是基于动态规划的阶段推进。外层循环 $k$ 的每一次迭代，都代表着允许在路径中加入前 $k$ 个顶点作为中转站。如果把 $k$ 放在内层，意味着我们还没完全计算出只允许经过较少中转节点的最短路，就直接去计算包含了较大编号节点的最短路，这样无法保证状态转移方程的正确性。

### 4. 稀疏图与稠密图的最短路径选择
*   当求解单源最短路时，需依据图的稠密程度选择算法：
    *   **稀疏图（边数 $E$ 较少，如树形或网格图）**：优先使用**堆优化版的 Dijkstra**（边权非负，复杂度 $O(E \log V)$）或 **Bellman-Ford / SPFA 算法**。
    *   **稠密图（边数 $E \approx V^2$）**：直接使用**数组实现的无堆 Dijkstra**，其复杂度 $O(V^2)$ 常数项极小，比堆优化的 $O(V^2 \log V)$ 执行效率更高。
*   求解多源最短路时：
    *   若图较稀疏，可以对每个顶点分别调用 Dijkstra（堆优化版），复杂度为 $O(V \cdot E \log V)$。
    *   若图较稠密，直接采用 **Floyd-Warshall 算法**（复杂度 $O(V^3)$），不仅编码极其简练，常数极小，且对负权边同样适用。
