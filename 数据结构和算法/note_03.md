# 数据结构与算法：快速排序、线性数据结构、图与树、二叉堆及散列表

## 一、 核心知识点与原理剖析

### 1. 分治算法之快速排序（Quick Sort）
快速排序是目前所有排序算法中应用最广泛的一种算法，例如 C++ 中的 `std::sort` 和 C# 的默认排序算法在底层都主要基于快速排序的思想实现。
- **分治策略三步法**：
  1. **分解（Divide）**：将数组 $A[p \dots r]$ 划分为两个（可能为空）子数组 $A[p \dots q-1]$ 和 $A[q+1 \dots r]$。选择一个主元（Pivot） $x = A[r]$，使得左子数组 $A[p \dots q-1]$ 中的每一个元素都小于等于 $A[q]$（主元归位后的最终位置），而右子数组 $A[q+1 \dots r]$ 中的每一个元素都大于等于 $A[q]$。计算出主元的索引 $q$。
  2. **解决（Conquer）**：通过递归调用快速排序，对子数组 $A[p \dots q-1]$ 和 $A[q+1 \dots r]$ 分别进行排序。
  3. **合并（Combine）**：由于快速排序属于**原址排序**（In-place Sorting），子区域就地排好序后整个区域即已排序完成，因此不需要显式的合并操作。
- **原址排序与非原址排序**：
  - **非原址排序**（如归并排序）：合并子数组时需要开辟与元素总数呈线性关系的额外内存空间（辅助数组）来暂存归并结果。
  - **原址排序**：在排序过程中，只需要常数级别的辅助空间（如交换变量），直接在原数组内进行元素的位置交换。
- **划分函数（Partition）的实现逻辑**：
  - 主元选取：通常选取当前待排区间的最后一个元素 $x = A[r]$ 作为主元。
  - 双指针机制：
    - 指针 $i$：表示小于等于主元区域（$L$ 区）的右边界。初始化为 $p-1$。
    - 指针 $j$：当前扫描元素的指针，从 $p$ 遍历至 $r-1$。
  - 遍历过程中：若发现当前元素 $A[j] \le x$，说明 $A[j]$ 应归入 $L$ 区，则先将边界 $i$ 向右扩展一格（$i = i + 1$），再交换 $A[i]$ 与 $A[j]$；若 $A[j] > x$，则无需任何操作，仅让指针 $j$ 继续自增，此时大于主元区域（$R$ 区）自然向右扩展。
  - 循环结束后，将主元 $A[r]$ 与大于区间的第一个元素 $A[i+1]$进行交换，使主元放到正确的位置，并返回该位置的索引 $q = i+1$。
  - **算法实现伪代码**：
    ```python
    PARTITION(A, p, r)
        x = A[r]                      # 选定主元为最后一个元素
        i = p - 1                     # 初始化小于等于区域的右端点
        for j = p to r - 1            # 遍历除主元外的所有元素
            if A[j] <= x              # 发现小于等于主元的元素
                i = i + 1             # 扩大小于等于区域
                exchange A[i] with A[j] # 交换到该区域中
        exchange A[i + 1] with A[r]   # 将主元移至中间
        return i + 1                  # 返回主元的最终下标

    QUICKSORT(A, p, r)
        if p < r                      # 递归基：区间内元素数量多于1个
            q = PARTITION(A, p, r)    # 划分并获取主元位置
            QUICKSORT(A, p, q - 1)    # 递归排序左子数组
            QUICKSORT(A, q + 1, r)    # 递归排序右子数组
    ```
- **快速排序性能与复杂度分析**：
  - 快速排序的时间复杂度由每次划分的对称性决定。
  - **最好情况（完美平衡）**：每次划分都将数组均分为大小各为 $n/2$ 的子问题。
    - 递推式为：$T(n) = 2T(n/2) + \Theta(n)$。根据主定理，解得时间复杂度为 $O(n \log n)$。
  - **最坏情况（极端不对称）**：每次划分选出的主元都是当前区间的最大或最小元素，导致子问题大小分别为 $0$ 和 $n-1$。
    - 递归式为：$T(n) = T(n-1) + T(0) + \Theta(n)$。
    - 展开为累加求和：
      $$T(n) = T(n-1) + cn = c \sum_{i=1}^n i = c \cdot \frac{n(n+1)}{2} = \Theta(n^2)$$
      该情况在数组已经是升序或降序排列时容易发生。
  - **一般情况（非均匀常数比例划分，如 9:1 分割）**：
    - 假定每次划分比例均为 $9:1$，递归式为：$T(n) = T(n/10) + T(9n/10) + cn$。
    - 在递归树中，左子树最快达到边界，深度为 $\log_{10} n$；右子树最慢达到边界，深度为 $\log_{10/9} n$。
    - 每一层的代价总和最高为 $cn$。因此总时间为：
      $$T(n) \le cn \cdot (\log_{10/9} n + 1) = O(n \log n)$$
    - 根据对数换底公式，$\log_{10/9} n = \frac{\log_2 n}{\log_2(10/9)} \approx 6.58 \log_2 n$。由此可知，只要划分比例保持为常数比，其时间复杂度依然是 $O(n \log n)$，区别仅在于常数因子。
  - **随机化快速排序优化**：为了避免遭遇最坏情况，可以在进行划分前，随机在区间内选择一个元素，与 $A[r]$ 进行交换。这能确保在概率分布上极大概率实现均匀划分，使算法期望运行时间保持在 $O(n \log n)$。

### 2. 基本线性数据结构与哨兵节点
- **栈（Stack）**：后进先出（LIFO）的数据结构。
  - 用数组模拟时需维护一个顶指针 $top$。
  - 边界溢出：
    - 栈下溢（Underflow）：在 $top = 0$（空栈）时执行弹出（Pop）操作。
    - 栈上溢（Overflow）：当元素个数超过数组最大容量 $m$ 时执行压入（Push）操作。
- **队列（Queue）**：先进先出（FIFO）的数据结构。
  - 采用循环数组实现。维护队头指针 $head$（指向队头元素）和队尾指针 $tail$（指向新元素即将插入的位置）。
  - 当 $head = tail$ 时队列为空。当队尾指针追上队头指针时，表示队列已满。
- **双向链表（Doubly Linked List）**：
  - 节点包含数据键值 `key`，前驱指针 `Prev`，后继指针 `Next`。
  - **哨兵（Sentinel）优化**：
    - 哨兵节点是一个哑对象（Dummy Object）`L.nil`，代表链表末端及头部。
    - 引入哨兵后，链表成为一个双向循环链表：`L.nil.next` 指向真正的头节点，`L.nil.prev` 指向尾节点；头节点的 `prev` 指向 `L.nil`，尾节点的 `next` 指向 `L.nil`。
    - **原理**：不需要在代码中为 `x.prev == null` 或 `x.next == null` 特别编写繁琐的特判逻辑，合并了边界逻辑，极大简化了指针重接的代码量。

### 3. 图与树的理论基础
- **图的基本定义**：
  - **有向图** $G = (V, E)$：边集 $E$ 中的元素为有顺序的顶点对。可能存在指向自身的边（自环，Self-loop）。
  - **无向图** $G = (V, E)$：边集 $E$ 中的元素为无顺序的顶点对。
- **树与森林**：
  - **自由树（Free Tree）**：一个连通且无环的无向图。
  - **森林（Forest）**：一个无环但非连通的无向图（包含多棵不相连的自由树）。
  - **自由树定理**：若一个无向图 $G=(V, E)$ 是自由树，则必然满足：
    $$|E| = |V| - 1$$
    *定理证明*：考虑合并过程。初始状态下有 $|V|$ 个孤立的点（即 $|V|$ 个连通分量）。每在不形成环的两个连通分量之间添加一条边，连通分量的数量就会减少 1。为了将整个图连通成一个单一的连通分量（只剩 1 个连通分量），且中途不形成任何环路，需要添加的边数恰好为顶点数减 1，即 $|V| - 1$。
- **有根树（Rooted Tree）关系术语**：
  - **根（Root）**：树中指定的唯一特殊起始节点。
  - **父节点（Parent）/子节点（Child）**：从根到节点 $x$ 的唯一路径上，在 $x$ 之前的最后一个节点 $y$ 是 $x$ 的父节点， $x$ 是 $y$ 的子节点。
  - **祖先（Ancestor）/后代（Descendant）**：从根到节点 $x$ 路径上的任意节点 $y$（包括 $x$ 自身）称为 $x$ 的祖先， $x$ 是 $y$ 的后代。若 $y \ne x$，则称其为真祖先/真后代。
  - **叶节点（Leaf）/内部节点（Internal Node）**：叶节点是没有子节点的节点；内部节点是至少有一个子节点的非叶节点。
  - **节点的度（Degree）**：在该树结构中，指其子节点的数目（注意：与图论中顶点的度数定义有区别）。
  - **深度（Depth）/高度（Height）**：节点在树中的深度是指从根节点到该节点路径上的边数。树的高度是所有节点深度的最大值。
  - **有序树（Ordered Tree）**：子节点排定先后次序的有根树。如果将子节点顺序对调，即构成一棵新树。
- **二叉树分类**：
  - **满二叉树（Full Binary Tree）**：每个节点的度数只能为 0 或 2。
  - **完美二叉树（Perfect Binary Tree）**：所有非叶节点的度数都为 2，且所有叶节点都在同一深度的层级。高度为 $k$ 的完美二叉树（若单一节点高度记为 1），其节点数为 $2^k - 1$。
  - **完全二叉树（Complete Binary Tree）**：除最后一层外所有层全满，最后一层的节点全部紧密靠左排列。
- **多叉树的高效表示（左孩子右兄弟表示法）**：
  - 若每个节点维护最大孩子数 $k$ 个指针，当许多节点的实际孩子数小于 $k$ 时，会造成严重的空间浪费。
  - **左孩子右兄弟（Left-Child Right-Sibling）表示法**中，每个节点只维护 3 个指针：
    1. `p`：父节点指针。
    2. `left-child`：指向该节点的最左侧子节点。
    3. `right-sibling`：指向该节点右侧紧邻的兄弟节点。
  - 该结构将任意节点的子节点以单向链表的形式串联起来，使空间占用大幅缩减为 $\Theta(1)$ 的固定指针开销。

### 4. 二叉堆与优先队列（Binary Heap & Priority Queue）
- **数组存储结构**：由于二叉堆满足完全二叉树的性质，我们可以将其映射到一个一维数组中进行存储，而无需使用指针。
  - 设堆的当前大小为 $n$，数组首位下标为 1，则对于索引为 $i$ 的节点：
    - 父节点索引：$\text{PARENT}(i) = \lfloor i / 2 \rfloor$
    - 左孩子索引：$\text{LEFT}(i) = 2i$
    - 右孩子索引：$\text{RIGHT}(i) = 2i + 1$
  - 堆的叶子节点下标区间为：$\lfloor n/2 \rfloor + 1 \dots n$。
- **最大堆（Max-Heap）与最小堆（Min-Heap）**：
  - **最大堆**满足：除了根节点，有 $A[\text{PARENT}(i)] \ge A[i]$。堆顶元素必定是最大值。
  - **最小堆**满足：除了根节点，有 $A[\text{PARENT}(i)] \le A[i]$。堆顶元素必定是最小值。
- **核心操作原理与分析**：
  - **`MAX-HEAPIFY(A, i)` （维护最大堆性质）**：
    - 假定以 $LEFT(i)$ 和 $RIGHT(i)$ 为根的子树都已经是最大堆，但 $A[i]$ 可能会小于它的两个孩子，从而破坏了堆性质。
    - 通过将 $A[i]$、左孩子、右孩子三者中的最大者找出来。如果是其中一个孩子最大，则将 $A[i]$ 与该孩子进行位置交换，随后递归对发生交换的子节点位置继续调用 `MAX-HEAPIFY`。
    - **非递归实现**：通过循环更新当前节点索引，避免函数递归的栈空间消耗。
  - **`BUILD-MAX-HEAP(A)`（自底向上建堆）**：
    - 从倒数第一层非叶子节点开始，即从下标 $\lfloor n / 2 \rfloor$ 递减到 $1$，依次对每个节点调用 `MAX-HEAPIFY`。
    - **运行时间分析**：
      堆中某节点调用 `MAX-HEAPIFY` 的代价是与其高度 $h$ 成正比。在一个有 $n$ 个节点的堆中，高度为 $h$ 的节点至多有 $\lceil n / 2^{h+1} \rceil$ 个。总建堆时间为：
      $$T(n) = \sum_{h=0}^{\lfloor \log n \rfloor} \lceil \frac{n}{2^{h+1}} \rceil O(h) = O\left( n \sum_{h=0}^{\infty} \frac{h}{2^h} \right)$$
      因为级数 $\sum_{h=0}^{\infty} \frac{h}{2^h}$ 收敛于一个常数，故建堆的时间复杂度为 $\Theta(n)$，而非直观上的 $O(n \log n)$。
  - **优先队列应用**：
    - `HEAP-EXTRACT-MAX(A)`：读取并返回 $A[1]$。把堆尾元素 $A[A.heap\text{-}size]$ 替换到根节点位置，将堆大小减 1，再对根节点执行 `MAX-HEAPIFY(A, 1)` 以重新归位。时间复杂度为 $O(\log n)$。
    - `HEAP-INCREASE-KEY(A, i, key)`：将节点 $i$ 的值增至 `key`（要求 `key` $\ge A[i]$）。不断将其与父节点比较，如果大于父节点则上浮交换，直到满足性质或浮至根节点。时间复杂度为 $O(\log n)$。

### 5. 散列表（Hash Table）
- **基本思想**：当关键字范围非常庞大时，直接寻址表（开辟与关键字全域大小相同的数组）会导致严重的内存浪费。散列表通过散列函数 $h(k)$ 将较大的关键字域 $U$ 映射到大小为 $m$ 的槽位（Slot）数组中。
- **散列冲突（Collision）**：两个不同的关键字被映射到了相同的槽中（即 $h(k_1) = h(k_2)$）。
- **解决冲突的方法**：
  1. **链接法（Chaining）**：
     - 将散列到同一个槽位的所有元素通过一个双向链表链接起来。
     - 若采用简单均匀散列假设，装载因子 $\alpha = n/m$（即每个槽 average 链表长度）。查找一个元素所需平均时间为 $\Theta(1 + \alpha)$。
  2. **开放寻址法（Open Addressing）**：
     - 所有的元素都在散列表中存储，没有外部链表。
     - 冲突时使用**探查序列**逐个检索空槽，形式为 $h(k, i) = \text{Slot}$，其中 $i$ 表示探查次数，范围为 $0 \dots m-1$。探查序列必须是表大小 $m$ 的一个排列。
     - **删除处理机制**：不能直接将删除的位置置为空（`Nil`），否则会导致后续插入的元素（之前发生冲突的元素）因链路断裂而“搜不到”。应使用特殊的 `Deleted` 状态标记删除的槽，查找遇到 `Deleted` 继续探查，插入时可以直接覆盖。
     - **三种探查实现**：
       - **线性探查**：每次往后移一位 $h(k, i) = (h'(k) + i) \bmod m$。容易形成**一次集群**（Primary Clustering），探查效率随集群长度线性增加。
       - **二次探查**：$h(k, i) = (h'(k) + c_1 i + c_2 i^2) \bmod m$。可以消除一次集群，但若初始散列位置相同，其探查路径依然相同，会导致**二次集群**（Secondary Clustering）。
       - **双重散列**：探查步长取决于第二个散列函数 $h_2(k)$，即 $h(k, i) = (h_1(k) + i \cdot h_2(k)) \bmod m$。消除了上述集群问题，是开放寻址中效率最高的手段之一。

## 二、 重点术语与概念解析

- **原址排序（In-place Sorting）**：
  算法执行过程中，仅需 $O(1)$ 的辅助工作空间，主要排序操作在原数据结构上完成的排序方法。
- **简单均匀散列假设（Simple Uniform Hashing Assumption）**：
  一种理想化的散列模型。假定任何一个关键字都被等概率地映射到 $m$ 个槽位中的任意一个，且该关键字映射的位置与其他关键字已映射的位置相互独立。
- **一次集群（Primary Clustering）**：
  开放寻址法中线性探查的主要缺陷。随着占用的槽变多，连续空位段被占满的可能性越来越高，导致原本无关的关键字也会共用越来越长的探查路径，导致性能严重退化。
- **哨兵节点（Sentinel Node）**：
  双向循环链表中的哑对象（如 `L.nil`），前驱指向尾节点，后继指向头节点。用来替代表示空指针的端点，省去了插入和删除操作中的许多边界判断。
- **双重散列互质要求**：
  在双重散列中，为了能保证探查序列能检索到表中的所有 $m$ 个槽位，辅助散列值 $h_2(k)$ 必须与表容量 $m$ 互质。否则，算法只能探索到 $\frac{m}{\gcd(m, h_2(k))}$ 个槽，很容易因为探查路径不完整而找不到已有的空闲槽，引发算法死循环或溢出错误。
- **二叉堆子树规模的极值（$2n/3$）**：
  对于大小为 $n$ 的二叉堆，当底层非空且正好半满时，其左子树拥有堆最多的节点数。数学推导表明，在此极值下，左子树的最大节点数约为整个堆的 $2n/3$（而右子树大小为 $n/3$）。这被用于最大堆下沉的最坏情况时间复杂度分析中。

## 三、 工程经验与避坑指南

- **C# / C++ 默认的排序底座**：
  在工程开发中，C# 和 C++ 的默认 `Sort()` 多半基于快速排序（或以快速排序为底座进行优化的 IntroSort 内省排序）。因为快排的局部性好且交换系数小，在实际表现上通常是同级别时间复杂度中最优的选择。
- **避坑：递归深度与空间超限（MLE）**：
  - **开发避坑**：在处理像“数据要排序”这类超大规模输入时，在快速排序的递归函数中绝对不能直接将大数组作为值拷贝传入，或者在每个递归栈帧内申请临时数组，否则极易由于递归过深导致栈溢出（Stack Overflow）或超空间限制。
  - **最佳实践**：应将待排序的数组声明为全局静态的（`static`），或者使用引用传递（C++ `&`），在递归中仅处理区间的左右下标 $p$ 和 $r$。
- **避坑：双重散列必须保证互质**：
  - **开发避坑**：使用双重散列时，如果不加控制地选择 $h_2(k)$，当它与 $m$ 存在公因子时，会导致很多空位置永远无法被写入，从而使得散列表即使有很多空位，也会报告“哈希表已满”。
  - **最佳实践**：若选择 $m$ 为 $2^p$，则辅助函数 $h_2(k)$ 必须返回一个**奇数**；若选择 $m$ 为素数，则必须设计 $h_2(k)$ 返回一个小于 $m$ 的**正整数**。
- **散列表除法散列法素数选择**：
  - **工程实践**：在使用除法散列法 $k \bmod m$ 时，表容量 $m$ 不能选择靠近 2 的整数幂的数。因为在大部分实际数据中，数据的二进制低位往往存在极大的相关性，容易造成分布不均。最佳的 $m$ 应该选择一个离 2 的整数幂较远的**素数**（例如 701，而不是 512 或 1024）。
- **进程优先级的动态调整与堆的应用**：
  - **工程经验**：操作系统的作业调度是优先队列的经典应用。作业通过最大堆来进行维护，每次取出优先级最高的进程。在实际系统开发中，为防止有些优先级稍低的进程饿死，通常会有一个动态“提升优先级”的机制，对应于堆的 `Heap-Increase-Key` 操作，将指定节点的键值改大并进行上浮重组。
- **非整数关键字如何做哈希映射**：
  - **编码转换**：在散列表中，非整数的关键字（如变量名字符串）必须首先编码转化为自然数。例如对字符串进行哈希时，可以把字符串视作一个大进制数。
  - **字符转换实例**：若以 128 为基底对 ASCII 字符串编码，对于字符串 `PT`：
    - 在 ASCII 码中，`P` 的码值为 112，`T` 的码值为 116。
    - 将其转化成 128 进制并计算十进制值：
      $$\text{Code} = 116 + 112 \times 128 = 116 + 14336 = 14452$$
    - 转换完的值即可安全地应用于除法或乘法散列函数中。
