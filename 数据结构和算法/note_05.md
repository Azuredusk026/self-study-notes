# 图算法基础、遍历与拓扑排序

## 一、 核心知识点与原理剖析

### 1. 图的基础概念与分类
* **图的定义**：图由顶点集合和边集合组成，记为 $G = (V, E)$。其中 $V$ 是顶点（点）的非空集合，$E$ 是顶点之间的关系集合（边集）。
* **有向图 (Directed Graph)**：边是有方向的。如果有一条边从顶点 $u$ 指向 $v$，则用有序对 $(u, v)$ 表示，称其“射出或离开”顶点 $u$，且“射入或进入”顶点 $v$。有向图中，$(u, v)$ 和 $(v, u)$ 是两条不同的边。自环（如从 $u$ 到 $u$ 的边）是有向图中的合法结构。不含自环的有向图称为**简单有向图**。
* **无向图 (Undirected Graph)**：边是无方向的。由无序顶点对 $(u, v)$ 组成，因此 $(u, v)$ 和 $(v, u)$ 代表同一条边。此时称边 $(u, v)$ 与顶点 $u$ 和 $v$ 关联。
* **邻接 (Adjacency)**：如果 $(u, v)$ 是图中的一条边，则称顶点 $v$ 邻接于顶点 $u$。
* **顶点的度 (Degree)**：
  * **无向图的度**：关联于该顶点的边的数目。
  * **有向图的度**：分为**出度**（离开该顶点的边的数目）和**入度**（进入该顶点的边的数目）。顶点的度为出度与入度之和。
* **路径 (Path)**：顶点序列 $\langle v_0, v_1, v_2, \dots, v_k \rangle$ 构成从 $u = v_0$ 到 $u' = v_k$ 的路径，其中对于所有的 $1 \le i \le k$，有 $(v_{i-1}, v_i) \in E$。路径的长度是该路径包含的**边数**（即 $k$）。如果从 $u$ 到 $u'$ 存在一条路径，则称 $u'$ 是从 $u$ 可达的。
* **简单路径 (Simple Path)**：路径中所有顶点互不相同的路径。
* **环 (Cycle)**：
  * **有向环**：起点和终点重合且至少包含一条边的路径（即 $v_0 = v_k$ 且 $k > 0$）。若序列中 $v_1$ 到 $v_k$ 互不相同，则该环是**简单环**。自环是长度为 1 的环。
  * **无向环**：满足 $k > 0$ 且 $v_0 = v_k$，同时路径上所有边互不相同的路径。若除起点终点外顶点互不相同，则为**简单环**。
* **连通图与连通分量**：
  * **连通图 (Connected Graph)**：无向图中，如果任意两个顶点之间都是互相可达的，则称该图是连通的（直观上是一整块区域，没有被分割）。
  * **连通分量 (Connected Component)**：无向图中顶点在“可达关系”下的等价类，是极大连通子图。
* **树的本质**：**树是一个无环连通无向图**。对包含 $n$ 个顶点的树，其边数必然为 $n-1$。在任意树中任意两点间连一条新边，都必定会且仅会形成一个环。

### 2. 图的表示方法
在计算机中表示图通常有以下两种方式：
* **邻接链表法 (Adjacency List)**：
  * 由一个包含 $|V|$ 个单链表的数组 `ADJ` 组成。每个顶点对应一条链表，链表中的节点表示与该顶点直接邻接的顶点。
  * **空间复杂度**：$\Theta(V + E)$。
  * **适用场景**：图比较稀疏时（$E \ll V^2$）。
  * **优缺点**：空间占用小；但无法在 $O(1)$ 时间内快速判断任意两个顶点之间是否有边关联（需要遍历对应顶点的链表）。
* **邻接矩阵法 (Adjacency Matrix)**：
  * 用一个 $|V| \times |V|$ 的二维矩阵 $A$ 表示图。如果顶点 $i$ 到 $j$ 有边，则 $A_{i,j} = 1$（或其权重），否则 $A_{i,j} = 0$。
  * **空间复杂度**：$\Theta(V^2)$。
  * **适用场景**：稠密图，或需要快速判断两点间是否有边的场景。
* **带权图 (Weighted Graph) 的存储**：
  * **邻接链表**：在链表的每个节点中额外存储一个权重值（例如顶点和长度的二元组）。
  * **邻接矩阵**：矩阵的元素 $A_{i,j}$ 直接存储对应边的权重值。**如果两点之间没有边相连，应当用无穷大 ($\infty$) 表示，而不能用 0 表示**（0 表示距离为 0 或无限接近，仍然代表有边）。

### 3. 广度优先搜索 (Breadth-First Search, BFS)
* **核心原理**：给定源点 $s$，BFS 按照“由近及远”的原则系统地扩展图中的边。它先访问所有距离源点为 1 的节点，然后再访问距离为 2 的节点，以此类推。因此，BFS 能够求出源点到所有可达节点的最短路径长度（最少边数）。
* **状态标记（三色模型）**：
  * **白色**：未被发现/访问的节点。
  * **灰色**：已被发现但未完全处理（其邻接节点中仍有白色节点），已加入搜索队列中。
  * **黑色**：已被发现且完全处理完毕（所有邻接节点都已变成灰色或黑色），已从队列中移出。
* **属性说明**：
  * $u.d$：从源点 $s$ 到顶点 $u$ 的最短路径长度。
  * $u.pi$（先驱节点）：指在搜索树中发现节点 $u$ 的上一个节点。
* **算法伪代码**：
  ```text
  BFS(G, s)
  1  for each vertex u in G.V - {s}
  2       u.color = WHITE
  3       u.d = INF
  4       u.pi = NIL
  5  s.color = GREY
  6  s.d = 0
  7  s.pi = NIL
  8  Q = ∅
  9  ENQUEUE(Q, s)
  10 while Q ≠ ∅
  11      u = DEQUEUE(Q)
  12      for each v in G.Adj[u]
  13           if v.color == WHITE
  14                v.color = GREY
  15                v.d = u.d + 1
  16                v.pi = u
  17                ENQUEUE(Q, v)
  18      u.color = BLACK
  ```
* **前驱子图 (Predecessor Subgraph)**：
  * BFS 搜索形成的前驱子图 $G' = (V', E')$，其中点集 $V'$ 是源点 $s$ 加上所有前驱非空的节点，边集 $E'$ 是由 $(u.pi, u)$ 构成的树边。该子图也称为 **BFS 搜索树**。不可达节点不会包含在该树中。
* **时间复杂度**：
  * 初始化所有节点耗时 $O(V)$。
  * 每个可达节点最多入队、出队各一次，队列操作总耗时 $O(V)$。
  * 每个节点出队时其邻接表仅被扫描一次，所有顶点邻接表长度之和为 $\Theta(E)$。
  * 总运行时间复杂度为 **$O(V + E)$**。

### 4. 深度优先搜索 (Depth-First Search, DFS)
* **核心原理**：尽可能深地探索图中的边。在搜索过程中，一旦发现某条路径没有未访问的邻接节点，就回溯到前一个节点，继续探索其他路径。
* **时间戳机制 (Timestamps)**：
  每个节点 $u$ 被记录两个重要时间戳：
  * $u.d$（发现时间）：节点第一次被发现（由白变灰）时的全局步数。
  * $u.f$（完成时间）：完成扫描该节点的所有邻接节点（由灰变黑）时的全局步数。
  * 全局时间 `time` 在每次状态转换时自增 1，所有时间戳位于 $1$ 到 $2|V|$ 之间。
* **算法伪代码**：
  ```text
  DFS(G)
  1  for each vertex u in G.V
  2       u.color = WHITE
  3       u.pi = NIL
  4  time = 0
  5  for each vertex u in G.V
  6       if u.color == WHITE
  7            DFS-Visit(G, u)

  DFS-Visit(G, u)
  1  time = time + 1
  2  u.d = time
  3  u.color = GREY
  4  for each v in G.Adj[u]
  5       if v.color == WHITE
  6            v.pi = u
  7            DFS-Visit(G, v)
  8  u.color = BLACK
  9  time = time + 1
  10 u.f = time
  ```
* **DFS 森林**：DFS 生成的前驱子图通常是由多棵树构成的**深度优先森林 (DFS Forest)**。
* **核心性质定理**：
  * **括号化定理 (Parenthesis Theorem)**：对于任意两个节点 $u$ 和 $v$，其生命周期区间 $[u.d, u.f]$ 和 $[v.d, v.f]$ 满足以下三种关系之一：
    1. 完全分离：区间互不重叠，代表它们在 DFS 树中互不为祖先或后代。
    2. $u$ 包含于 $v$：$[u.d, u.f] \subset [v.d, v.f]$，代表 $u$ 是 $v$ 的后代。
    3. $v$ 包含于 $u$：$[v.d, v.f] \subset [u.d, u.f]$，代表 $v$ 是 $u$ 的后代。
  * **后代区间嵌套定理 (Descendant Interval Nesting Theorem)**：节点 $v$ 是 $u$ 的真后代，当且仅当 $u.d < v.d < v.f < u.f$。
  * **白色路径定理 (White-Path Theorem)**：在 DFS 森林中，节点 $v$ 是节点 $u$ 的后代，当且仅当在发现 $u$ 的时刻 $u.d$，存在一条从 $u$ 到 $v$ 且全部由白色节点组成的路径。
* **边的分类**：
  根据首次探寻边 $(u, v)$ 时目标顶点 $v$ 的颜色，可将边分为以下四类：
  1. **树边 (Tree Edge)**：$v$ 是白色。表明发现了新节点。
  2. **后向边 (Back Edge)**：$v$ 是灰色。表明将节点连向其在 DFS 树中的祖先节点（包括自环）。
  3. **前向边 (Forward Edge)**：$v$ 是黑色，且 $u.d < v.d$。表明将节点连向其在 DFS 树中的后代节点。
  4. **横向边 (Cross Edge)**：$v$ 是黑色，且 $u.d > v.d$。表明连接同一棵树中没有祖先后代关系的节点，或者连接不同的深度优先树。
* **无向图的特殊性质**：
  **在无向图的 DFS 中，永远不会出现前向边和横向边**，所有非树边均被归类为后向边。
  * **证明**：设无向边为 $(u, v)$，且 $u.d < v.d$。这说明 $u$ 比 $v$ 先被发现。由于图是无向的，且 $u$ 与 $v$ 之间有边相连，在 $u$ 的生命周期结束（变成黑色）之前，DFS 必然会通过邻接关系去探索 $v$。
    * 若从 $u$ 探寻该边时，$v$ 尚未被发现（为白色），则 $(u, v)$ 成为树边。
    * 若从 $v$ 探寻该边时，由于 $u$ 处于搜索栈中且未完成（为灰色），则该边作为后向边被发现。
    因此，无向图中不可能存在已完成扫描且与当前点不属于祖先关系的黑色节点，故不存在前向边或横向边。

### 5. 拓扑排序 (Topological Sort)
* **概念**：针对有向无环图 (DAG) 的所有顶点进行的一种线性排序，使得对于图中的任意有向边 $(u, v)$，在排序序列中顶点 $u$ 都出现在顶点 $v$ 的前面。
* **有环图无法进行拓扑排序的证明**：
  假设存在环路 $v_1 \to v_2 \to \dots \to v_k \to v_1$。按照拓扑排序要求，$v_1$ 必须排在 $v_2$ 前面，$v_2$ 排在 $v_3$ 前面，……，$v_k$ 必须排在 $v_1$ 前面。然而，线性次序具有传递性，这要求 $v_1$ 必须排在它自己前面，这显然是不可能满足的。因此，包含环路的图无法进行拓扑排序。
* **基于 DFS 的拓扑排序算法**：
  1. 调用 `DFS(G)` 计算每个顶点的完成时间 $u.f$。
  2. 每当某个顶点被标为黑色（完成访问）时，将其插入到线性序列的**最前端 (Head)**。
  3. 最终输出的线性序列即为拓扑排序结果。
* **时间复杂度**：与 DFS 相同，为 **$O(V + E)$**。

### 6. 经典算法应用
* **应用一：地牢计数 (连通分量求解)**
  * **问题描述**：在 0-1 二维网格中，1 表示可以通过的通路，0 表示墙体。要求统计其中包含多少个封闭的房间（连通分量）。
  * **算法设计**：遍历整个矩阵。对于每个遇到的“1”且其未被访问（白色状态），以该点为起点启动一次 BFS 或 DFS。在单次遍历中，通过队列或递归将与其相邻的“1”全部标记为已访问（黑色状态）。每次成功启动一次完整的搜索，房间计数加 1。最后输出计数器的值。
* **应用二：上下级就餐判定 (括号化定理应用)**
  * **问题描述**：公司人员的上下级关系构成了有向图。当查询某两个人时，如果他们之间存在上下级关系（可达路径），则不能同桌吃饭。
  * **算法设计**：首先统计所有节点的入度，找出所有入度为 0 的根节点，避免漏掉独立分支。然后进行 DFS 遍历，为每个节点打上发现时间戳 $d$ 和完成时间戳 $f$。根据**后代区间嵌套定理**，节点 $v$ 是 $u$ 的后代当且仅当 $u.d < v.d < v.f < u.f$。对于每组查询 $(u, v)$，通过 $O(1)$ 时间对比两者的区间嵌套关系即可判定是否存在上下级冲突。
* **应用三：八数码问题 (状态空间 BFS 搜索)**
  * **问题描述**：在 $3 \times 3$ 的格子上移动数字 1~8 及空格 0，求从初始状态复原至目标状态（如 `123456780`）的最小步数。
  * **状态表示一维化**：将二维棋盘按行主序展开成长度为 9 的一维数组或字符串。下标 0~8 对应棋盘位置：
    ```text
    0 1 2
    3 4 5
    6 7 8
    ```
  * **状态转移规则**（设 `pos0` 为 0 所在的下标）：
    * **上移 (MoveUp)**：`pos0 >= 3`，与 `pos0 - 3` 交换。
    * **下移 (MoveDown)**：`pos0 <= 5`，与 `pos0 + 3` 交换。
    * **左移 (MoveLeft)**：`pos0 % 3 != 0`，与 `pos0 - 1` 交换。
    * **右移 (MoveRight)**：`pos0 % 3 != 2`，与 `pos0 + 1` 交换。
  * **广度优先搜索求解**：使用队列 `Queue` 保存状态。每次取出队头状态，按照上述四种移动规则生成新的棋盘状态。若是目标状态则直接返回步数。为了避免出现互逆操作导致无限循环，需要使用哈希表记录已访问的状态，防止死循环。

## 二、 重点术语与概念解析

* **有向图 (Directed Graph)**：图的边是有方向的，用有序对 $(u, v)$ 表示，指向性代表了单向关系。
* **无向图 (Undirected Graph)**：图的边是无方向的，用无序对 $(u, v)$ 表示，关联性代表双向等价关系。
* **连通图 (Connected Graph)**：无向图中，任意两个顶点之间都存在路径相连。
* **连通分量 (Connected Component)**：无向图中的极大连通子图。
* **简单路径 (Simple Path)**：顶点序列中所有顶点互不相同的路径。
* **简单环 (Simple Cycle)**：起点与终点相同，其余顶点互不相同的环。
* **邻接链表 (Adjacency List)**：图的一种链式存储结构，为每个顶点保存一个与其相邻的其他顶点的链表。
* **邻接矩阵 (Adjacency Matrix)**：图的一种矩阵存储结构，用二维数组的值表示两个顶点间是否存在边及其权重。
* **先驱节点 (Predecessor Node)**：搜索算法中，指向当前节点在搜索树中的父节点的指针（常用 `pi` 表示）。
* **时间戳 (Timestamp)**：DFS 中记录的节点生命周期界限。$d$ 代表发现时间，$f$ 代表完成时间。
* **树边 (Tree Edge)**：深度优先森林或广度优先搜索树中的边。
* **后向边 (Back Edge)**：DFS 树中将节点指向其祖先节点的边。
* **前向边 (Forward Edge)**：DFS 树中将节点指向其后代节点（且非直接父子）的非树边。
* **横向边 (Cross Edge)**：DFS 树中连接没有直接祖先后代关系的两个节点的边。
* **有向无环图 (DAG)**：一个不包含任何有向环的有向图，是拓扑排序的核心操作对象。
* **拓扑排序 (Topological Sort)**：将有向无环图中的所有顶点排成线性序列，满足所有边的前驱关系。

## 三、 工程经验与避坑指南

### 1. OJ 平台 C# 内存限制避坑
* **问题表现**：在“金木塔”等题目中，很多使用 C# 的同学遇到了内存超限（MLE）错误。
* **避坑指南**：C# 采用垃圾回收（GC）管理内存。在递归或循环中频繁创建小对象（如每次状态转移都 `new` 路径数组或结构体）会引发大量的临时堆内存分配，导致 GC 频繁运行且内存碎片化，从而极易超限。
* **优化策略**：
  1. 尽量将对象转化为基础一维数组（如 `int[]` 或 `string`）存储。
  2. 使用静态预分配的空间或重复使用同一内存缓冲区（例如类成员变量或静态临时数组），避免在搜索树的深度递归中频繁 `new` 空间。

### 2. 大数乘加过程防溢出
* **问题表现**：在进行“矩阵链乘”等动态规划计算中，由于乘积和累加结果增长极快，导致结果在中间计算过程中就已经超出了 $64$ 位有符号整型 `long` 的表示范围，溢出变成负数，最终导致计算错误。
* **避坑指南**：**计算过程必须全程取模**。不能仅在最后输出结果时才取模。在动态规划或搜索的累加和累乘循环中，对于每一步中间结果都要立即进行取模操作（如 `(ans + val) % MOD` 和 `(ans * val) % MOD`），确保其始终保持在 `long` 的安全范围内。

### 3. BFS/DFS 状态标记的工程简化
* **经验指南**：在理论学习中，BFS 和 DFS 使用“白、灰、黑”三种颜色来维护节点生命周期。但在工程实践中：
  1. 不需要专门定义一个 Color 枚举。
  2. 可以通过一个布尔型数组 `visited[]` 或 `HashSet` 来替代颜色：
     * **白色**：`visited[u] == false`。
     * **灰色/黑色**：`visited[u] == true`。对于 BFS，只要节点入队就设为已访问；对于 DFS，由于要用到括号化时间戳，可用节点是否已被发现来代替灰色。
  3. 对于需要记录路径长度的 BFS，初始化距离数组 `dist[]` 为 -1 即可，若 `dist[v] == -1` 说明是白色节点，入队时更新 `dist[v] = dist[u] + 1`，不需要额外的状态判断。

### 4. 拓扑排序与 DFS 起点的确定
* **问题表现**：部分同学默认从顶点 `1` 开始对有向图进行 DFS，导致无法获得正确的拓扑排序结果（如任务依赖关系未完全覆盖）。
* **避坑指南**：
  1. 在有向图结构中，顶点 `1` 并不一定是拓扑结构中的最前置任务（甚至可能存在多个入度为 0 的任务起点）。
  2. 应当在建图时维护每个节点的**入度 (In-degree)**。
  3. **起点的正确选择**：遍历所有顶点，首先对所有**入度为 0** 的节点启动 DFS 搜索。如果发现搜索完之后仍然存在未访问的白色节点，需要外层循环继续搜索这些白色节点，确保所有孤立依赖子图或多源头有向图的所有节点都被正确排序。

### 5. 八数码问题的关键实现技术
* **判重以防死循环**：八数码状态空间很大，搜索过程中必须对状态进行**判重**（如空格“向左”移动后又“向右”移动，会产生重复的多余状态）。如果没有判重机制，BFS 队列将呈指数级膨胀，程序会在极短时间内因内存超限（OOM）崩溃或发生死循环。
* **一维化数组下标的越界边界判定**：一维化存储后，边界判定要极其精确。例如：
  * 向左移时，必须保证 `pos0 % 3 != 0`（不能在最左列）。
  * 向右移时，必须保证 `pos0 % 3 != 2`（不能在最右列）。
* **面向对象封装设计**：在编写该题目时，建议将棋盘状态封装为一个完整的 C# 类，通过方法返回值标志移动是否合法，使得搜索主逻辑结构清晰、易于调试。完整标准实现结构如下：

```csharp
using System;
using System.Collections.Generic;
using System.Text;

public class BoardState
{
    // 用一维数组保存 3x3 的棋盘状态，0 代表空格
    public int[] Board { get; private set; }
    public int Pos0 { get; private set; } // 存储 0 所在的数组索引位置
    
    // 构造函数：解析初始化字符串
    public BoardState(string stateStr)
    {
        Board = new int[9];
        for (int i = 0; i < 9; i++)
        {
            Board[i] = stateStr[i] - '0';
            if (Board[i] == 0)
            {
                Pos0 = i;
            }
        }
    }

    // 拷贝构造函数：在 BFS 转移状态时进行深拷贝
    public BoardState(BoardState other)
    {
        Board = (int[])other.Board.Clone();
        Pos0 = other.Pos0;
    }

    // 交换数组中两个位置的元素
    private void Swap(int i, int j)
    {
        int temp = Board[i];
        Board[i] = Board[j];
        Board[j] = temp;
    }

    // 向上移动：空格向上移动（与上面一行的数字交换）
    public bool MoveUp()
    {
        if (Pos0 < 3) return false; // 空格在最顶行，无法上移
        Swap(Pos0, Pos0 - 3);
        Pos0 = Pos0 - 3;
        return true;
    }

    // 向下移动
    public bool MoveDown()
    {
        if (Pos0 > 5) return false; // 空格在最底行，无法下移
        Swap(Pos0, Pos0 + 3);
        Pos0 = Pos0 + 3;
        return true;
    }

    // 向左移动
    public bool MoveLeft()
    {
        if (Pos0 % 3 == 0) return false; // 空格在最左列，无法左移
        Swap(Pos0, Pos0 - 1);
        Pos0 = Pos0 - 1;
        return true;
    }

    // 向右移动
    public bool MoveRight()
    {
        if (Pos0 % 3 == 2) return false; // 空格在最右列，无法右移
        Swap(Pos0, Pos0 + 1);
        Pos0 = Pos0 + 1;
        return true;
    }

    // 将当前棋盘数组转换为状态字符串，供 BFS 进行 visited 判重
    public string ToStateString()
    {
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < 9; i++)
        {
            sb.Append(Board[i]);
        }
        return sb.ToString();
    }
}
```
