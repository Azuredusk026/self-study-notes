# 算法进阶专题课：二叉搜索树、最长公共子序列、A* 寻路算法与 KMP 算法

## 一、 核心知识点与原理剖析

### 1. 二叉搜索树 (Binary Search Tree, BST)

#### (1) 基本概念与存储结构
二叉搜索树（BST）是一种非常经典的动态数据结构，也是理解红黑树、Splay 树等平衡树的基础。与堆（Heap）类似，二叉搜索树在逻辑上也是一种树形结构，但它们在实现与内部规则上有着本质差别：
* **存储方式**：堆在逻辑上是完全二叉树，在物理上通常存储于**一维数组**中（利用下标公式计算父子节点关系）；而二叉搜索树是通过**链式节点对象**来实现的。
* **节点结构**：每个节点在内存中都是一个独立的对象（包含指针与数据的集合），包含以下属性：
  * `key`：节点所存储的关键字（用于排序与检索的数值/字符等）。
  * `left`：指向该节点左孩子的指针。
  * `right`：指向该节点右孩子的指针。
  * `p`：指向该节点双亲节点（父节点）的指针。
  * **卫星数据 (Satellite Data)**：除主键键值之外，伴随节点一同存储的业务应用数据。
* **二叉搜索性质**：
  设 $x$ 是二叉搜索树中的任意一个节点。
  * 若 $y$ 是 $x$ 的**左子树**中的一个节点，则必满足：$y.\text{key} \le x.\text{key}$。
  * 若 $y$ 是 $x$ 的**右子树**中的一个节点，则必满足：$y.\text{key} \ge x.\text{key}$。
  也就是说，对于树中的任意节点，其左子树中所有节点的值都小于或等于该节点，右子树中所有节点的值都大于或等于该节点。这一性质使整棵树以根节点及各个子树根节点为界，实现了有序的划分。

#### (2) 树的遍历方式
通过不同的遍历路径，我们可以从二叉搜索树中输出不同特征的序列：
* **中序遍历 (In-order Traversal)**：
  * 遍历顺序：先递归访问左子树 $\rightarrow$ 再输出当前根节点 $\rightarrow$ 最后递归访问右子树。
  * 特点：由于二叉搜索树左小右大的性质，中序遍历能以**非降序（升序）**的顺序输出所有的关键字。这也意味着如果能构建出一棵二叉搜索树，就可以实现对一段序列的排序。
* **先序遍历 (Pre-order Traversal)**：
  * 遍历顺序：先输出当前根节点 $\rightarrow$ 递归访问左子树 $\rightarrow$ 递归访问右子树。
* **后序遍历 (Post-order Traversal)**：
  * 遍历顺序：递归访问左子树 $\rightarrow$ 递归访问右子树 $\rightarrow$ 最后输出当前根节点。
  * 注意：在二叉搜索树中，先序与后序遍历得到的输出序列是无序的。

#### (3) 核心检索与查询操作
二叉搜索树的所有基础查询操作，其时间复杂度均与树的高度 $h$ 相关，即为 $O(h)$。
* **查找操作 (Search)**：
  类似于二分搜索。从根节点出发，如果目标值 $k$ 比当前节点的值小，则向左子树递归或迭代；如果比当前节点大，则向右子树搜索。
  我们可以写成非递归（迭代）的形式，从而省去函数调用栈开销，提高实际运行效率：
  ```python
  def iterative_tree_search(x, k):
      while x is not None and k != x.key:
          if k < x.key:
              x = x.left
          else:
              x = x.right
      return x
  ```
* **寻找极值 (Minimum & Maximum)**：
  * **最小值 (Minimum)**：从指定的子树根节点开始，顺着左孩子指针一路向左摸索，直到遇到没有左孩子的节点。
    ```python
    def tree_minimum(x):
        while x.left is not None:
            x = x.left
        return x
    ```
  * **最大值 (Maximum)**：同理，顺着右孩子指针一路向右移动，直至右孩子为空的节点。
    ```python
    def tree_maximum(x):
        while x.right is not None:
            x = x.right
        return x
    ```
* **寻找前驱与后继 (Predecessor & Successor)**：
  查询前驱和后继操作接受的参数是**节点对象**而非具体的键值。
  * **后继 (Successor)**：指中序遍历下，比当前节点值大的最小节点。
    * **情况1**：节点 $x$ 有右子树。后继即为右子树中的最小值，即 `tree_minimum(x.right)`。
    * **情况2**：节点 $x$ 没有右子树。此时需要沿着父指针往上找。其后继是 $x$ 向上追溯过程中的某个祖先，这个祖先满足：它是其自身父节点的左孩子。
    ```python
    def tree_successor(x):
        if x.right is not None:
            return tree_minimum(x.right)
        y = x.p
        while y is not None and x == y.right:
            x = y
            y = y.p
        return y
    ```
  * **前驱 (Predecessor)**：指中序遍历下，比当前节点值小的最大节点（与后继逻辑完全对称）。
    * **情况1**：节点 $x$ 有左子树。前驱为其左子树中的最大值，即 `tree_maximum(x.left)`。
    * **情况2**：节点 $x$ 没有左子树。向上追溯找到第一个“作为右孩子”的祖先节点的父亲。
    ```python
    def tree_predecessor(x):
        if x.left is not None:
            return tree_maximum(x.left)
        y = x.p
        while y is not None and x == y.left:
            x = y
            y = y.p
        return y
    ```

#### (4) 插入操作 (Insert)
插入一个新节点 $z$ 时，需要利用两个指针向下搜寻：指针 $x$ 用来寻找合适的空插槽，指针 $y$ 作为 $x$ 的父节点指针用于追踪插入的双亲位置。
```python
def tree_insert(T, z):
    y = None
    x = T.root
    while x is not None:
        y = x
        if z.key < x.key:
            x = x.left
        else:
            x = x.right
    z.p = y
    if y is None:
        T.root = z  # 树本为空，直接作为根节点
    elif z.key < y.key:
        y.left = z
    else:
        y.right = z
```

#### (5) 删除操作 (Delete)
删除操作是二叉搜索树中最难维护的部分。它需要处理子树和父指针的替换。
首先需要实现一个基础替换辅助函数 `Transplant(T, u, v)`。它的作用是：用以 $v$ 为根的子树替换以 $u$ 为根的子树，同时重置 $u.\text{p}$ 的指向。
```python
def transplant(T, u, v):
    if u.p is None:
        T.root = v
    elif u == u.p.left:
        u.p.left = v
    else:
        u.p.right = v
    if v is not None:
        v.p = u.p
```
在执行 `tree_delete(T, z)` 时，从逻辑上可以归纳为四种具体的分支处理：
* **情况 1：没有左孩子**。如果 $z.\text{left} == \text{NIL}$（此时不论右孩子是否为空），直接调用 `transplant(T, z, z.right)`，让右子树整体替换自己。
* **情况 2：只有左孩子**。如果 $z$ 仅有一个左孩子（右孩子 $z.\text{right} == \text{NIL}$），直接调用 `transplant(T, z, z.left)` 替换。
* **情况 3：双孩子俱全**。我们需要找到其后继 $y = \text{tree\_minimum}(z.\text{right})$。因为 $z$ 左右子树都不为空，后继必定在右子树中且没有左孩子。对此分为两种情况：
  * **子情况 A**：$y$ 是 $z$ 的直接右孩子。此时我们直接将 $y$ 替换 $z$，并把 $z$ 的左孩子挂载到 $y$ 的左孩子上，更新父指针。
  * **子情况 B**：$y$ 在右子树深处但不是 $z$ 的直接右孩子。
    1. 用 $y$ 的右孩子替换 $y$ 所在的位置。
    2. 将 $z$ 的右孩子挂载为 $y$ 的右孩子，重新维护父指针。
    3. 用 $y$ 整体替换原有的 $z$。
    4. 将 $z$ 的左孩子接在 $y$ 的左孩子指针上，重新维护父指针。
```python
def tree_delete(T, z):
    if z.left is None:
        transplant(T, z, z.right)
    elif z.right is None:
        transplant(T, z, z.left)
    else:
        y = tree_minimum(z.right)
        if y.p != z:
            transplant(T, y, y.right)
            y.right = z.right
            y.right.p = y
        transplant(T, z, y)
        y.left = z.left
        y.left.p = y
```

### 2. 最长公共子序列 (Longest Common Subsequence, LCS)

#### (1) 基本概念
给定两个序列 $X$ 和 $Y$，如果序列 $Z$ 既是 $X$ 的子序列又是 $Y$ 的子序列，且在 $X$ 和 $Y$ 中**不需要连续出现**，则称 $Z$ 为公共子序列。最长公共子序列（LCS）问题就是要找出所有公共子序列中长度最长的那一个。

#### (2) 最优子结构定理
令 $X = \langle x_1, x_2, \dots, x_m \rangle$ 和 $Y = \langle y_1, y_2, \dots, y_n \rangle$ 为两个序列，$Z = \langle z_1, z_2, \dots, z_k \rangle$ 是它们的任意一个 LCS：
1. 若 $x_m = y_n$，则 $z_k = x_m = y_n$，且 $Z_{k-1}$ 是前缀 $X_{m-1}$ 与 $Y_{n-1}$ 的一个 LCS。
2. 若 $x_m \ne y_n$ 且 $z_k \ne x_m$，则 $Z$ 是 $X_{m-1}$ 与 $Y$ 的一个 LCS。
3. 若 $x_m \ne y_n$ 且 $z_k \ne y_n$，则 $Z$ 是 $X$ 与 $Y_{n-1}$ 的一个 LCS。

#### (3) 状态转移方程与重构
设 $c[i, j]$ 表示前缀 $X_i$ 和 $Y_j$ 对应 LCS 的长度，则有：
$$c[i, j] = \begin{cases} 
  0 & \text{若 } i=0 \text{ 或 } j=0 \\
  c[i-1, j-1] + 1 & \text{若 } i,j > 0 \text{ 且 } x_i = y_j \\
  \max(c[i, j-1], c[i-1, j]) & \text{若 } i,j > 0 \text{ 且 } x_i \ne y_j
\end{cases}$$

引入辅助数组 $b[i, j]$ 用于追溯转移来源：
* `↖`：代表匹配成功，由左上方转移而来。
* `↑`：代表由上方 $c[i-1, j]$ 转移而来。
* `←`：代表由左方 $c[i, j-1]$ 转移而来。

通过双层循环枚举 $i$ 与 $j$ 计算出全表后，我们从右下角 $(m, n)$ 开始溯源：
* 如果 $b[i, j] ==$ `↖`，记录匹配字符，向左上方移动。
* 如果 $b[i, j] ==$ `↑`，直接向正上方移动。
* 如果 $b[i, j] ==$ `←`，直接向正左方移动。
* 当碰到边界（$i=0$ 或 $j=0$）时，回溯完毕。将记录的字符进行反转，即为所得的最长公共子序列。

### 3. A* 寻路算法 (A* Search Algorithm)

#### (1) 网格寻路与启发式引出
在二维网格地图中（0可通过，1为障碍物），为了从起点走到终点，如果使用广度优先搜索 (BFS) 或 Dijkstra 算法，它们就像“一滩水”一样层层向外做同心圆式的无方向蔓延，会向目标的反方向扩展出大量无用节点。
为解决这种算力浪费，我们引入**启发式（Heuristic）估值**，让寻路带有清晰的方向感。

#### (2) 估值函数与设计
A* 算法维护当前待扩展集（Open List），每次提取 $F(n)$ 值最小的节点进行扩展。其估值公式为：
$$F(n) = G(n) + H(n)$$
* **$G(n)$**：从起点到当前节点 $n$ 的实际物理代价（已经走过的路程）。
* **$H(n)$**：当前节点 $n$ 到终点的估计代价（估计还需走多远）。

#### (3) 估值函数的可容性约束 (Admissibility)
设 $H^*(n)$ 为节点 $n$ 到终点的**实际真实最短路径长度**。估值函数 $H(n)$ 必须严格满足：
$$H(n) \le H^*(n)$$
如果违反此定理（即高估了未来的代价），算法可能会出于对某路径的悲观判断而避开了最优解，导致失效。
* **曼哈顿距离**：在网格寻路中，最常用的估值为曼哈顿距离 $H(n) = |x_1 - x_2| + |y_1 - y_2|$。因为实际地图有障碍物限制，真实距离一定大于或等于两点的曼哈顿距离，所以必有 $H(n) \le H^*(n)$，能确保寻路结果最优。

#### (4) 8数码问题中的估值应用
在 8 数码问题中，我们要将乱序棋盘还原为目标状态 $\langle 1,2,3,4,5,6,7,8,0 \rangle$（0代表空格）。
* **$G(n)$**：已移动的实际步数。
* **$H(n)$ 的合理设计**：可选择“当前状态中与目标状态相比，**数字不同的格子个数（注意不计空格0）**”。
  * **避坑分析**：如果在不同的格子中，把空格0也算进去，会出现单次移动 1 步使估值 $H(n)$ 减少 2 的特殊情况，导致违反 $H(n) \le H^*(n)$，算法会失效。而在排除 0 后，因为每次物理移动最多只能有一个数字归位（即使混乱格子数减少 1），因此实际步数一定大于或等于混乱格子数，符合悲观估计定理，可确保获取最少移动步骤的最优解。

### 4. KMP 算法 (Knuth-Morris-Pratt Algorithm)

#### (1) KMP 算法核心思想
经典的单模式串匹配问题是在大字符串 $T$ 中寻找模式串 $P$。暴力算法中一旦发生失配，主串指针就需要回溯，效率低下。
KMP 的核心思想是：**主串指针在失配时绝对不回溯，而是通过分析模式串本身的重复特征（已知信息），让模式串一次性向右滑动到最近的一个“最可能完成匹配的位置”，直接对齐先前匹配过的前缀，继续向后比对。**

#### (2) 失配数组（Next 数组/F 数组）
KMP 算法的滑行距离，依靠对模式串进行预处理来得出。
* **定义**：我们定义一个失效函数数组 $F[i]$。其值表示在模式串前缀 $P[1..i]$ 中，**最长公共真前缀与真后缀的相同匹配长度**。
* **物理意义**：当我们尝试在模式串的第 $i+1$ 位进行匹配但遭遇失败时，我们已知前 $i$ 位均已匹配。如果 $F[i] = k$，说明匹配部分头部的前 $k$ 个字符与尾部的后 $k$ 个字符是完全一致的。此时直接将主串当前对齐的后缀移换成模式串的前缀，即直接用模式串 of $k+1$ 个字符与主串当前字符对比。

#### (3) 递推构建 $F$ 数组算法
构建 $F$ 数组本质是让模式串“自己匹配自己”。假设已知 $F[i-1]$：
1. **成功扩展**：若模式串第 $i$ 个字符与第 $F[i-1] + 1$ 个字符相同，则可直接在原有的公共前后缀后面接上当前相同字符，从而 $F[i] = F[i-1] + 1$。
2. **回溯匹配**：若字符不同，说明不能直接累加。我们令 $k = F[i-1]$。进入循环回溯：
   * 检查第 $k$ 位前缀的下一位 $P[k+1]$ 是否等于 $P[i]$。
   * 若不相等，则使 $k = F[k]$ 沿失效链往传统回退，直至 $P[k+1] == P[i]$ 或 $k=0$。
   * 若在某一处相等，则 $F[i] = k + 1$；若直到 $k=0$ 依然无法匹配，则 $F[i] = 0$。

## 二、 重点术语与概念解析

* **卫星数据 (Satellite Data)**：在带键值的数据结构中，除了主键标识（key）之外，随节点一同保存的其他具有具体业务价值的实际载荷数据。
* **中序遍历排序特性**：由于二叉搜索树的底层性质约束，其中序遍历（左 $\rightarrow$ 根 $\rightarrow$ 右）必定能获得按照键值大小进行升序（非降序）输出的有序序列。
* **后继节点 (Successor)**：指中序遍历中，排在某节点之后的紧邻节点。有右子树时为右子树的最左（最小）节点；无右子树时为向上溯源遇到的第一个“作为其双亲左孩子”的祖先。
* **前驱节点 (Predecessor)**：指中序遍历中，排在某节点之前的紧邻节点（与后继相反）。
* **可容性条件 (Admissibility)**：A* 算法估值函数 $H(n) \le H^*(n)$。它是确保启发式寻路算法一定能搜索到最优解的数学前提。
* **真前缀与真后缀 (True Prefix & True Suffix)**：子串的前缀与后缀，但均不包含该子串自身。
* **LCS 最优子结构**：两个序列的最长公共子序列一定包含其对应前缀子问题的最长公共子序列。这保证了可以使用自底向上的动态规划算法来解决问题。

## 三、 工程经验与避坑指南

#### 1. 裸 BST 退化与高度维护
* **工程隐患**：裸的二叉搜索树在遇到有序的数据输入（如递增或递减）时，会退化为一条单向链表，导致检索、插入、删除性能直接跌落至 $O(n)$ 级别。
* **开发避坑**：
  * 在离线处理数据时，可以通过提前对待插入数组进行洗牌打乱（Shuffle），随机构建二叉搜索树。数学期望上，随机构建的树高为 $O(\log n)$，能保证 $O(n \log n)$ 的构建开销。
  * 在频繁执行动态插入和删除的复杂工业级场景中，必须使用具备自动平衡性质的变种树（如**红黑树**、**AVL 树**或**伸展树 Splay**）来动态控制树高。

#### 2. A* 算法的优先队列（堆）优化
* **性能瓶颈**：在 A* 算法主循环中，每一次搜索扩展都需要从 Open List 中寻找一个 $F(n)$ 值最小的节点。若简单使用顺序遍历的动态数组（vector/list），每次检索开销为 $O(N)$，面对宏大地图时寻路将产生严重的计算卡顿。
* **开发避坑**：在写代码实践时，**Open List 必须结合优先队列（最小堆）来管理**。这能确保提取最小值与新插入操作开销压缩至 $O(\log N)$，提升实际工程运行性能。

#### 3. 裸题与竞赛包装题的转换
* **学习建议**：课堂上实现的各种经典算法（如 BST 插入删除、LCS、KMP 数组计算）基本都是纯粹的算法原型（裸题）。但真正的面试和算法竞赛（蓝桥杯、ICPC 等）中，绝大部分题目都裹着一层厚厚的应用背景（如将最小波动值计算转化为 BST 前驱后继检索）。一定要通过大量的刷题来提升转化思维，看破其底层的算法模型。

#### 4. KMP 的工业地位定位
* **开发避坑**：尽管 KMP 算法是各大厂面试和算法竞赛中的高频经典题，但在实际的后台开发或日志检索业务中，极少需要开发者自己手写 KMP 模块。
* **原因解析**：大部分现代高级语言内置的查找函数（如 `string.find`）都是在汇编层面高度优化过的，且底层广泛采用了在实际长文本检索下效率比 KMP 更高的 **BM (Boyer-Moore) 算法** 或 **Sunday 算法**。KMP 作为思维训练、理解递推回溯机制是一个好教材，但不应盲目用于所有实际的开发检索环境。
