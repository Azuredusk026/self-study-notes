## 1. 计算机存储器金字塔 (The Memory Hierarchy)

存储器的设计遵循一个铁律：**越靠近 CPU 的存储设备，速度越快，容量越小，造价越贵。**

### 1.1 金字塔结构（从下到上）

* **断电不丢失数据（非易失性存储）：**
* **L6 远程设备**：云盘、局域网共享文件夹等（速度最慢，本课不深入探讨）。
* **L5 硬盘 (Disk)**：机械硬盘 (HDD) 或固态硬盘 (SSD)。（容量大，速度较慢）。


* **断电即清空数据（易失性存储，L4 ~ L0）：**
* **L4 主存 / 内存 (DRAM - 动态随机存取存储器)**：平常说的内存条（如 DDR4/DDR5）。以字节（Byte）为单位进行寻址。
* **L3 ~ L1 高速缓存 (Cache / SRAM - 静态随机存取存储器)**：位于内存和 CPU 之间。速度极快，造价极高。分为三层（L3最大最慢，L1最小最快）。
* **L0 寄存器 (Registers)**：CPU 内部最核心、最快的存储单元。数量极少（只有几十个），直接供 CPU 的运算逻辑单元 (ALU) 读写计算。

## 2. 核心机制：高速缓存 (Cache) 与命中 (Hit/Miss)

由于 CPU 的运算速度远超内存的读取速度，如果没有 Cache，CPU 大部分时间都在“干等”内存传送数据（通过总线交互效率极低）。Cache 就是为了解决这个“速度差”而诞生的。

### 2.1 逐层访问逻辑

1. CPU 运算时需要数据 `A`。
2. 首先找**寄存器**；没有，找 **L1 Cache**；没有，找 **L2** -> **L3** -> **主存**。
3. 在主存找到数据 `A` 后，并不是只把 `A` 拿回去，而是把 `A` 及其**周围的一块数据**打包，一层层往上搬运（替换掉 Cache 中旧的数据），最终送达 CPU。

### 2.2 命中 (Hit) 与 不命中 (Miss)

* **Cache Hit (缓存命中)**：CPU 在靠近自己的 Cache 层（如 L1）直接找到了需要的数据。**（性能最高，开销极小）**
* **Cache Miss (缓存不命中)**：Cache 中没有所需数据，被迫去更底层的 Cache 或主存拿数据，并将新数据替换进 Cache。**（性能杀手，造成严重延迟）**
* **核心结论**：**衡量一个程序性能好坏的关键指标之一，就是 Cache 命中率。**

### 2.3 Cache 的底层结构与寻址（扩展概念）

Cache 将内存地址拆分为三部分进行匹配寻址：

1. **Set Index (组索引)**：决定数据该放在 Cache 的哪一组。
2. **Tag (标记)**：用于比对当前组里的数据是不是我们要找的具体目标。
3. **Block Offset (块偏移)**：定位具体是数据块里的第几个字节。
*(还有一位 `Valid` 有效位，标识当前 Cache 块的数据是否有效可用。)*

### 2.4 Cache 的三种映射方式（硬件概念）

1. **直接映射 (Direct Mapping)**：一个地址只能固定映射到 Cache 的唯一一个位置。硬件成本低、比对快，但极容易发生冲突（多个数据争抢同一个位置，导致疯狂 Miss）。
2. **全相联映射 (Fully Associative)**：数据可以放在 Cache 的任意位置。无冲突，但每次寻找都要遍历所有 Tag（硬件电门成本极高）。
3. **组相联映射 (Set Associative)**：折中方案。分成若干组，每组包含几个位置。兼顾了硬件成本和冲突率（现代 CPU 常用方案）。

### 2.5 缓存替换策略与写策略

* **替换策略（当 Cache 满了，新数据该踢走谁？）**：
* **LRU (最近最少使用)**：踢走最长时间没被访问过的数据。（最常用且合理）
* **LFU (最不常用)**：踢走历史上访问总次数最少的数据。
* **FIFO (先进先出)**：踢走最早进来的数据。（效果不稳定）
* **Random (随机)**：随便踢走一个。（不靠谱）


* **写策略（Cache 数据被修改后，什么时候同步回主存？）**：
* **Write-Back (写回)**：平时只改 Cache，只有当这个 Cache 块被替换踢走时，才把新数据写回主存。（性能好，但断电易丢失数据不一致）。
* **Write-Through (写穿透)**：只要 Cache 改了，立马同步写穿透到主存。（数据绝对安全一致，但性能损耗大）。

## 3. 寄存器 (Registers) 简介

CPU 的“私人办公桌”，直接参与计算。

* **通用寄存器**：如 64 位系统中的 `RAX`（常存函数返回值）、`RDI`, `RSI`（常存函数参数）等。程序员（通过汇编）可以直接读写。
* **特殊寄存器**：
* `RIP` (程序计数器)：存着下一条要执行的指令地址。
* `RSP` (栈顶指针)：指向当前函数调用栈的栈顶。
* 状态/标志寄存器：存储条件判断（如大于、小于、等于零）的中间状态，控制 `if/else` 的跳转。这些大多由 CPU 内部控制，程序员不可随意修改。

## 4. 终极实战：编写“缓存友好”的代码 (Cache-Friendly Code)

理解了存储结构，我们该如何优化 C++ 代码的性能？核心在于**利用“局部性原理 (Principle of Locality)”**。

### 4.1 空间局部性 (Spatial Locality)

* **原理**：如果一个内存位置被访问了，那么它**附近**的内存位置很快也会被访问。因为 Cache 搬运数据是一块一块搬的。
* **实战策略**：
* **多用数组 (`Vector`/`Array`)，少用链表 (`List`)**。数组的内存是连续分布的，访问 `a[0]` 时，`a[1]` 到 `a[n]` 可能已经被顺手打包进 Cache 了，后续访问全是 Cache Hit！而链表的节点在内存中是散落的，极易造成 Cache Miss。
* **数据对齐 (Data Alignment)**：我们在第三节课讲过，合理的结构体对齐（`struct`）能确保数据完美贴合 Cache Block 的大小，减少跨块读取。



### 4.2 时间局部性 (Temporal Locality)

* **原理**：如果一个数据刚刚被访问过，那么它在**不久的将来**大概率会被再次访问。
* **实战策略**：
* **优化循环变量**：像 `for(int i=0; ...)` 中的 `i` 就会一直驻留在寄存器或 L1 Cache 中，速度极快。
* **数据重用**：在同一个作用域/循环内，尽量重复利用同一个变量，而不是在循环体内部疯狂 `new` 出新变量重新申请分散的内存地址。



### 4.3 整体架构优化

* 保持函数短小精悍，不要写长篇大论的巨型函数，这样能确保当前执行的代码指令也能很好地呆在指令 Cache 中。
