# C++ 面向对象进阶与泛型编程 (第四节课笔记)

## 📌 课前复盘：Vector 自定义类实现细节

* **运算符重载的核心**：
* 加减法（`+`, `-`）：必须传入另一个 Vector 作为参数，且返回值应当是新的对象本身（通常返回 `*this` 或新实例），以支持链式操作。
* 输入输出流（`<<`, `>>`）：重载输出流时，强烈建议传入 `const Vector&`，上好“安全锁”，保证输出流绝对不会意外修改向量的内部数据。


* **`this` 指针的妙用**：
* `this` 是一个隐藏的指针，永远指向**当前对象实例本身**。
* **解决命名冲突**：当传入参数名与类成员变量同名时（例如参数叫 `x`，成员变量也叫 `x`），使用 `this->x = x;` 可以明确告知编译器：把外面的参数赋给实例自己的成员变量。不仅保险，而且在现代 IDE 中会被高亮，代码可读性极强。

## 1. 面向对象编程 (OOP) 核心三大特性

### 特性一：封装 (Encapsulation)

* **概念**：将数据（成员变量）和行为（成员函数）打包成一个单独的实体（类）。
* **核心目的：隐藏细节**。就像快递纸箱，对外只暴露快递单（`.h` 头文件，标明有哪些接口可以用），隐藏箱子里的具体物品构造（`.cpp` 源文件的具体实现）。
* **重构**：好的面向对象设计是逐步演进的（如植物大战僵尸：从简单的植物/僵尸对象，到逐步拆分出子弹、阳光、状态机等新对象）。

### 特性二：继承 (Inheritance / 派生)

* **概念**：子类（派生类）自动获取父类（基类）的特征和行为，极大减少重复代码。
* **C++ 特有的继承控制边界（⚠️ 与 C#/Java 的重大区别）**：
* **无论何种继承方式，父类的 `private` 成员，子类绝对无法直接访问！**（在 C# 中子类可能可以拿到，但在 C++ 中严格隔离）。
* **公有继承 (`public`)**：最常用。父类的 `public` 和 `protected` 在子类中保持原样。
* **私有继承 (`private`)**：父类的 `public` 和 `protected` 到子类中全部变成 `private`（会阻断孙子类的继续访问）。
* **保护继承 (`protected`)**：父类的 `public` 变成 `protected`。


* **多重继承**：C++ 支持一个子类同时继承多个父类，用逗号隔开即可（例如 `class Professor : public Person, public Teacher`）。

### 特性三：多态 (Polymorphism)

* **概念**：允许使用父类的指针来调用子类重写的方法。同一种行为，根据具体的子类形态，表现出不同的执行结果。
* **虚函数 (`virtual`) 与 重写 (`override`)**：
* 父类中使用 `virtual` 关键字声明函数。
* 子类中使用 `override` 关键字（写在函数括号后）进行重写。
* **动态绑定**：程序运行到这一行时，才会去“虚函数表”里查，确定当前指针到底指的是哪个子类，从而执行正确的实现代码。


* **虚析构函数**：**极度重要！** 当使用父类指针管理子类对象时，如果父类的析构函数不是 `virtual` 的，`delete` 时只会调用父类的析构，导致子类自己申请的内存（如指针）无法释放。
* **纯虚函数与接口 (Interface)**：
* **纯虚函数**：没有实现，直接声明为 `= 0`。例如 `virtual void getArea() = 0;`。逼迫所有继承它的子类必须自己写出具体实现。
* **抽象类**：只要包含至少一个纯虚函数的类，就是抽象类。**抽象类绝对不能被直接实例化。**
* **接口类**：如果一个类里**只有**纯虚函数，没有任何普通变量和普通函数，它在 C++ 中就等价于其他语言中的接口（Interface）。

## 2. 泛型编程：模板 (Template)

当同一个逻辑需要应用于 `int`、`float`、甚至自定义类型时，为每种类型写一遍重载极其浪费。模板是 C++ 解决代码复用和通用性的究极武器。

### 函数模板 (Function Template)

* **定义语法**：
```cpp
template <typename T>
void swap(T& a, T& b) {
    T temp = a;
    a = b;
    b = temp;
}

```


* `T` 就是一个类型占位符。调用时，编译器会根据传入的参数自动推导 `T` 是什么，并暗中生成一个对应的真实函数去执行。
* **⚠️ 陷阱**：模板内部使用的操作符（如 `+`），传入的类型 `T` 必须支持该操作。如果传入自定义的 `MyClass`，而该类没有重载 `+` 运算符，编译会直接报错。

### 类模板 (Class Template)

* 最常用于自定义**数据容器**（如我们自己写的 `Stack<T>` 或 `List<T>`）。
* 让类内部的成员变量类型和函数返回值都使用 `T` 替代。

### 模板特化 (Template Specialization)

* **应用场景**：模板虽然通用，但总有“例外”。当传入某种特殊类型时（例如 `char`），通用的算法可能没有意义或会出错，就需要为其单独写一份特殊实现。
* **语法**：声明 `template <>`，并在类名或函数名后显式标出特化的类型。
* **讲师建议**：对于自定义类产生的运算歧义，更好的做法是在类内部老老实实重载运算符（如重载 `MyClass` 的 `operator+`），而不是动辄去特化通用的模板函数，这样代码更清晰且不易产生隐性 Bug。

## 3. 课堂实战与课后作业

### 课堂实战：几何图形的抽象与多态

* **要求**：构建抽象基类 `Shape`（包含计算面积和周长的纯虚函数），并派生出 `Triangle`、`Square`、`Circle`。
* **核心逻辑**：
* 基类只做接口定义，具体的数据（边长、半径）全部下放给子类自己管理，因为不同图形确定自身形态所需的数据结构完全不同。
* 数学公式：三角形面积使用海伦公式 $\sqrt{p(p-a)(p-b)(p-c)}$ ；求圆面积时使用了之前讲过的 `constexpr double PI = 3.1415926;` 在编译期预处理，提升性能。



### 课后作业清单

1. **函数模板实战**：写一个**通用选择排序（Selection Sort）算法**，能对任意类型的数组进行排序（假设传入的类型都已经重载了 `<` 和 `>`）。
2. **类模板实战**：写一个**队列（Queue）数据结构**（遵循 FIFO 先进先出规则），实现入队、出队、判空等操作，并支持任意数据类型 `T`。
