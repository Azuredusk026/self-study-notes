## 1. 类 (Class) 与 结构体 (Struct) 的本质

在 C++ 中，`class` 和 `struct` 的底层逻辑与功能几乎**完全一致**（都可以拥有成员变量、成员函数、构造/析构函数等）。

* **唯一区别（默认访问权限）**：
* **`class`**：默认访问权限是 **`private`**（私有）。
* **`struct`**：默认访问权限是 **`public`**（公有）。


* **访问权限修饰符**：
* `public:`：公有，外部可随意访问。
* `private:`：私有，仅类内部和“友元”可访问。
* `protected:`：受保护的，类内部及其**派生类（子类）**可访问（用于继承）。


* **讲师建议**：为了避免混乱，无论使用哪种，都**强烈建议手动写明** `public:` 或 `private:`。在现代开发中，高层逻辑封装多用 `class`，而网络发包、GPU 渲染数据传输等需要严格控制内存布局的底层数据，多用 `struct`。

## 2. 规范化工程：头文件与源文件的分离

为了代码可读性、规避编译顺序报错，强烈建议将类的定义与实现分离。

* **`.h` 头文件（只做声明）**：
* 使用 `#pragma once` 防止重复编译。
* 在此处声明类名、成员变量、函数签名（无具体实现逻辑）。
* **优势**：充当“说明书”，别人调用你的类时，只需看头文件和上面的 XML 注释（如 `/// <summary>`），一秒就能看懂有哪些 API，无需关心底层实现。


* **`.cpp` 源文件（只做实现）**：
* 引入对应的头文件：`#include "Person.h"`。
* 编写具体逻辑，函数名前需标明所属类作用域：`void Person::setName(string name) { ... }`。

## 3. 核心机制：构造、析构与参数传递

### 构造函数 (Constructor)

用于在对象创建时进行初始化。与类同名，无返回值。

1. **默认构造函数**：`Person() {}`。若不写，编译器会自动生成一个空的。
2. **有参构造函数**：`Person(string n, int a) { ... }`。用于在创建时直接赋初值。
3. **复制构造函数**：`Person(const Person& p) { ... }`。**极其重要！** 用于从同类对象完整拷贝一份新数据，防止多对象共用同一套指针带来的交叉污染。

### 析构函数 (Destructor)

标志符为波浪号 `~`，例如 `~Person()`。在对象生命周期结束时**自动调用**。

* **核心使命：擦屁股（释放内存）**。如果在构造函数或类运行期间 `new` 了指针对象（如动态数组、特殊容器），**必须**在析构函数中手动调用 `delete` 将其释放，否则会导致严重的内存泄漏。也会用于断开数据库连接或释放文件句柄。

### 传参优化：`const` 引用传参

* **反面教材（值传递）**：`void setName(string name)`。会把外部传入的字符串在内存里再完整复制一份，如果传入的是体积巨大的类对象，极度消耗性能。
* **终极解法（const 引用）**：`void setName(const string& name)`。
* `&`（引用）：直接传入原对象的内存地址，省去深度拷贝开销。
* `const`（常量限定）：给原对象上了一把“锁”，保证函数内部绝对无法修改传入的原始数据，安全高效。

## 4. 友元函数 (`friend`)

* **作用**：让特定的外部函数或其他类成为当前类的“好朋友”，从而**赋予其直接访问当前类 `private` 私有成员的特权**。
* **声明方式**：在类内部写明 `friend void changeName(Person& p, string newName);`。
* **注意**：由于 C++ 是顺序编译的，涉及到多个类/函数互相引用的情况，必须利用头文件做好**前向声明**，否则编译器遇到不认识的类名会直接报错。

## 5. 底层性能核武：数据对齐与极限压缩 (CG/GPU 编程必备)

在 GPU 编程等海量数据（几十万级面片/顶点）场景下，一个字节的浪费都会被放大几十万倍，导致性能暴跌。

### 结构体内存对齐 (Memory Alignment)

* **默认规则**：结构体会以**内部最宽的基础成员变量**（如 `double` 占 8 字节）作为对齐基准（对齐模数）。这就导致较小的变量（如 1 字节的 `char`）后方会填充大量无用的“空白内存（Padding）”，导致空间浪费率极高（可能超 50%）。默认这样设定是为了“空间换时间”，让 CPU 寻址变快（做乘法即可快速定位）。
* **手动干预对齐方式**：
1. **`#pragma pack(n)`**：强制设定整个结构体按 `n` 字节对齐（如 `#pragma pack(1)` 取消所有缝隙，体积最小，但寻址变慢）。**最强大，但也最容易产生跨平台兼容问题。**
2. **`alignas(n)`**：C++ 标准安全方法。可针对单个变量或类使用，但其设定的对齐值**不能小于**变量默认的对齐值。
3. **位域 (Bit-fields)**：打破字节(Byte)限制，直接按位(Bit)分配！
* 例如星期只有 7 天，只需要 3 个 Bit（$2^3=8$），可写为 `int weekday : 3;`。极致压缩，但要注意数值溢出风险。





### 联合体 (`union`)

* **核心特性**：多个不同类型的变量**共享同一块内存空间**。整体大小等于其最大成员的大小。
* **机制**：修改联合体中的任一变量，其他变量的值也会跟着变（因为读写的是同一块物理内存）。
* **应用场景**：某个对象有多种可能的数据形态，但**同一时刻绝对只会使用其中一种**时，用 `union` 可以极大节省内存。
* **限制**：内部不能存放自带复杂构造/析构函数的对象（如 `std::string`），但可以放它们的指针。

## 6. 课堂实战作业：Time 类与运算符重载

* **目标**：实现一个包含小时、分钟、秒的 `Time` 时间类。
* **核心动作**：
* 提供 `confirm()` 方法用于合法性格式化（满 60 秒进 1 分钟，满 24 小时归零等）。
* 重载核心运算符：`+=`、`-=`、`++`、`--`、输入输出流 `<<` `>>`。


* **运算符重载规范**（以 `+=` 为例）：
```cpp
// 返回自身的引用，以支持连加操作 (a += b += c)
Time& operator+=(const Time& t) {
    this->hour += t.hour;
    this->minute += t.minute;
    this->second += t.second;
    this->confirm(); // 调用内部格式化方法
    return *this;    // 返回自身
}

```
