# C++ 标准库与核心算法 (第二节课笔记)

## 📌 课程核心目标

本节课主要介绍 C++ 标准库（Standard Library）中**最基础且最常用**的模块，包括输入输出流、文件读写、容器（相当于数据结构）以及通用算法。重点在于建立全局印象，知道“遇到什么需求该去调用什么库”。

## 1. 标准输入输出流 (`<iostream>`)

基础的流式交互操作，在没有 GUI（如图形化界面/Unity）的命令行程序中最为常用。

* **四大核心对象**：
* `cin`：标准输入。
* `cout`：标准输出。
* `cerr` / `clog`：标准错误输出和日志输出（类似于 Unity 中的 `Debug.LogError` 和 `Debug.Log`）。


* **格式化输出**（常用于命令行制表、对齐）：
* **`setprecision(n)`**：设置有效数字/精度（对浮点数）。**注意：** 设置一次后，对后续所有输出永久生效，直到被再次修改。
* **`setw(n)`**：设置输出间距/宽度。**注意：** 仅对紧跟其后的**下一个**数据有效。
* **`setfill(char)`**：设置空白处的填充字符（默认为空格，可改为 `-` 等）。设置后全局生效。

## 2. 文件读写流 (`<fstream>`)

C++ 使用流式操作（`<<`, `>>`）进行本地文件的读取与写入。

* **三大核心类**：
* **`ifstream`** (Input File Stream)：只读文件流。
* **`ofstream`** (Output File Stream)：只写文件流。
* **`fstream`**：可读可写文件流。


* **文件路径**：若不写绝对路径，默认以当前运行的应用程序（`.exe` 或 `.cpp` 所在工作目录）为准寻找相对路径。**讲师建议**：为了防止读不到文件，测试时尽量使用绝对路径。
* **读取机制**：流式读取会自动识别**空格**和**换行符**并进行截断/分词。例如文件里有 `15 25 151`，流会依次把这三个数读取并赋给不同的变量。
* **⚠️ 强制好习惯**：文件读写结束后，**务必调用 `close()` 关闭文件流**。否则会一直占用文件权限，导致其他程序无法修改甚至数据丢失。

*(注：由于纯文本流读写在处理复杂格式时较繁琐，现代游戏开发中除特定桌面应用外，直接操作原生文件的场景相对较少。)*

## 3. 标准模板库 (STL) - 核心容器

容器需要用到什么就 `#include` 什么。讲师详细讲解了四大类容器的使用场景。

### 3.1 顺序容器 (Sequence Containers)

数据在容器中按线性的顺序排列。

1. **`vector` (动态数组)**：**🌟最常用🌟**
* 会自动扩容的数组（类似 C# 的 `List`）。
* 方法：`push_back()` (尾部插入), `pop_back()` (尾部删除), `at()` (安全下标访问), `size()` (大小)。
* **讲师极力推荐**：日常开发完全可以用 `vector` 替代原生数组，不仅支持动态长度，而且带有安全检查，防止越界导致程序/系统崩溃。


2. **`array` (定长数组)**：C++11 引入的安全版原生数组。固定长度，不可增删，只能通过下标修改/读取。
3. **`deque` (双端队列)**：`vector` 升级版。支持头部增删（`push_front`, `pop_front`）和尾部增删。
4. **`list` (双向链表)**：基于指针实现，内存空间不连续。可以在任意位置高效插入/删除，但**无法通过下标直接随机访问**，只能从头到尾顺序遍历。
5. **`forward_list` (单向链表)**：只能从头开始单向访问。增删函数较为特殊（如 `insert_after`, `erase_after`），切断后续节点。仅在极端关注头部性能时使用。

### 3.2 关联容器 (Associative Containers)

非线性结构，元素不按插入顺序排列，而是**根据键值 (Key) 自动排序**（通常底层为红黑树/二分查找），查找效率极高。

1. **`set` (集合)**：
* **特点**：元素绝对唯一（自动去重），并自动排序。
* **方法**：`insert()` (插入), `erase()` (删除), `find()` (查找，返回迭代器)。


2. **`map` (字典)**：**🌟非常常用🌟**
* **特点**：存储键值对 (`Key-Value`)。通过 Key 自动排序，Key 必须唯一。类似 C# 的 `Dictionary`。
* **使用**：插入需要使用 `make_pair(key, value)`。支持便捷的下标访问（如 `m["Alice"] = 20`）。
* **迭代器特性**：遍历或 `find` 得到的迭代器，需要使用 `it->first` 获取键(Key)，`it->second` 获取值(Value)。
* **注意**：下标访问虽然方便，但如果 Key 不存在会报错（越界），不确定是否存在时最好先用 `find`。


3. **`multiset` / `multimap**`：
* 允许插入**重复**的元素或键值对。
* 注意点：`erase` 会删掉所有同名的 Key；`find` 会返回连续相同 Key 序列中的**第一个**迭代器。



### 3.3 容器适配器 (Container Adapters)

基于原有容器封装出的特殊结构，通常用于特定算法逻辑。

1. **`stack` (栈)**：
* **后进先出 (LIFO)**：只能访问栈顶。`push` (压栈), `pop` (弹栈), `top` (看栈顶)。
* **应用**：计算器表达式解析、正则表达式、深度优先搜索 (DFS)。


2. **`queue` (队列)**：
* **先进先出 (FIFO)**：只能从头出，从尾进。`push` (排队), `pop` (出队), `front`, `back`。
* **应用**：网络消息队列处理、广度优先搜索 (BFS)、寻路算法 (如 A*)。



### 3.4 容器通用遍历法 (迭代器 & C++11 语法)

* **迭代器遍历**（万能写法）：
```cpp
for(auto it = map.begin(); it != map.end(); it++) { ... }

```


* `begin()`：指向第一个元素。
* `end()`：指向容器末尾的标志位（**本身没有元素**）。


* **范围 for 循环** (C++11 新特性，等同于 C# 的 `foreach`)：
```cpp
for(auto x : v) { cout << x; } // 极大简化了遍历代码

```

## 4. 通用算法库 (`<algorithm>`)

算法库提供了高度封装、安全可靠的通用算法，配合容器的迭代器使用。

* **排序 (`sort`)**：`sort(v.begin(), v.end())`。进行全排序（默认按从小到大）。若需自定义排序逻辑，可传入 Lambda 表达式。
* **部分排序 (`partial_sort`)**：`partial_sort(v.begin(), v.begin()+5, v.end())`。出于安全机制，必须同时传入末尾迭代器以防止越界。
* **查找 (`find`)**：通用查找，返回迭代器。若没找到，会返回 `v.end()`。
* **计数 (`count`)**：统计某个特定元素在容器中出现了多少次。
* **拷贝 (`copy`)**：`copy(src.begin(), src.end(), dest.begin())`。
* **⚠️ 危险点**：目标容器必须**提前分配好足够的容量**！如果目标容器容量不足，`copy` 会引发内存越界报错，它不会自动帮你扩容。


* **变换 (`transform`)**：遍历源容器的数据，经过指定函数（如 Lambda）处理后，存入目标容器。相当于集成了遍历、处理、拷贝三大操作，非常实用。
