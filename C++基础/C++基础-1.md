## 1. C++ 与 C# 的核心区别

作为有一定 Unity 基础的开发者，理解 C++ 与 C# 的差异是第一步：

* **编译方式**：
* **C# (动态/受管)**：可以直接调库，支持灵活的热更新和打 Mod（运行时注入组件）。
* **C++ (静态)**：编译后即固定，添加新内容必须重新编译（类似 Unity 的重新打包），在通用性和实时注入组件方面不如 C# 灵活。


* **内存管理**：
* **C#**：有完善的垃圾回收机制（GC），对象不使用时会自动释放内存。
* **C++**：内存大多需要**手动管理**。如果申请了内存（如使用指针）但不手动释放，会导致严重的**内存泄漏 (Memory Leak)** 或溢出问题。

## 2. 认识第一个 C++ 程序

与 Unity 将 `main` 函数隐藏并封装成了 `Start`/`Update` 不同，原生的 C++ 程序必须有一个显式的入口。

```cpp
#include <iostream>

int main() {
    std::cout << "Hello World" << std::endl;
    return 0;
}

```

* `#include <iostream>`：引入标准输入输出流库，类似 C# 中的 `using System;`。
* `int main()`：程序的唯一入口。程序运行的第一行必然是 main 函数的第一行。
* `std::cout`：标准输出流。`std::` 是命名空间，`cout` 是输出方法。
* `<<`：流式传输符号（流式符），用于按顺序向控制台输出内容。
* `std::endl`：换行符（等同于回车/`\n`）。

## 3. 文件结构与预处理机制

### 头文件 (`.h`) 与源文件 (`.cpp`)

* **设计目的**：为了方便阅读和规避重复定义的问题。
* **`.h` 头文件**：负责**声明**（如函数名、参数类型）。
* **`.cpp` 源文件**：负责**实现**具体的逻辑。`cpp` 文件需要 `#include` 对应的头文件。
* **重复定义问题**：如果多个文件互相 `include`，会导致头文件被多次编译。
* *老式解法*：使用 `#ifndef`、`#define`、`#endif` 宏定义将内容包裹起来。
* *新式解法*：在头文件首行加入 `#pragma once`（现代编译器自动生成），保证只编译一次。



### 命名空间 (Namespace)

* **作用**：防止变量或函数重名。
* **解析运算符 (`::`)**：C++ 中使用双冒号来定位命名空间（如 `std::cout`），而不是 C# 中的点（`.`）。
* **`using namespace`**：例如 `using namespace std;`，告知编译器默认在此命名空间内寻找方法，从而省略 `std::` 前缀。也可以单独指定某个方法：`using std::cout;`。

### 宏定义 (`#define`)

* **作用**：在程序**预编译阶段**进行单纯的“字符串替换”。
* **常见用法**：
* 定义全局常量：`#define PI 3.1415926`。比全局变量更安全，因为常量值无法在运行时被恶意修改。
* 字符串化操作 (`#`)：把标识符转换成字符串。
* 拼接操作 (`##`)：将两个标识符拼接成一个新的变量名。


* **讲师建议**：宏定义能简化代码，但嵌套过多特殊符号会导致代码可读性极差，难以 Debug。新手不建议过度使用复杂的宏。

## 5. 内存管理：堆 (Heap) 与栈 (Stack)

* **栈区 (Stack)**：
* 由**编译器**自动管理（分配与释放）。
* 存储函数、局部变量、参数等。
* **战溢出 (Stack Overflow)**：通常由于死循环或无限递归引起。


* **堆区 (Heap)**：
* 由**程序员**手动管理（动态分配）。
* 使用 `new` 申请内存，使用 `delete` 释放内存。
* **内存泄漏 (Memory Leak)**：如果 `new` 了一块内存，但在程序结束前忘记调用 `delete`，这块内存将永远被占用。
* **讲师建议**：写下 `new` 的同时，立刻在合理的位置写下配套的 `delete` 和 `指针 = nullptr`，养成良好习惯。

## 6. 核心难点：指针 (Pointer) 与引用 (Reference)

### 原始指针

指针本质上是一个**内存地址**。可以直接绕过编译器直接访问和修改计算机内存，效率极高。

* **定义**：`int* p;`
* **初始化**：极其重要！必须指向有效的地址（`int* p = &a;` 或 `new int` 或 `nullptr`）。**绝对不能出现未初始化的“野指针”**，否则可能导致程序崩溃或破坏其他数据。
* **解引用 (`*`)**：使用 `*p` 来访问指针指向地址里存放的具体数值。
* **指针运算**：指针加减（如 `p + 1`）会根据**数据类型的大小**向前/后移动内存地址。
* **数组即指针**：C++ 中，数组名本身就是指向第一个元素的指针（`a[7]` 等同于 `*(a + 7)`）。

### 智能指针 (C++11 新特性)

为了解决忘记 `delete` 导致内存泄漏的问题，C++ 引入了智能指针（需 `#include <memory>`）。**强烈建议现代 C++ 开发中优先使用智能指针**。

1. **独占式 (`unique_ptr`)**：同一时间只能有一个指针指向该对象。生命周期结束时自动销毁内存。
2. **共享式 (`shared_ptr`)**：多个指针可以指向同一个对象。当最后一个使用该对象的指针被销毁时，内存才会被自动释放。

### 左值引用 (`&`) 与右值引用 (`&&`)

* **左值引用 (`&`)**：
* 针对等号左侧的变量（常态变量）。
* 本质是对原变量内存地址的引用。对引用的任何修改都会直接反映在原变量上。常用于**函数参数的引用传递**，避免传参时发生拷贝。


* **右值引用 (`&&` - C++11)**：
* 针对等号右侧的常量或临时表达式。
* **移动语义 (`std::move`)**：跳过一次深度拷贝，直接将大型对象从一个地址“偷”到新地址，极大节省性能开销。
* **完美转发 (`std::forward`)**：主要用于泛型模板中，根据传入参数的原始类型（左值还是右值）原样向下转发。

## 7. 运算符与控制流

* **成员访问**：实例对象使用点（`.`）；指针访问对象成员必须使用箭头（`->`）。
* **位移运算符 (`<<`, `>>`)**：对二进制位进行整体左移/右移（一般相当于乘以/除以 2）。
* **按位运算符 (`&`, `|`, `^`)**：逐位进行逻辑运算。
* *实战场景*：经常用于**状态机**或复合状态判定（例如 Unity 里的 LayerMask），开销极低。


* **条件运算符 (`? :`)**：简洁版的 `if-else`。`条件 ? 满足执行此行 : 不满足执行此行;`
* **运算符重载**：使用 `operator+` 等关键字自定义类的运算规则。通常传入 `const` 引用以防止修改原参数。

## 8. Lambda 表达式 (匿名函数)

C++11 引入的便捷特性，适用于仅临时使用一两次、不需要全局定义的轻量级函数。

* **基础语法**：`[捕获列表](参数列表) -> 返回类型 { 逻辑体 }`
* **捕获列表 (`[]`)**：
* 用于捕获 Lambda 外部的变量，使其在内部可用。
* 传值捕获：`[x]`
* 引用捕获：`[&x]`（内部修改会影响外部）


* **特点**：随取随用，生命周期随所在作用域结束而立即销毁；可以作为参数传递，甚至嵌套调用（套娃）。
